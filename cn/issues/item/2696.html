关于DEFI综合指数

 交易常见问题    |      2020-08-26
1.0 關於DEFI綜合指數
指數是一種以標準化的方式跟蹤和衡量所對標的一籃子資產表現。幣安指數主要包含兩種：單一品種價格指數與綜合市場指數。單一品種價格指數反映某個品種（幣種）在不同市場的綜合價格，而綜合市場指數則體現整體市場或其代表性子集若干品種的綜合價格，指數價格計算方式通常爲這些市場品種的價格加權平均。
幣安合約推出的第一個指數是 DEFI綜合指數合約。DEFI綜合指數的底層標的資產是由在
幣安上線的去中心化金融（DeFi）幣種綜合指數組成的綜合價格指數。DEFI綜合指數是以加權平均價格計算指數價位，以 USDT 爲結算單位。
DEFI綜合指數的幣種成分會隨情況進行調整，具體可參考文章中【4.0 指數成分和權重調整】部分。
請點擊這裏查看最新DEFI綜合指數信息，包括價格與成分信息。

3.0 指數計算與公式
計算方法：
第一步
計算權重因子需要參考以下表格中的信息 （表格每週四更新，請參照此幫助文檔中最新信息 “DEFI綜合指數成分構成加權因子”）：
權重因子公式：
以Band爲例：
√市場價值 Market Cap = √228138790 = 15104.263967502686
√永續合約交易量 Perpetual Volume = √462009026 = 21494.395222941257
相加 15104+21494=36598 ， 除以1000， = 36.598
用這個方法計算出10個幣種，相加= 484.947， 然後用36.598/484.947 = 7.55% (即F列）
計算出來每一個幣種在整體index裏面的權重（因子，代表初始或調整時的價值佔比）。
第二步
權重 (數量) Wit = 指數價格（index）x 權重因子weighting factor / Pit （i這個幣在t時點的價格）
假設t時點的指數價格index爲1000點。
以Band爲例：
Wit = 1000* 7.55% / 12.61（G列，在rebalance時候單一幣種指數價格，也就是U本位那些合約用的指數）= 5.98464
這裏可以理解爲index裏面配置5.98464個band 幣。
第三步
Divisort爲當期穩定除數，當直接給定權重（成分幣數量）而不是權重因子（成分幣價值佔比）時需要用到，保證指數成分和權重調整時能穩定連續變化，使指數點位僅受成分價格影響，對於基期：
計算最新成分值 Latest value（pi * wi，用於求和計算指數點位）, 即J列， J列求和其實就是最新的指數價格（在除數 divisor 不變，保持初始值即1000的情況下）。
以Band爲例：
最新成分值 Latest value = pi * wi = 13.1998 * 5.98464= 78.996 USDT
第四步
計算指數：
由於當前直接用權重因子計算，除數本身沒有變化，還是初始值1000, 所以index t = Pi*Wit (*1000/1000) 的求和，所以即爲表格種J列的求和，得出最終指數(Index)的數字。
以圖表中計算結果爲例：
指數(Index) = 78.99625274 + 73.492 +69.297+ … +96.076 = 1007.821
4.0 指數成分和權重調整
4.1 定期成分調整
指數成分及權重將會定期調整以體現最新市場情況，進行季度或月度定期調整。
4.2 特殊處理調整
當成分面臨重大變故（例如，下架/退市或者長期無交易），認定需要特殊處理時候，則有兩種調整方法：直接剔除並重新歸一權重（重算Divisor t）；剔除後利用備選名單樣本補足成分，也需要重新計算權重。
這裏舉例討論一下除數Divisort 需要變動來使得指數連續變化的情況。

所有幣種的最新成分值（特殊調整後）Latest value加總，得884.89 （H欄加總）

大家可以看到K3 和K5 兩欄的數字是一樣的，意味着即使我們調整了一個幣種的權重（weight）， 那麼我們可以用除數divisor這個東西來保證指數還是以原來的價格繼續運行。
4.3 區塊鏈分叉/空投/映射事件調整
由於加密貨幣指數成分區塊鏈的特性，在成本幣種發生分叉/空投/映射事件時候按照以下原則處理：
  • 軟分叉或者不產生新幣種的硬分叉事件不作調整
  • 硬分叉/空投產生新幣種且新幣種穩定交易（有穩定價格來源）的情況下，適時將新幣種按照原比重權重/當期數量加入指數計算，並在下一次調倉期依據指數目標確定是否剔除
  • 空投/映射事件中，不產生新的幣種而由原有幣種按照一定的比例轉換爲新幣種的，則將該成分幣種替換爲新幣種，並將當期權重乘以映射比例進行計算，在下一次調倉期依據指數加權目標進行正常調倉
