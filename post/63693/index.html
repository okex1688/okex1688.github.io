共享安全，就是一条乃至多条区块链可以通过共享另一条区块链的「去中心化信任源」，增强自身的安全性，实现去中心化的启动和运行。这降低了区块链从 0 开始的启动门槛和运行成本，并有助于防止财阀统治，可以让团队更专注在 Web3 的价值创新上。

众所周知，区块链是信任的机器，区块链的根本价值就是去中心化信任。 公链之所以一直处在聚光灯之下，就在于其商业模式就是：为自己链上的智能合约提供去中心化的信任。

也就是说，去中心化信任是公链提供给智能合约的一种服务，我们链上交互时支付的 gas，就是为这份去中心化信任买单。估计绝大多数人都没有意识到公链的这个本质，正因为此，整个公链赛道才能拥有超万亿美金的估值。

共享安全服务，就是要让「去中心化信任」从一条区块链扩展到更多的区块链之上。让原来必须以 PoW/PoS 方式自建的去中心化信任，成为了可以随时灵活购买的第三方服务。

至此，去中心化信任第一次变得和云计算云储存一样可以灵活租用，成为了 Web3 项目运营的一部分成本而非目标。团队可以更加专注在业务创新本身之上。

与此同时，去中心化信任肩负着链的安全，随着区块链越来越多，链上资产越来越多，去中心化信任的市场规模将会变得极其庞大，我认为其规模至少是千亿美元起步。

目前，共享安全赛道知名度较高的项目主要是 Polkadot、Octopus Network、Cosmos、EigenLayer 和 Babylon。这些项目的解决方案各有特点，优缺点不能一概而论，应该说都有其对应时代背景和垂直场景。

Polkadot

服务波卡生态平行链的去中心化信任的自卖模式

2016 年创立的 Polkadot，是最早提出并实现了共享安全的项目。
Polkadot 的共享安全机制是，平行链竞拍插槽接入中继链，由中继链的验证节点提供安全性，从而获得中继链相同的安全性。这套方案可以让 Substrate 链直接获得 Polkadot 几十亿美元级别的安全性，解决了新区块链安全性自举时间周期长的问题，但是因为只能选择 Polkadot 非常高等级的安全性，再加上拍卖出售的机制，导致新链启动成本高昂，对资金实力和团队精力都很宝贵的初创项目极不友好。

近几年的实践证明，绝大多数波卡平行链无需这么高级别的安全性， 2 年独占插槽的方式也让中继链的「去中心化信任」资源的利用效率非常低，显然 Polkadot 也意识到了这个问题。

2023 年 6 月 28 日，Polkadot 2.0 希望构建一种从时间周期和安全等级都更为灵活的去中心化信任资源的分配方式。 基于中继链的「区块空间」这个最小单元资源，让  Substrate 链使用 DOT 按需购买「区块空间」的数量和使用时长，实现去中心化安全的灵活获取。

10 月 25 日，波卡中继链的测试网 Rococo 上已经实现了按需购买中继链区块空间的能力，正在准备测试，替代「插槽」的「敏捷核心时间」预计在 2024 年第一或者第二季度实现。

可以看到，Polkadot 都是将自身中继链的去中心化信任当作商品，销售给加入生态的 Substrate 链，无论是 1.0 时代的 DOT 质押，还是 2.0 时代的购买。

事实上，Polkadot 是所有共享安全赛道中，唯一选择将去中心信任进行直接销售的项目。

Octopus Network

专注服务应用链（没有区块链生态限制）去中心化信任的双边市场 

2019 年创立的 Octopus Network，专注为应用链提供共享安全服务， 首创了 LPoS 租用权益证明机制。

Octopus Network 构建了一个买卖去中心化信任的安全性双边市场，应用链是安全性的需求方也是购买方，用原生通证作为租金，支付给验证人节点即可，验证人就是去中心化信任的安全性提供方。章鱼网络的这套安全性双边市场的底层逻辑，基本上奠定了后续共享安全服务的基本范式。

章鱼网络 V1 阶段，Substarte 应用链的验证人用章鱼网络的 $OCT 做质押， 保障应用链的安全性，就是应用链租用共享了章鱼网络$OCT 的安全性。

章鱼网络 V2 阶段，在 V1 的基础上，增加了 $NEAR Restaking 机制，使用 $NEAR 为基于 Cosmos SDK 的应用链做  Restaking 从而提供安全性。计划于 2023 年第四季度计划上线。

本质上，Restaking 是一种共享安全机制，由 Eigenlayer 首创。可以让相同的资产同时质押到多个「去中心化应用/区块链」上，为其提供安全性。其突破性的创新在于在于实现了「去中心化应用/区块链」、「提供共享安全的区块链」和「质押者」三者利益最大化。

——《一文读懂 Restaking》

Octopus Network 的共享安全方案最大的优势就是专注和灵活。一举解决了原有 PoS 机制安全性自举成本高和周期长的问题，V2 时代 $NEAR Restaking 的加入，更是让可以提供的安全性等级同步提升到十亿美元级别。

专注，在于专注服务一个应用独享一条链的应用链。
灵活，在于安全性级别的可选择性。

事实上，Octopus Network 已经将 NEAR 打造成一个极具竞争力的区块链互联网的胖枢纽，成为 NEAR 不容忽视的潜在竞争力。

Cosmos

由 Cosmos 社区管理决策的去中心化信任的分配机制

Cosmos 创立于 2014 年，最早提出了多链网络概念，但是直到 2023 年也就是今年 3 月 15 日，才发布 Replicated Security 复制安全机制，补上了共享安全解决方案这块短板

Cosmos 复制安全的核心逻辑就是将所有 Cosmos 验证人看作了一个整体，社区投票同意为消费者链提供安全性后，验证人可以用已经质押在 Cosmos 里的$ATOM，再为 Cosmos 生态消费者链单独跑验证节点，实现去中心化安全。

这套方案参考了章鱼网络双边市场的构建逻辑，同样也集结了 Eigenlayer 提出的 Restaking 思路，但在灵活度方面差异很大，这就带来 2 个问题：

1、消费者链的价值需要能赋能 Cosmos hub，才能争取到足够多投票获得社区的同意。比如作为  Cosmos 生态的 DeFi 中心的 Neutron。

2、消费者链需要为 Cosmos hub 20 亿美元级别的安全性提供租金，势必带来巨大的经济压力。前一段时间 Cosmos 社区有一个很有趣的提案：基于复制安全启动和运行的流动性质押项目 Stride ，提议并入 Cosmos Hub，$STRD 全部转换为 $ATOM，成为 Cosmos Hub 的卫星链。虽然后续不了了之，但是也为复制安全这套解决方案蒙上了一层阴影。

EigenLayer

服务以太坊中间件的去中心化信任的双边市场

2021 年创立的 EigenLayer，将以太坊级别的信任引入中间件：允许将已经质押在以太坊上的 ETH，再次质押在以太坊中间件上，让其共享到以太坊几乎不可撼动的经济安全性。 EigenLayer 首次提出 Restaking 机制，这也是上面提到章鱼网络 2.0 版本中 $NEAR Restaking 机制的创意来源。 

其优势主要在于两个方面：

1、专注于以太坊中间件的共享安全解决方案，弥补整个以太坊生态经济安全性的短板。

2、可以灵活调整安全等级，中间件协议都可以按照自身不同发展阶段的安全性需求，灵活地获取和调整质押量和安全等级。

Babylon

基于 BTC，拥有最大去中心化信任池去中心化信任的双边市场

2022 年创立的 Babylon，把目光瞄准了比特币。Babylon 是一条独立的 PoS 区块链，一方面，比特币持有者通过 Babylon 将他们的比特币 Staking 以及 Restaking 到需要安全性的区块链上，获取安全性奖励；另一方面，获得安全保障的区块链也在 Babylon 上进行安全性的管理和控制。

BTC 即便是当下深熊，也是一种超过 5000 亿美元的资产，加上 PoW 工作量证明天然的安全优势，以及 BTC 大量闲置的流动性，这就成为了 Babylon 的核心价值，即拥有区块链世界无可争议的最大去中心化信任池。

梳理优劣势：

基于「共享安全」的核心优势

对于新的去中心化应用/区块链而言（安全性需求方）

1、安全性获取成本低，避免应用链 Token 过度增发稀释价值。

众所周知，独立 PoS 通常需要每年增发 10% 左右的 Token 奖励验证人群体，高增发稀释了 Token 价值，在二级市场形成持续的抛售压力。

2、安全性等级可以灵活调整。

区块链都可以按照业务场景和发展阶段需求，灵活设定应用链的安全等级。一般来说可以早期安全等级更低一些，等到应用链经济规模和资产规模扩大，通过链上治理形成提高安全租金的决议，然后技术团队调整应用链的出块奖励，就可以很快获得更高等级的安全性。

3、验证人招募难度低，可以更快速地启动区块链。

启动阶段，无需让原生 Token 具备了价值共识再去做验证节点就，这就让团队至少减少数以年计的时间和数以百万美元的资金，跳过验证人群体招募阶段，省时省力，实现应用链快速安全启动。

对于质押者而言（安全性提供方）

1、质押风险更低

用具备价值共识的 $DOT、$OCT、$NEAR、$ATOM 乃至更健壮的 $ETH 和$BTC 来做质押，相较于质押应用链原生 Token ，资产贬值的风险更低。

2、质押收益更高

Restaking 和 复制安全机制，可以让质押者获取多重收益。

除此之外，其实还有一个极其重要，但是不易被发现的深层优势，那就是「避免区块链被财阀统治」。

传统 PoS 的验证人群体由于持续获得增发 Token，逐渐控制大部分治理权成为财阀。甚至提案投票提高出块奖励，增加自身收入和链的安全成本。而采用共享安全机制的区块链，验证节点并不质押有治理效用的原生 Token，这就从根本上解决了验证人控制区块链的问题。采用共享安全机制的区块链，其验证人只是纯粹来赚取奖励，不会影响新生区块链的治理和决策，也就避免了区块链被验证节点统治的尴尬境地。

创新需要不断迭代，需要更低的启动门槛和试错成本。

新去中心化协议和区块链的启动，采用共享安全方案将会是大势所趋，毕竟只有指数级的降低区块链的启动风险和运行成本，才有机会再次加速 Web3 的创新。

由此可见，共享安全必将成为加密领域最重要的基础设施服务之一。
