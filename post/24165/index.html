1. 什么是Swarm？它能解决什么问题？

Swarm是一个能集中提供去中心化储存和沟通服务的点对点节点网络。内置的激励系统能保障Swarm的经济可持续性，而激励系统通过以太坊区块链和BZZ通证来实施。

Swarm促进信息自由，以响应对网络安全性日益增长的需求。Swarm解决了去中心化激励存储的问题。这意味着节点运营商将获得奖励，以保持网络健康和完美运行。至于用例：新闻自由、可扩展的Dapp托管和真正的去中心化社交媒体平台都是唾手可得的成果。

2. 团队为何决定把Swarm建立在以太坊之上？

Swarm最初是一个以太坊基金会项目。这主要是 Gavin Wood 的心血结晶。当创始人完全实现Web3的这一愿景时，一个世界级的计算机将变成类似以太坊、Swarm和Whisper的三位一体。

我们生态系统中的大多数人都熟悉以太坊，它是世界计算机的处理部分。而Swarm负责存储，Whisper旨在传递信息。Swarm和其他各方都采用了Whisper的概念并将其集成到他们的解决方案中。在Swarm的案例中，消息层变成了一个完全去中心化的抗审查消息系统，具有木马功能。这意味着不仅消息的内容是秘密的，而且消息的发送者和接收者也会保密。因此，表示消息的那段数据不会被识别为信息。

2015年，Swarm迫切致力于在基金会的Geth团队内构建存储和消息传递系统。四年后，Swarm经历了一个阶段性转变。为了补充文件存储的原始范围，它很快就涵盖了免许可发布和私有数据交换的方方面面。

去年，Swarm从以太坊基金会“毕业”并成为一个独立项目。虽然 Swarm 的目标是成为一个多链系统，但目前该团队的工作重点是以太坊。

3. Swarm的目标用户是谁？这些用户在你的平台上有什么样的互动？

Swarm是一个底层基础设施项目，旨在成为重新去中心化互联网的基础。这是一个涉及全人类的项目。因为Swarm旨在解决当今互联网面临的最大问题之一。

对于开发内容和应用程序的人来说，存储成本一直是最大的负担之一。是的，任何人都可以创建博客，从而在全球辩论中发表意见，但是随着内容的日益流行，成本会上升，直到原始创建者无法再发表他们的意见。在Web 2.0 中，这个问题以一种对用户来说不是最佳的方式解决了：大公司支付了存储和带宽的费用，作为回报，他们要求拥有数据。

用户的数据最终在数据市场上出售，为控制带宽和存储容量的公司赚取了巨额利润。隐私被排除在外，亚马逊、Facebook和谷歌等公司则获得了权利。人变成了他们的产品，用户的个人数据被出售给营销和研究公司。作为回报，用户获得了Facebook和Google等免费服务。

有了Web3，新的范式和经济成为可能。但在建立一个可靠的、分散的存储系统之前，即使是Web3 DApp也需要将数据存储在集中控制的环境中。

Swarm的使命就是为Web 3提供这样的解决方案。用户可以自己决定是将他们的数据保密，是在数据市场上出售，或是捐赠给他们自己选择的研究机构。

由于存储和带宽不是由开发人员/内容创建者支付的，他们可以专注于创建好的内容和服务，从而能够快速采用。

4. Swarm如何在Swarm网络中分发和存储数据？它如何确保这些数据的持续可用性？

DISC（Distributed Immutable Store of Chunks）是Swarm的底层存储模型。它由协作存储和提供数据的节点组成，使得利润最大化节点遵循的策略集合能有以下属性：

隐私保护和未经许可的上传和下载；

一旦特定内容上线，就有强大防御来阻止或更改对特定内容的访问；

随着需求的增加自动扩展；

完整性保护内容；

最终忘记不再与保存相关的内容；

任何拥有备用存储和带宽容量的人都可以作为节点运营商参与 DISC 并因此获得奖励。加入 DISC 后，节点运营商成为网络的一部分，基本上负责管理这个全球硬盘驱动器的一个扇区。

Swarm中的规范存储单元称为块。一个块由4 KB的数据组成，并有一个地址。由于块的地址与节点地址包含在相同的地址空间中，因此可以计算它们的相对接近度。 Swarm的存储方案声明每个块由地址接近块本身的节点存储。

Swarm节点不会选择它们存储的块。一个块可能是一个或多个内容的一部分，而节点运营商没有简单的方法来确定一个块属于什么内容这一事实为节点运营商提供了保护，使其免于对其托管的内容承担责任。

DISC 最终会忘记很少访问且未付费的内容。通过固定，节点可以确保在本地保留特定内容。此外，此类节点可以参与该内容的被动或主动恢复。

5. Swarm 网络如何实现公平的数据经济？ 换句话说，使用Swarm和运营网络节点的动机是什么？

Swarm Accounting Protocol (SWAP) 确保节点运营商有动力在路由消息中进行协作，同时提供针对无意义消息传递的保护。

当节点协作路由请求和响应时，它们会与对等方一起跟踪带宽的相对消耗。 在一定范围内，这可确保对等方参与服务对服务的交换。

一旦达到限制，使用超过服务的对等方可以等到其债务随时间摊销或通过发送支票来支付，代表 BZZ。

6. 您最感兴趣的 Swarm 用例有哪些？

作为开发团队，我们对流经网络的任何数据感到兴奋。但社交媒体平台、新闻自由和信息共享、激励内容创建 (NFT)、安全通信 (Waggle) 似乎是唾手可得的成果。

举几个例子，这里有一些已经在 Swarm 上构建的项目：

ZetaSeek 是一个搜索引擎，其主要关注点是供每个用户单独使用——将他们的文件、链接和上传内容中的引用组织到即将到来的 p2p 存储网络；

Upala 正在为 dApp 构建一个反 Sybil 系统。它提供以美元计价的数字身份唯一性分数；

GetLoginEth 是一个 Đapp，用于为基于以太坊的社交应用程序提供去中心化的单点登录解决方案；

Waggle - 在 Swarm 之上的无垃圾邮件、安全和分散的电子邮件服务，其灵感来自 Daniel J. Bernstein 提出的邮件传输协议的想法，在该协议中，发件人而不是收件人负责存储电子邮件；

Swarmify - Swarmify 希望使用 Swarm 进行分散和公平的音乐流媒体；

RISC-V 上的嵌入式 Swarm - 想象每个冰箱、电视或物联网设备中的 Swarm。让解决方案的渺小不会愚弄您影响有多大；

而这只是少数。你可以在官网查看更多关于 Swarm 的项目。

7. BZZ 代币的用例是什么？

BZZ 代币是一种兼容 ERC20 的实用代币，供节点运营商和最终用户使用。

Swarm 节点参与 Swarm Accounting Protocol 来跟踪相对带宽贡献，不平等的消费最终在 BZZ 中得到解决。

内容上传者必须购买一批邮票，以便他们在上传内容生成的每个块上贴上一张邮票，以表明他们希望提前保留该块。存储节点使用这个信号来保存价值最高的块，从而优化 DISC 的效用。

8. 在接下来的 12-24 个月内，您最感兴趣的加密趋势是什么？

Swarm 热切地遵循任何使扩展 Swarm 更容易的 L2 解决方案。有许多有趣的解决方案。另一方面，更严格的隐私法（如欧洲的 GDPR）最终将导致 Swarm 等解决方案。用户应始终控制自己的数据。 Swarm 相信它可以提供这种服务。
