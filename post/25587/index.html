我担心没人会在乎 Rollups。

过去6个月里，以太坊最大的故事是**交易需求的爆炸式增长** 。交易费已经超过历史最高水平，许多普通用户因价格过高而无法使用以太坊。

上图：以太坊网络交易费的美元价格。图源：Coinmetrics

但别担心，一位「救世主 」已经出现了。

显然，我说的是 Rollups 。Vitalik 预言 Rollups 是以太坊扩容的未来。目前，头部的 Rollups 方案已经获得了超过 1 亿美元的风险投资。每个主要的 DeFi 协议都选择了自己的立场，致力于将其中一个 Rollup 方案作为自己未来的家园。

如果你不知道 Rollups 是什么，这里有一个简短总结：Rollups 本质是微型区块链，继承了它们构建于其上的区块链 (比如以太坊) 的安全属性。即使某个 Rollup 的验证者/运营者不可信，他们也无法盗取你的资金 (当然前提是该 Rollup 以正确的方式被实现)。有两种基本类型的 Rollups：Optimistic Rollups 和**零知识 (ZK) Rollups** 。Optimistic Rollups 通过**欺诈证明** (froud proofs) 加以保护，这意味着任何人都可以自由地证明计算是否正确；而 ZK Rollups 通过**密码学** 保护，也即以数学的方式证明计算是正确的。对 Rollups 的更全面论述超出了本文的范围，如果你想更深入地了解，请查看 Vitalik 的这篇文章：

https://vitalik.ca/general/2021/01/05/rollup.html

Rollups 的美妙之处在于，它们是完全**无须信任的** 。如果你信任以太坊，那么你也应该信任 Rollups。这是一个无懈可击的扩容方案，并因此受到了以太坊领导层的欢迎。

现在让我们看看以太坊最近 6 个月的第二件大事：币安智能链 (BSC) 和 Polygon (前身为 Matic)。这二者并非 Rollups，它们更像是 (以太坊的) 侧链 ，因为它们是完全独立的基于 PoS 机制的以太坊克隆体 。它们基本上只是使用了现成的 Geth，去掉了其中的共识代码，增加了 Gas 上限，并使用多签 (multisig) 来桥接回以太坊。

瞧，它们实现扩容了。

上图：2021年1月以来，Polygon 的交易笔数 (绿线 )、币安智能链的交易笔数 (黑线 ) 和以太坊网络的交易笔数 (蓝线 _) 对比图。图源：_Our Network

Polygon 和币安智能链 (BSC) 都获得了巨大的成功，它们现在的每日_交易笔数_都超过了以太坊本身 。此外，诸如 Avalanche 和 NEAR 等其他区块链也在加紧开发自己的 EVM (以太坊虚拟机) 兼容系统，这些系统也可以连接到以太坊。

现在的加密世界基本上是这样的：

这几乎就像分片，但作为「信标链」的是以太坊1.0链。跨链转账和消息传递是由**多签** 、桥梁 以及一些**可信参与方** 来辅助实现的。我把这种架构称为「可怜之人的分片」。这就是如今 DeFi 实际的扩容方式。

但是！这种可悲的状况很快就要结束了。因为 Rollups 很快就会准备好了。

我对 Rollups 很是兴奋。是真的很兴奋。Rollups 具有颠覆性和优雅性，而且设计得非常出色。

但我担心用户们不会在乎 Rollups 。以下是我为何会担心的原因。

L2 的救世主情结

我给你讲个古老的故事。

曾经有一个不能扩容的区块链网络。但是后来一些非常聪明的人发明了**无须信任的 L2 技术** ，可以实现该区块链的扩容。用户为此兴奋不已。然后，经过多年的努力，这些聪明的人最终构建了 L2 (第二层)。而当用户最终能接触到 L2 时，却没人在意它，因为他们已经在使用其他更简单但不那么可靠的扩容方案了 。

这个故事听起来耳熟吗？

还记得闪电网络吗？人们只是使用 WBTC。还记得 Plasma 吗？人们只是使用 xDai。现在，人们只是正在使用 Polygon 和 BSC，而不是等待 Rollups。

伟大的叙事「解决」每个人的问题。Rollup 的扩容叙事对每个人都有意义。去中心化极大主义者们讲述了一个关于不需要做出权衡就可以扩容以太坊的宏大故事。交易员们在价格图表上划出了有关 Rollups 将如何推动 ETH 涨到 1 万美元的线条。而普通用户在使用 Polygon 疯狂地耕作 AAVE-MATIC 收益和参与数字赛马博彩的同时，也好奇地对 Rollups 的叙事点了点头。

我并不是在搞笑！Rollups 获得了以太坊知识分子们的压倒性支持，他们乐于对去中心化不足的事物加以批判。我能理解，因为我也是他们中的一员 。

但我们很难忽视的是，大多数人已经采用了 Polygon 和 BSC 。没有一个风投 (VC) 思想领袖预见到了这一点。来自印度、印度尼西亚、泰国、菲律宾等发展中国家的大量用户正在拥抱这些平台，他们中的许多人以前似乎从未使用过以太坊 (费用太高了！)。还记得“bank the unbanked” (让没有银行账户的人享受银行服务) 这句话吧？嗯，诸如 Polgyon 和 BSC 这些平台实际上已经覆盖全球，吸引了用户真正关心的东西。

我经常说，目前驱动加密用户的动机有三点：

  1. 赚钱

  2. 好玩

  3. 意识形态

在这三者中，意识形态 (ideology)是最弱的 。我担心意识形态最终会成为支撑 Rollups 采用的主要驱动力。

L2s 面临的情况是：它们在理论上听起来比实际情况要好。

Rollups 甜蜜的痛苦

现在在 Polygon 上，一笔简单的 Uniswap 类型的交易的成本是 0.0001 美元。在币安智能链 (BSC) 上，成本是 0.20 美元。在以太坊上，成本约为 7 美元。而在 Optimistim Ethereum (基于 Optimistic Rollup 的 L2 方案) 上，成本约为 0.68 美元。

为何 Rollups 上的交易成本要比侧链更贵？ 这是因为，每个 Rollup 最终都需要将 calldata 发布到以太坊主链上；这就把 Rollups 的费用与以太坊的费用绑定在了一起。每个 Rollup 只能以一个常数因子扩展以太坊。因此，与许多 (Polygon或者BSC) 用户已经习惯的低费用相比，Rollups 的费用不会_那么_低廉。

而且没有一个 Rollup 是与 EVM 完全兼容的 ，每个 Rollup 的虚拟机与 EVM 之间都有细微的区别。比如 Arbitrum 使用 AVM 虚拟机，Optimistim Ethereum 使用 OVM 虚拟机，这些 Rollups 虚拟机都略微打破了一些合约和 EVM 兼容的工具。而对于 ZK-Rollups，则是另一个不同的世界：ZK-Rollups 将把 Solidity 编译成对等的零知识证明电路，并在 ZK 虚拟机中执行。

现在将 Rollups 与 Polygon 进行比较，在 Polygon 中你只需要**复制并粘贴你的合约** 便可。

然后考虑一下资金在 Rollups 的流入和流出。

对于 Optimistic Rollups 而言，当你想要将资金**提款** 至以太坊主链上时，会有一个大约一周时间的挑战期 ，在此期间你的提款会被冻结。因此为了实现“快速提款”，做市商们将随时准备帮助你实现快速提款，当然他们会收取一笔费用 。他们向你收取的费用取决于你的**提款量** 和**该资产的流动性** 。如果你需要提款 ETH，那么这将花费大约 0.2% 或更多的费用，但如果你尝试提款「犬类币」，那么可能花费更多，可能是 1% 甚至更高。如果某项资产没有足够的流动性，那么该资产可能无法快速地提款。

作为用户，当你在规划基于 Rollup 的 DeFi 投资组合时，你需要考虑所有这些因素。话虽如此，如果你使用一个传统的基于多签的「桥梁」来桥接到 Rollup 中，就可以避免这些提款问题。但如果你承担了基于多签的「桥梁」带来的_托管风险_，那么这种方式又与 Polygon 有何改进之处呢？

(请注意，ZK-Rollup 不会受到这个提款问题的影响，因为它们的提款实际上是即时的。)

我担心由于**所有这些费用** ，Rollups 将不能迎合用户需求频谱的任何一端。如果你是一个非常关心安全性的超级鲸鱼大户，那么支付以太坊主网的 (高昂) 费用是没什么问题的；而如果你是平民大众，那么你会接受 Polygon。除此之外，还剩下谁呢？

好吧，Rollups 如何能赢？

在这一切开始之前，我对 L2 的看法是：以太坊上的每个 DeFi 协议都将承诺使用 L2，有些协议会选择 Optimism，有些会选择 Starkware，获得了最多协议采用的  L2 方案将最终成为占主导地位的 Rollup。

但很明显，这并不是正确的思维模式。不可抵抗的是，DeFi 协议往往选择多个扩容方案 。Aave、Sushiswap 和 Curve 已经在 Polygon 上线，将其 TVL (总锁仓价值) 提高至 80 亿美元以上。但 Sushiswap 已经在 Fantom、Polygon、xDai、BSC 和 Moonbeam 这 5 条链上部署；Curve 也已经在 Polygon、Fantom、xDai 等链上部署。很长一段时间以来，Uniswap 只与 Optimism 结盟，但随着 Arbitrum 的推出，Uniswap 改变了调子，也将采用了这种在多条链上部署的方式。

币安智能链 (BSC) 告诉我们的是：如果你不在 BSC 上部署，我们将会推出一个你的分叉版本，然后夺走你本来可以获得的收入 。展望未来，我预计每个主要的 DeFi 协议都将先发制人地在每个主要的链上部署。

那么，是由 DeFi 协议来决定用户的去向？还是由用户来决定 DeFi 协议的去向？

目前，Polygon 和 BSC 的教训似乎是后者：协议跟随用户 ，且这些协议因此得到了丰厚的回报。

作为一名投资者，我可以告诉你，目前的共识是 Rollups 会赢。Vitalik 喜欢 Rollups。每个人都喜欢 Rollups。

但是我担心 。我担心没人会在乎 Rollups。人们已经拥有了 Rollups 最初承诺的东西：快速、廉价、与 EVM 兼容的区块链，这些链可以流畅地与以太坊生态系统集成。

那么，从长远来看，Rollups 将如何能够取得胜利呢？

在我看来，有两种方式：一是这些非 Rollups 的侧链灾难性地失败了，加密行业从中吸取了类似 Mt.Gox 的教训。灾难性的失败并不仅仅意味着“节点不能同步”，而是“资金没了”或者“这条链完全停止运行了。”这是可能发生的，但可能性不大。

这就给我们留下了另一种方法：Rollups 必须要变得比其他方法好得多 。Rollups 去中心化的优势并不足够。对于这一点，我个人看到了一条前进的道路，那就是密码学和零知识证明的承诺 。

在过去的几年里，作为零知识证明基础的**密码学** 经历了类似摩尔定律的发展轨迹，而且没有放慢的迹象。以前人们认为在 ZK Rollups 中执行类似 EVM 的计算是不可行的，而现在 zkSync 和 Starkware 即将启动这种计算，通过递归的方式来构造 ZK-SNARKs 用于证明该计算。随着时间的推移，我预计我们将看到 Rollups 带来比今天更多的扩容性提升 ：大规模计算压缩、保护隐私的智能合约、可证明的抗 MEV (矿工可提取价值)，等等。

状态增长在 ZK-Rollups 中也不是很大的问题，因为无论状态有多大，用户总是可以通过简单地验证 SNARKs 的顺序来验证状态的正确性。

长期来看，零知识技术将只会越来越好。

但即使在短期内，我也为 Matter Labs 所做的 zkSync 2.0 及其 zkPorter 架构感到兴奋。zkPorter 是 Validium 和 ZK-Rollup 的混合体，允许用户在两者之间无缝迁移。由于 Validium 的数据保存在链下，因此其收费可与 Polygon 的费用相当，而对于想要更高安全性的用户来说，可以使用更昂贵的 ZK-Rollup 模式。zkPorter 在同一屋檐下集成了用户的所有选择，这些选择之间具有充分的互操作性。

zkPorter 架构。图源：Matter Labs

在我看来，这就是未来的发展方向。之前我们会说：「不，傻瓜，你做了错误的选择，我不在乎其费用是否更低」，但现在这么说显然已经不够了。我们不应该关闭 进一步创新的大门 。

至于 Rollups，我押注其会赢。但谁知道呢！我错的次数比对的次数更多，如果用户一开始就接受了 Rollups，那对他们和以太坊来说都是好事。
