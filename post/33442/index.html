大约一周前，我们发布了下面这条推文，并收到了大量回复:

文档和团队知识管理工具是 DAO 的一大痛点。目前还没有可行的基于 web3原生的解决方案，所以我们只能使用常见的 web2 技术工具：Notion、Google Docs 和 Roam。

在本文中，我们首先讨论了使用何种协作工具可以更好的完成DAO的工作，然后我们评估了市场上现有的工具，看它们是如何配置管理的，这其中包括我们自己的技术栈。最后介绍了我们 CabinDAO期待的理想工具，它是一个 Frankenstein 文档工具，而且集合了大多数 web2 工具中最适用于DAO的功能特性。

我们借鉴了在CabinDAO 编写文档和管理项目过程中获得的经验，而且请教了 web3 生态系统中正在构建 DAO 及协议的一些人，同时汲取了他们分享的经验和团队工作流程。

最后，我们并没有对写作和协作工具（例如 Google Docs）和知识管理工具（例如 Notion）进行明确划分。因为对于 DAO来说，非正式的笔记、文档和发布都属于团队沟通的范畴， 而我们需要的是一个可以实现从最初的会议记录，头脑风暴，到最终版本获批这一整个流程的工具。

那接下来我们深入聊聊吧。

DAO 知识管理工具的基本功能

多人访问权限管理

支持多人访问是 DAO 在知识管理工具中面临的最困难的问题，特别是其基于人数的定价及权限问题。

因为传统公司在员工和非员工之间有非常明确的界限。因此，企业软件在最初设计时就考虑到了这种结构， 从而使其定价基于企业员工的数量。

相比之下，DAO 的参与度是具有一定的梯度的。在任何时候，DAO都是一个由少数核心贡献者及参与者、社区成员、Discord 潜伏者及大众构成的同心圆。

所以这自然就涉及到了访问权限问题。传统的公司可能拥有超过300名员工，他们有 IT 部门来负责授予、撤销和管理每个用户的访问权限， 然而在大多数拥有同样规模的DAO里，只有一个或几个人员来负责管理所有成员的权限。

企业应用程序在管理多人访问权限方面做得很好，比如Google Docs 可以管理单个文档的权限，Notion 可以将不同页面设置为公开或私有。但是如果只是为了让每个成员（尤其是非贡献者）拥有查看，评论及共享文档的功能，而按照成员数量支付高额软件费用，这对于DAO 来说是没有太大意义的。

维基百科贡献模型是一个可以解决DAO的访问问题的理想方案。维基百科在贡献者和编辑者之间有一个幂律分布，作为一个开源项目，它不区分雇员和非雇员，但设置了各种不同的门槛机制以确保持续的高质量信息的输出。

分布式项目管理

传统公司的企业知识管理应用程序具有一个自上而下的结构。项目和任务是通过上层管理人员来分配到下属小组及成员。

相比之下，DAO 更像是一个相互交互的独立承包商网络，而不是一个由员工构成的公司。它的结构是自下而上的，贡献者需要写下自己的工作描述，并提议项目来获得社区的批准。

例如，Github 鼓励开源贡献，并通过内置的审查过程来确保贡献代码的质量。它允许开发人员针对自己希望添加的功能提交需求并可将其分配自己，总之，Github 鼓励并提倡分布式项目管理。

当潜在的贡献者提交提案时，一个理想的知识管理软件应该能够根据项目提案的规格和范围，匹配出合适的治理或批准流程，并估算出报价。

Web3 原生

包括 CabinDAO 在内的大多数 DAO 仍在通过整合 web2 的应用程序，来实现协作和信息的传播，但这种方法并非长久之计。

通过使用 web3 原生应用程序，DAO可以针对一个组进行文档的权限的管理，而不再是针对单个成员。Web3 不再需要不断地添加新用户和删除离开的用户来修改权限，而是以代码方式来分配权限。这意味着DAO 可以通过成员是否拥有代币或 NFT来判断其会员资格，从而更准确地管理访问权限。

DAOHaus 的 Spencer Graham（Twitter @spengrah）向我们描述了他心目中理想场景。

“当我使用我的钱包地址登录 web3 原生工具时，该工具可以让我查看我在 DAO 中的状态，然后它可以向我展示我可以访问的内容，同时隐藏掉无法访问的内容。这种程序化的功能与人工决定是否给予个人编辑或访问权限的工作方式相同。”

更高级的 web3 原生功能是能将项目与赏金绑定的能力，甚至可以在项目中嵌入支付系统，并基于各成员的贡献水平来支付赏金。

目前 DAO 使用的知识管理工具

知识管理是 CabinDAO 的一大痛点。我们目前同时使用 Roam、 Google Docs、 Notion 和 Discord，因为单独的每一个都无法完全满足我们的需求。

Notion

我们使用 Notion 来分享有关 DAO 的常识和我们的长期愿景，因为我们喜欢 Notion 的所见即所得（相比于 Markdown）以及它可以针对不同贡献者及页面管理权限。

我们承认Notion 至少解决了我们在文档和知识管理方面至少一半的问题，但是作为一个企业工具，Notion 的用户每月费用为 8 至20 美元。如果没有其他解决方案，那么为CabinDAO 的 Discord 的 300 多名成员支付使用费用将非常昂贵。

Roam

我们使用 Roam Research 进行项目管理和内部知识管理。

Roam 的多用户架构适用于我们。首先任何人都可以为我们的 Roam 服务器做贡献，同时不必担心会删除其他人写的内容。这样一来，在访问权限方面，它比 Google 文档或者 Notion 更容易，而且它的双向链接结构也非常适合自下而上的项目管理方式。

然而，根据我们的经验，Roam这种基于bullet 写作的界面并不适合长篇幅的写作，所以对于不熟悉其块状结构或 Markdown 的人来说，熟练掌握 Roam 会比较难。

还有一些人在使用类似一个笔记应用程序叫做 Obsidian，与 Roam 不同的是，它不支持多用户功能。

Google Docs

我们使用 Google Docs 进行协作写作。它的评论功能及多人操作的功能很强大。但是Docs 没有链接功能。即使经过了这么多年，其写作体验仍然类似于与略显过时的 Microsoft Word。

（我们当然知道，如果它没有损坏的话，就没必要去修复它。但是，与以往不同的是，我们正处于 web3 时代）。

HackMD

我们从未使用过 HackMD，但它是一个在推特上被推荐了多次的工具，所以我们采访了几个使用它的 web3 团队。

Jet Protocol 和 DAOHaus 都将 HackMD 作为他们团队的主要协作工具。这两个组织使用它用于内部协作和速记。

针对外部协作，Jet 使用的是 Discourse，而DAOHaus 使用 Docusaurus 发布公开的及更正式的文档。

Hackmd 的很酷的一个功能是它融合了 Github 。这样以来，编辑过的文档可以先经过合并请求的审查流程，然后再被推送上线。

Clarity

Clarity 是此列表中唯一以 DAO 为目标受众的应用程序。

由于我们的团队最近才开始测试 Clarity，因此我们通过外界的反馈对其进行了深入了解。

作为一个不是 DAO 但功能类似 DAO 的写作社区，Foster 放弃了Notion 并转向了 Clarity 来进行团队的知识和项目管理。因为他们意识到 Notion 越来越臃肿，尤其是在自定义任务方面占用了太多时间。

Foster 团队喜欢 Clarity 处理任务和知识的方式，其联合创始人 Stew Fortier 说：“Clarity 的维护成本很低，你可以很轻松地启动多个任务并直接在文档中分配它们。”

通过对 Clarity 的初步测试，我们发现双向链接会是一个挑战。其当前的定价模式仍然基于用户数量，而且数据存储是在传统的 web2 架构上，但 Clarity 团队向我们透漏，他们正在计划调整模型以更好地为 DAO 服务。

据创始人 Richie Bonilla 的所说，Clarity 首先想解决人员协调的问题，他们正在积极地与用户合作，以寻找最佳的方式将应用实现去中心化。

DAOs 的文档和团队知识管理: 谁是赢家？

DAO 需要一种以 web3 原生方式创建、编辑和传播知识的方案。当前可用的工具主要还是 web2工具。它们并不是基于 DAO 自下而上，多人协作的需要而构建的。

在 CabinDAO，我们梦想可以有一种知识管理工具，它结合了市场上最好的 web2 协作工具中最接近 web3 的功能特性。我们期望它具有 …

  • Notion 的所见即所得界面

  • 维基百科的可渗透社区协议

  • Roam 的双向链接

  • Reddit 的贡献模型

  • Github 的变更管理

  • Web3 原生代币门槛、身份和支付工具

那么你的 web3 组织的文档和知识管理是什么样子的呢？我们非常期待收到你的来信。
