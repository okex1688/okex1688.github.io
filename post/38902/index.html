在用户零转换成本和所有代码是可分叉的时候该如何竞争？

竞争是获取和管理稀缺资源以创造优势的艺术。技术的变化改变了资源的稀缺性，从而改变了竞争的基础。“新竞争者”看起来与“老竞争者”不同，很大程度上是因为现在稀缺资源与以前稀缺资源不一样了。

开放性为竞争增加了一个有趣的维度。开放组织可以发现潜在的能力，使之变得丰富，转换了其稀缺性。同样地，开放以前的稀缺资源，让它成为商品，并以此改变竞争的基础。

随着 web3的兴起，我们正在进入一个竞争优势的新时代。稀缺性再次发生变化，竞争的基础也是如此。这篇文章(我们即将出现的 web3攻略的部分)围绕这种新的稀缺性以及公司将如何在这个新时代竞争，提出了一些初步的想法。

竞争伴随着稀缺性

为了理解新的稀缺性所在，我们需要更好地理解企业如何获取和管理稀缺资源。

基于资源的竞争

在工业时代，企业竞争的基础是获得稀缺资源。稀缺资源包括获取特殊供应来源(如油田和矿山)或独特的知识产权(如可口可乐的配方)。

对于数字时代的公司来说，这些大多都会转化为竞争优势。谷歌的搜索算法作为商业机密继续受到保护，这也是提供他们竞争优势的关键资源。

当 Airbnb 成功抢占了 Craigslist，并将 Craigslist 的用户基础拉到自己的时候，Craigslist 失去了其稀缺资源——由卖家创建的挂牌信息——转移到 Airbnb。作为回应，Craigslist 开始向所有试图效仿 Airbnb 整合的初创公司发放了停止并终止律师信。

以资源为基础的竞争是以获取和管理稀缺资源为中心的。获取稀缺资源可能涉及获取新的供应来源，或者获得能够创造独特知识产权的最佳创新者。因此，管理这些稀缺资源需要建立必要的护城河(兼具结构性和法律性的) ，以防止其他人获得这些资源，并利用这些稀缺资源获得竞争优势。

Web2: 数据作为新的稀缺性

随着社交网络的兴起，企业发现了一个新的竞争基础: 用户数据。稀缺性从仅仅获取和管理内部资源转向获取和管理用户数据。

数据收集—— web2的拥护者在努力获取用户数据——因此公司开发了让用户提交数据的新方法(还记得 LinkedIn 的个人资料完成进度条吗?)而成为争议点，同时也发展了交换和处理数据的完整的后端行业。

将耕种类比于数据收集是极其不舒服的，当用户“播种”他们的注意力时，数据可能就被收集了。因此，在数据上的竞争实际是在注意力上的竞争。公司投入在行为设计上，以“吸引”用户，并让他们循环地增加数据的消费。黑暗的用户体验在偷偷地提取数据。公司合并随之而来。当工业时代的公司收购了拥有相似或互补的稀缺资源的竞争对手时，数字时代的公司则忙于获取相似或互补的用户关注和数据资源。Facebook 收购了 Instagram 和 WhatsApp，谷歌收购了 Waze，所有这一切都是为了囤积更多稀缺资源。

为了管理这种稀缺资源，企业起初大量投资于大数据处理技术(2008-2013年) ，然后将重点转向利用人工智能和机器学习创造价值(2010年以后)。

由松散的数据保护监管体制授权的集中式平台，在这种新的稀缺性上展开了竞争。但这种情况正在日益改变。GDPR 和其他数据保护制度的兴起使数据提取变得越来越困难。同样，web3的崛起预示着一个新时代的到来，在这个时代，生态系统可以在没有中央集权和抽取的情况下组织起来。

稀缺性又开始改变了！

Web3: 追踪新的稀缺性

Web3的兴起改变了许多关于稀缺性的假设。数据是可携带的，不再是竞争的基础。用户可以很容易地切换服务，用户的注意力和数据可能不再像在 web2世界那样被访问、收集和管理。

Web3 的开放生态系统也侵蚀了 web2平台从中受益的垂直整合优势。例如，随着用户界面从底层平台分离出来，对用户界面(以及用户注意力)的竞争将在 web3中加剧。Web2市场平台垂直整合到搜索界面，保证了两者的垄断。管理市场交易的 web3协议可能支持许多搜索代理，正如我前面解释的那样。

例如，像 Ebay 这样的市场将卖家入驻、卖家分析、买家入驻、买家决策支持、搜索功    能和交易基础设施捆绑在一起。所有这些组件都将在 web3世界中进行分拆。

因此，我们从中央做市商体系结构转向分散的基于代理的体系结构，所有这些都通过一个   共享的通用协议来协调。这就好比好莱坞式的人才市场，减去了基于关系的把关。

那么，新的稀缺性会出现在哪里呢？

开发者的参与将成为开放生态系统时代的新稀缺。

正如我在《 拆分再拆分 》中解释的那样，开发者参与是 web3生态系统的核心价值驱动力，因为市场基础设施不再是由中央平台业务构建的，而是由生态系统开发者构建与核心协议互补的模块。开发者的参与是 web3生态系统基础设施层的主要价值来源。对于开发人员来说，这需要开发人员投入时间、资源、能力等等。

开发人员的稀缺还有另一个原因。开发人员的参与和资源承诺涉及到高昂的多宿主成本(开发人员为同时构建多个平台而付出的成本)。通过为特定的生态系统建立和投入资源，开发者含蓄地选择不将他们有限的时间和资源投入其他(竞争和非竞争的)生态系统.

并不是所有的开发者生态系统都是一样的

管理开发者生态系统也是 web2平台的一个重要考虑因素。那么 Web3 究竟是如何改变的呢？

为了了解 web3开发者生态系统的不同之处，我们需要重新回顾一下我在早期时事通讯中提出的构建模块的论点。

在任何开发人员生态系统中，开发人员执行两个关键类别的活动: 他们可能有助于解决方案开发(价值创造) ，或者他们可能创建驱动解决方案使用的不同集成(价值分配)。

开源项目主要依赖开发人员生态系统进行解决方案开发。然后通过传统的公司所有的渠道(例如 Redhat 与 Linux 的关系)对软件进行打包、交付和支持。这在左上象限中显示。

许多 web2平台与开发者生态系统有关。然而，价值主张的核心平台是由内部开发者构建和交付的。大多数 web2平台依赖于开发者生态系统，或者创建核心平台技术的补充，或者建立集成和推动平台的使用。后者在 Stripe 和 Twilio 这样的公司中尤其明显，它们提供基于 api 的功能(作为服务)。这些公司在开发人员生态系统上投入了大量资金，但主要是为了推动解决方案的使用(价值分配)。这在右下象限中显示。

Web3生态系统的独特之处在于（就此而言，任何遵循 Building Blocks Thesis 的生态系统），这些生态系统通过促进解决方案的使用，吸引开发者从事价值创造、解决方案开发和价值分配。这在右上象限中显示。

例如，Lens Protocol 的开发人员参与活动跨越了解决方案开发和用途。类似地，Boson Protocol 的臭虫赏金计划开发平台利用开发人员参与解决方案和安全性的开发，而 Boson 的 走向 v2 路线图 扩展了开发人员参与解决方案开发和解决方案使用的范围。

Web3生态系统的价值和防御

如上所述，web3生态系统是独一无二的，整个价值链的价值创造是由开发者生态系统驱动的。

Web3项目需要建立针对解决方案开发的开发人员参与程序，以便为其代币标记积累价值。随着解决方案变得更加全面，代币的价值也会增加。

同样重要的是，web3项目需要通过集成鼓励解决方案的使用建立开发人员参与计划。当解决方案被集成到更多的项目中时，解决方案的可防御性就会增加。新的竞争对手分享代码库可能会复制代码库，但是很难复制通过开发者参与项目建立的无数集成，这些项目推动了解决方案的使用。

这就是为什么 web3的竞争优势将围绕吸引开发人员的能力而展开。

繁荣的开发者生态系统发挥着两种变革性的功能。

  1. 它们增加了代币价值(增加了项目的吸引力) ，并在一个自我强化的循环中吸引了更多的开发人员。

  2. 它们增加了可防御性(确保更难取代相关的项目)。
