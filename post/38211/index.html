1. Vitalik 新论文摘要

《去中心化社会：找寻 Web3 的灵魂》是 Vitalik 等人联合发表的最新论文，本文讲述了如何通过 “灵魂绑定” Token（本质上来说就是 DID）来建立一个更加多元、丰富的 DeSoc（去中心化社会）。论文内容非常长，本节对这篇论文的重要观点进行转述与总结，便于读者快速解去中心化社会相关知识。

1.1 Web3 成立之初的局限性

在不到十年的时间里，Web3 创造了一个庞大的金融帝国，这一成就令世界瞩目，连 FaceBook 都以 all in 的形式参与到这场金融盛宴之中。加密和经济领域的基础元素，为金融交易带来了一个丰富、开放且多元的生态系统。但是这个过程中出现了一个矛盾，金融交易的经济价值是由人们现实生活中的社会关系等决定的，也就是说它的内部运转逻辑并不完善甚至可以说并不完整。正是由于 Web3 缺乏代表社会身份的基础元素，它从根本上依赖于它所要超越的中心化的 Web2 结构，这具有非常大的局限性。

这种依赖性在具体应用方面也有很明显的体现：

  • 虽然 NFT 通过 Opensea 等去中心化平台进行交易，但是许多 NFT 艺术家会通过 Twitter 等中心化平台进行宣传，以向人们证明其真实性并达到推广的目的

  • 各种去中心化项目经常依赖 Web2 基础设施来实施投票

  • 许多 Web3 参与者依赖 Coinbase 或 Binance 等中心化实体管理的托管钱包

此外由于缺乏原生的 Web3 身份，今天的 DeFi 生态系统无法支持实体经济中普遍存在的活动，如等额抵押贷款甚至超低额抵押贷款（目前加密贷款平台一般只支持超额抵押）或者签订公寓租赁等合同。在此我们试图通过“灵魂绑定”的 Token（“soulbound” tokens，SBTs，也就是 DID 去中心化身份）来打破这些局限。除此以外通过 Web3 社会身份，还可以就财富集中和金融攻击的治理脆弱性等更广泛的长期问题上取得巨大进展，同时刺激 web3 的政治、经济和社会应用的大幅发展。我们把这些用例和它们所促成的更丰富的多元生态系统称为“去中心化社会”（DeSoc）。本文第二部分会就 web2 的社交与 Desoc 进行对比进一步展示出 Desoc 的一些特点与优势。

1.2 DeSoc 的要素与畅想

首先我们应先了解一下 DeSoc 的基本元素，它（“灵魂绑定” Token, SBTs） 围绕着人们持有的不可转让的账户（或钱包），代表着承诺、证书和关系，这种 Token 就像一份扩展的简历。可以简单地把这种东西理解为我们的身份证，有了身份证我们就可以缴纳社保、绑定银行卡等。没有身份证或者说法律认可的身份这些都很难实现，而目前的 web3 就处在这种没有身份证明的困局当中。在这里把起身份证明的东西被称为“灵魂”， 将账户持有的 Token 称为“灵魂 Token”（SBTs）。

想象一下这样一个世界，大多数参与者都有“灵魂 Token”，它对应一系列的关系、会员资格和证书。因此人们不用公开自己的真实身份，就可以让他人看到自己的教育证书，就业历史，或自己的著作或艺术作品的一串哈希值，这种 Token 对相关信息的公开类似于人们在简历中分享关于自己的信息。更重要的是，通过这种身份人们可以互相识别，甚至可以实现目前无法达到的等额抵押、合同签订等。人们甚至可以无纸化，因为数据可以记录一切，并且因为它不可篡改的特点，又让一切变得更加可信。在这些技术的支持下 web3 一定会越来越丰富且美好。

1.3 相关实例

艺术与“灵魂”

艺术家可以通过“灵魂”将作品与自己连接。 当发行可交易的 NFT 时，艺术家可以从他们的“灵魂”中发行 NFT。艺术家的“灵魂”携带的 SBTs 越多，买家就越容易识别出“灵魂”是属于该艺术家的，从而也就能确定 NFT 的真实性。艺术家可以更进一步，在他们的“灵魂”中发布一个链接的 SBTs，证明某个 NFT 是该“系列藏品”的成员。因此，“灵魂”将创造一种可靠的、链上的交易和识别方式。 很明显，它的现实应用还有很多，NFT 艺术家的例子只是其中最具有代表性的。实际上，除了 web3 领域外，它也能有解决一些现实问题，比如知识产权、司法文件真伪判断等。毕竟随着深度造假技术的进步，人类和算法的直接检查将越来越无法检测真实性。虽然区块链的加入使我们能够追踪特定作品的制作时间，但 SBTs 将使我们能够追踪相关内容的社会出处，为我们提供丰富的社会背景，让我们了解发布作品的人的“灵魂”——他们的会员资格、关系、证书的组合。通过这种方式“高仿”可以很容易地被识别出来。

“灵魂”空投

在具体说明“灵魂”空投之前我们需要明白空投是什么，以及空投的意义到底是什么？区块链领域的空投是一种将数字货币通过区块链技术大规模分发到一些已有的数字货币钱包上的行为，比如以太币、比特币、EOS.IO 钱包。简单来说就是向广大网友“发钱”的过程。

当面对空投人们会有四种行为：（1）上网搜索相关内容（2）和朋友聊相关内容，使朋友知晓有关信息（3）之前或者之后疯狂注册账号，以获得空投（4）直接无视

空投本身是一种非常流行的营销策略，并不是所谓的“慈善”，因此在上述几种情况中，（3）和（4）属于无效空投，并不能带来实际的收益或者说宣传的效果。而其中的第三条属于危害性最大的一条，也就是常说的女巫攻击。为了避免第三条项目方往往设立许多空投限制，但是很明显并不能完全避免。通过 DID 可以有效解决上述问题，因为项目方可以依据 DID 进行空投，可以有效提高投放的精准性和准确性。

很显然，上文那两个所谓的相关实例都还是比较理想化的概念。本文将于第三部分举出一些现有的比较具有代表性的相关去中心化项目，各位读者可以通过这些例子对相关内容予以把握。同时会在第四部分就上文提到的 DID 的使用、价值和发展前景进行详细的说明。

1.4 结论

通过这篇文章大家可以预见到 DeSoc 的美好未来，但同时也需要意识到 DID 只是这一远大历程中的一小步。此外，上面描述的场景只是 DeSoc 对现有项目与应用的强化作用，但是并没有涉及对比如物理、政治、社会以及它们与现实世界的复杂关系的描述，而它们实际上占据了我们现实生活的很大比例。所有这些都表明，上面描绘的远大理想，可能只是 DeSoc 最终形态的“雏形”。论文中就 SBTs 的威胁与挑战也进行了比较细致的描述，在本文的第五部分将结合现实情况进行细致描写这里就不再细说。

2.  新型社交为何悄然而至

2.1 Web2.0 传统社交弊病

Web2.0 本质是平台经济，是以一小撮为代表的垄断性寡头构架，互联网在该阶段变成全民级别应用，聚集了社交、搜索、电商等超级生态应用开发，且每个行业领域龙头公司几乎垄断行业资源和发展上限，这使得 Web2 公司在拥有最终解释权的情况下去剥削和占有用户资源信息时游刃有余，例如微信和QQ在中国产生的影响力和不可置换性，使得用户无法维护本身利益，用户在 Web2.0 中更像是平台的附庸而不是服务接收者。而 Web3.0 是属于 Token 经济的后端程序革命，其本意是在为非平台内部的生态参与者获取更多的话语权和创造权，并拥有产品本身的价值支配权利，以及隐私性和安全性。一定程度的避免了不平衡的垄断机制。

2.2 SocialFi 是 Web3.0 的必要组件

SocialFi 的出现是对抗传统社交平台弊端的必要产物，Web2.0 时代的社交平台被各大运营巨头控制，从用户数据中获取商业价值，忽略用户的感受。SocialFi 基于区块链技术，用户身份信息的匿名性有助于保护隐私和安全，更有利于解决用户数据的确认问题以及权利下放。

相信随着用户对于隐私以及自由的追求，越来越多的人会投身于 Web3.0 的世界，在用户渴望加密技术带给的数据主权时也渴望获得自己在这个世界独特的身份和地位。在 Web2.0-3.0 过渡的过程中，互联网用户需要一些中间件来从简单迁移到复杂，从而从原始空间迁移到元空间。元空间的实现需要从现有的互联网世界出发，利用一系列的 Web3.0 组件，分批分阶段的迎接新的用户。在尝试了 DeFi 的去中心化资产流、NFT 的数字工作价值、发现 GameFi 的P2E之后，去中心化的意见发布和个人展示已经下一个需要探索和开发的 Web3.0 组件，在互联网发展历史中可以发现社交化可以说是网民的刚性需求。

3. 理念的实际运用

3.1 ENS

ENS”，全称 Ethereum Name Service， 是一个基于以太坊区块链的分布式，开放，可扩展的命名系统。简单来说，通过它可以把原本难以记忆的以太坊地址变成自定义的短地址。通过这种方式，人们可以轻易记下有关内容，既可以减少工作的繁琐程度，也可以减少不必要的风险。同时让用户可以通过以太坊账户和 ENS 配置文件控制自己的数字身份，不再依赖于传统的“中介”。

技术

这往往是在注册中心和解析器实现的：

ENS 注册中心由一个智能合约组成，该合约维护所有域和子域的列表，并存储所有域和子域的三个关键信息：

  • 域的所有者

  • 域的解析器

  • 域下所有记录的缓存时间

ENS 注册中心作用很简单，只是为了从名称映射到负责它的解析器。解析器负责将名称转换为地址。任何有相关功能的合约都可以作为 ENS 中的解析器。

在 ENS 中解析名称有两个步骤：首先，询问注册中心是哪个解析器负责该名称，其次，请该解析器回答查询。

比如，我们试图找到 “foo. eth” 指向的以太坊地址。首先，我们询问注册中心哪个解析器负责 “foo. eth”。然后，我们向解析器查询 “foo. eth" 的地址。

3.2 Bright ID

Bright ID 是 Web3.0 用户的身份系统，基于以太坊开发，它是完全独立于传统和互联网身份认证的系统。一般用于DAPP用户验证；活动验证；信誉构建等。

具有以下特点：

无门槛：任何人都有权利拥有 Bright ID。

隐私性：Bright ID 不会向用户索取或捕获任何个人隐私信息。

开源：公共产品，完全免费且开源。

运作和验证方式

Bright ID 需要经过人脸和多人认证的方式来通过审核。认证结合链下会议视频等非隐私信息渠道的方式，采用视频会议的形式，由以上六位验证官负责主持验证会议并进行验证工作。验证会议的具体时间日历可在 Bright ID 官网查看。

3.3 MetaMask

MetaMask，又或是人们口中常说的“小狐狸”钱包，日益成为加密圈内不可或缺的工具，你的钱包地址在一定程度上，可以代表你个人在链上的身份。

MetaMask 是一个通过 web2 互联网浏览器就能访问，且以低门槛与以太坊网络进行交互的工具钱包。它是 Consensys 目前面对终端用户的唯一产品，是以太坊最早的钱包之一，也是目前用户量最大的钱包。始于 2015 年，长期以网页端、免费、“半官方”的形态为大众所熟知，属于非托管型钱包。随着 2020 年 Defi 应用热潮与 2021 年 NFT 的火爆市场效应，MetaMask 的用户量极速增长。2020 年新增手机客户端和交易功能（名为 Swap）。2021 年 5 月推出 MetaMask 机构版，让机构用户能将 MetaMask 与机构级托管结合、方便工作流管理、易于汇报等。

但起初，MetaMask 其实并不是作为钱包插件之类的消费者项目，而是希望成为一种开发工具。Metamask 母公司，是以太坊联合创始人 Joseph Lubin 所创办的区块链巨头公司之一的 ConsenSys。ConsenSys 是以太坊重点项目的企业孵化器，旗下的 MetaMask 是 ConsenSys 布局的六大业务板块基础设施板块的重点项目。

技术

现 MetaMask 已兼容绝大部分的区块链网络 Dapp（包括 Ethereum、Binance、Polygon），可集成托管解决方案来管理秘钥存储和多重签名批准。团队也在逐步优化产品本身，简化交易流程、减少交易费用、签字程序等。并逐步提升其合规能力，让钱包内可以进行高风险评估，安全进行去中心化和匿名交易等。对于大量的用户选择 MetaMask 作为迈入 Web3 的第一步，团队除了打磨好产品，提升钱包本身的安全性能，用户友好度之外。面对发展迅猛的加密货币行业，使用需求量的暴增与跨网络的需求都是不可忽视的问题。

小结

如今，MetaMask 的使用已相当普及，这似乎已成为了进入 Web3 的第一步——生成你的去中心化钱包地址。而钱包也逐渐成为区块链世界中的一种身份象征，作为用户使用频率最高的小狐狸钱包，swap 与 NFT 购买，是其最常用的两个功能。也因为以太坊生态中任何一个应用，都会支持 MetaMask，两者的发展相辅相成，逐渐将钱包地址与 Web3 世界里去中心化个人身份的划上等号。 未来，伴随着多链时代的到来，通用钱包的概念或将被逐渐被弱化，取而代之的可能是针对性更强的点对点产品服务，如专门的 NFT 钱包，打金钱包等。随着时间的推移，MetaMask 自身以及新生代的钱包的发展都同样值得期待。

3.4 CyberConnect

Cyberconnect 旨在解决社交图表（Social Graph）再分配问题，这个问题其实很好去解释，就是说 Web2.0 中各种平台的信息无法互通，造成重复资源浪费且难以找到各种志同道合的用户，社交图谱相对单一。

而 Cyberconnect 所表明的内涵或者本质也很简单（用户之间的关系不仅仅是双向或者三向的，更是各种交叉和重叠的混合网络关系）例如 A 本身，与 A 关注的用户，和关注 A 的用户进行不同连线组合，形成一张独特的“图谱”，证明该用户的社交关系网，而从这些与用户 A 有联系的人又重新编织，又将形成新的图谱。这样网络世界中拥有相同的兴趣爱好的用户，同游戏玩家，商品贸易中的的买家卖家等通过社交图谱逐渐变得清晰。如此一来，CyberConnect 便打破了不同平台之间的壁垒。

技术

关于数据和存储：CyberConnect 社交图数据存储时使用的 IPFS 和无许可协议 Ceramic 构建，联通 Ceramic 区块链锚定和自定义数据固定服务，可以保证 CyberConnect 链上数据长期存储。（使用 Ceramic DID 方法）关于用户数据在不同的平台如何集成，则需要采用 Ceramic 身份协议 IDX，它为开发人员提供一个统一的界面，在 CyberConnect 内部查询这些数据分析，反过来 CyberConnect 也会给与 DID 链上系统提供更多的身份数据。

与其它项目集成：同时 CyberConnect 还会与其它 Web3 项目集成获取信息，例如 NFT 平台 Foundation 与 Rarible 等，其实最重要的一点是让玩家通过一个统一的身份来参与不同种类的生态应用，同时也能与 Web2 链接使链上和链下数据结合起来成为更强大的去中心化数据提供商。

3.5 Mirror

提到去中心化的写作平台，大部分区块链用户都会脱口而出“Mirror”，其平台在成立不到一年就被联合广场风险投资公司估值 1 亿美元，获 1 千万美元投资。作为去中心化的写作平台赛道代表——Mirror 是依靠哪些去中心化特性和功能”荣登“宝座呢？

功能简介

Mirror 跟中心化的写作平台最大的不同是，他具备加密区块链产品的特性和功能。参与用户无需通过邮箱账号，手机号码，社交帐户等中心化的身份验证手段来注册账号，只需要绑定自己的 0 x 开头的钱包地址即可生成独一无二的“作者专栏”，类似于上文所提及的 ENS（以太坊域名）。对于部分创作者来说，跟外界的接触仅仅由一串小小的字符连接，完美的融合了去中心化身份的理念。

除此之外，创作者可以在 Mirror 平台上发布文章/创作图片等作品，每一件公开发布的作品不是保存在中心化服务器上，而是透过区块链存储专案 Arweave 进行链上的数据保存，经用户进行加密签名后生成一段哈希值记录，将作品永久的保存在区块链上。Mirror 上更是支持创作者将自己的作品铸造成 NFT，让原先没有边际成本、可被无限复制的内容作品具有稀缺性，从而被赋予价值。不仅无法被他人复制或修改，而且可以在各大 NFT 市场上进行售卖。

其次，Mirror 也提供读者募资功能，读者喜欢的文章可以直接使用加密货币”打赏“创作者；共同的创作者可以按照设定好的盈利比例来获取各自部分的收益；以及其平台上的作品拍卖功能等等。

技术

Mirror 在保存用户文章时使用的两类 Web3 组件，一个是用于对文章进行永久存储的 Arweave，另一个是用于对文章进行确权与交易的 NFT 铸造平台以太坊主网。

  1. Mirror 的存储组件-Arweave，是一条针对存储进行了底层性能优化的公链，因此用户在 Arweave 链上存储数据的成本相较其他智能合约公链平台有极大的降低。同时，链上存储的特征保证了在 Arweave 之上存储数据的永久性，因此，基于 Arweave 的存储模式也常被归类为永久存储。创作者在保存文章修改的时候，Mirror 会在后台自动对新的文章进行二次上链存储。

  2. Mirror 的确权组件，在存储的同时，通常会需要另一组 Web3 组件，以便对存储的内容进行确权，也就是将内容铸造为 NFT 格式的资产。但此步骤是在以太坊网络上以 NFT 的形式进行确权，成本由创作者自行承担支付。

3.6 Project Galaxy

Project Galaxy 的愿景是想要打造一个最大的 Web3 数据凭证（credential data）网络，建造一个开放协作数据聚合基础设施，助力开发者和项目方建造更好的产品和社区。项目方在 Project Galaxy 平台上面为用户创建并奖励 NFT，用户完成项目方的任务，比如在链上进行特定的交互、参加会议、关注推特、加入 Discord 社区、提交问卷、参与测试等方式获得 NFT。

技术：

Project Galaxy 目前支持两种方式来提供数据凭证 GraphQL 和 REST API。Galaxy 上有两个角色，分别是数据贡献者和使用者。数据贡献者可以用子图查询或静态快照（subgraph queries or static snapshots）的方式贡献链上数据，同时在链下，Galaxy 也集成了信息来源，比如推特和 GitHub。

数据使用者使用 Application module, Credential Oracle Engine, and Credential API 三种方式使用数据。并使用 Galaxy 的代币 $GAL 支付，因此数据贡献者可以获得激励。

4. 新型社交的展望

最初的载体与未来场景？

绝大部分用户最熟悉 DID 理念实际运用应该是区块链钱包，例如 Metamask 是作为用户接入整个区块链的入口端，具有隐私性、安全性、不可追踪性。这是 DID 赛道最初应用成果，但是其缺点也很明确。钱包地址的注册缺少独特性，一个人可以注册大量毫无价值的钱包。其次大钱包只能显示代币难以展示 NFT 和参与证明等未来势必流行的资产，更不用说 SBT 所畅享的“无纸化”证明，这是钱包的局限性。第三钱包无法配置资产可编程属性，无法限制其他人查看钱包资产，从而导致黑客攻击情况发生。不过理念的实践是处于不断完善的过程。现在已经出现了可以展示 NFT 和参与证明（POA）的去中心化身份应用，还有隐私赛道对于钱包和其它 DID 应用进行代币隐私化、资产可视化等功能赋能等，预计不短时间内所有区块链赛道应用可以进行用户 DID 绑定，可记录用户等额抵押、合同签订、参与证明和哈希参数等不可篡改的数据，潜在提升单一用户 DID 价值

从结构和理论上来阐述，有社交的地方更加注重身份特征，所以从根本属性上来说 SocialFi 就是实现该理念的最初赛道，只有用户们尝试并参与进来，才能够从本质上理解 DID 在区块链各赛道中重要程度，在未来去中心化社交领域当中，寻找相同理念思想的用户从而构建和谐关系，通过新兴匹配技术进行身份特征快速转移和整合，快速的梳理出用户兴趣爱好与社交圈，可以大程度的减少网络中诈骗行为，提高社交沟通交流效率。

如何评估用户 DID 价值赋能？

未来 DID 账号的价值根据其内部的资产总数和分配比例，如加密货币、NFT、参与证明等进行价值评估，以及个人身份信息验证，与其他用户之间互动，评论留言等进行附加属性评估。最后就是项目之间的筛选机制和会员资格评判，如拥有某些 NFT 或者证明的用户获得相应空投奖励、参与测试的资格等，这些都进一步的丰富了 DID 整体价值属性。

对于目前市场来说，DID 基础设施赛道仍未出现热点项目能引领该赛道的热潮，例如 DeFi 的火热是 Uniswap 带领的，GameFi 的出圈前者与 Axie 有关，后者 StepN 再次引领，所以扩大潜在用户群体还需要落实到实际具体项目当中，单一可行的理念是无法吸引到足够的投资者的。

DID 赛道本身是没有主动生产价值的能力，创建了钱包地址不代表钱包有钱，但是从另一角度来说 DID 也是最能兼容区块链中所有领域的基础设施，其价值决定极大程度上基于与其它项目应用的兼容，毕竟 DID 属于后端程序，与个性化应用并不冲突。

趋势预测

在如今整体环境影响下，以 NFT 和 GameFi 赛道为首的炒作属性偏高的赛道整体估值下降较多，新用户和资金参与量明显下跌，而以 Web 3.0 等基础设施属性较强的底层架构融资情况增长。可以明确的是 DID 与 SocialFi 等赛道会长时间处于发展道路中，寻找相关实际应用和价值层次赋能，且随着未来涌入区块链赛道的用户会越来越多，下一个牛市用户数量可能会达到三到五倍的增幅，用户之间的社交属性和凭证需求会随着用户数等比例增长，所以目前早期部署该赛道，其未来潜力也是无法估量的。

5. DID 与 SocialFi 发展过程中可能的挑战

DID 赛道 和 SocialFi 目前处于早期探索阶段，有着巨大的发展前景，隐藏着许多仍未被发掘的机会，但同时也有着不少问题等待 Web3 建设者们去挑战：

  • 不可转让这一概念本身就存在着技术上的挑战，在限制或防止转移的需求和区块链生态系统之间有着难以跨越的障碍，因为目前来说，几乎所有的项目标准都是围绕着最大可转移性来设计的。

  • 隐私是此类生态系统良好运作的一个重要前提。许多情况下，用户并不希望透露他们所有的身份信息。去中心化的身份信息如何做到仅在必要的时候披露给必要的对象，也是在技术上所面临的一大挑战。

一个只规范资本而不规范人类个人行为的制度可能只会无情地服务于金钱而非人类本身。我们作为 Web3 的参与者和建设者，需要更多的思考和更多的努力来解决这些挑战。而如果我们能够成功解决，则为区块链那些并非只关注金钱的生态系统，打开了一扇通往更为广阔天地的大门。
