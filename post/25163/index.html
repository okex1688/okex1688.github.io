Zenlink

1、技术/产品进展

DEX Protocol

  • 补充完善 EVM 合约版本的测试用户和代码结构，即将开展外部代码审计。

  • 优化改进与 EVM 合约版本其相关的 API 接口服务，包括了交易通知，余额查询，交易流动性查询等。

  • 预留了调整交易手续费接口。通过经济激励的方式，提升生态参与者（交易者、平行链、流动性提供者和做市商等）收益，助力生态发展。

DEX DApp

  • 通过建模抽象和架构重组，DEX DApp 已经成功地同时接入 DEX Protocol 的两种不同实现方式（EVM 合约版本和 Module 版本）。这将有效地抹平用户的认知摩擦，提升用户的使用体验，使得用户可以顺滑地从以前的单链 DeFi 生态迁移到结构复杂的 Polkadot DeFi 生态。

  • 由于 EVM 合约版本的内核机制与 Module 版本有很大的不同，因此 DEX DApp 针对 EVM 合约版本合约做了一轮适应性设计，其中包括了账户地址的转换，增加交易授权流程等。

  • 通过使用 Polkadot 网络自带的 Batch Transaction（批量交易）功能，我们实现了一键完成「授权 + 兑换」的功能。

SlotVault DApp

  • 第一批合作伙伴项目基本已经接入完毕。针对每个项目的不同玩法，用户将会体验到各自不同的使用路径。

  • 完成移动端适配。用户将可以在各大去中心化手机钱包（例如 imToken、Math Wallet 和 TokenPocket 等）直接访问 SlotVault 使用。

Astar (原 Plasm 网络)

1、将 Plasm 重新命名为 Astar Network

因为名称为“Plasm”，因此经常被人认为 Layer2 相关的项目。而实际 Plasm 并不是一个 Layer2 项目，它是 Polkadot 上的多链 dApp 中心，支持以太坊虚拟机和 WebAssembly，并且 Layer2 不是主要功能，而只是一个补充功能。所以 Plasm 团队决定更新品牌，将 Plasm 改为 Astar Network！

2、Sota 接受微软的专访

我们的目标是为 Web3.0 打造一个开放、透明和可持续的平台。Astar Network 将成为一个去中心化的平台。我们已经开始与微软和亚马逊等云提供商进行交流。

3、Shiden 在 Kusama 启动 Crowdloan 活动

Kusama 的第三次拍卖应开始，目前出价最高的是 Shiden，在赢得拍卖后，Shiden 将成为 Kusama 平行链，然后 Shiden 将一步步启动。所以我们的第一阶段是赢得拍卖，在第二阶段准备智能合约功能和收集人以稳定整个网络。我们的第三阶段是集成 XCMP 以开发跨链用例。

4、与 Oraichain 建立合作关系

Plasm Network 宣布与 Oraichain 建立合作伙伴关系，这使两家公司能够共同整合产品、服务和人工智能数据，以扩展两家公司的人工智能生态系统产品。通过这种合作关系，我们将 AI 数据引入我们的生态系统，供开发人员在 Shiden 和 Plasm Network 上使用。

这种新的合作伙伴关系将 Oraichain 与 Plasm 生态系统（Kusama 和 Polkadot 上的 dApps 中心）连接起来，以帮助加快这两个平台的发展。我们正在 Shiden 和 Plasm 网络开发跨虚拟机通信 (X-VM) 库，这使得所有在 Plasm 生态系统中构建的开发人员都可以使用 Oraichain。

5、Plasm & Shiden 背后的 Stake 科技完成 1000 万美元的战略融资！

Plasm Network 和 Shiden Network 背后的核心公司 Stake Technologies 宣布获得 1000 万美元资金的支持，由分布式资本领投，其他知名投资者也参与其中，包括 Hypersphere Ventures、Gumi Cryptos、IOSG Ventures、TRG Capital、AU21 Capital、Digital Strategies、SNZ、 Digital Strategies、Sub0 Capital、Altnomy、East Ventures 等。这轮融资还包括著名的天使投资人，索尼前首席执行官兼董事长 Nobuyuki Idei。

7、Astar 介绍 X-VM

我们对 X-VM 的愿景是建立一个抽象层，允许智能合约从不同的合约引擎（虚拟机）和语言（例如，与用 ink! 编写的 Solidity dApp 交互）执行调用和读取存储数据。

正如 Gavin Wood 博士之前提到的，WebAssembly 是智能合约的未来，但以太坊虚拟机现在也很重要。基于 WASM 的引擎有着光明的未来，因为有像 Patract Lab 这样的许多伟大的开发者正在让 WebAssembly 更容易让开发人员在 Polkadot 上使用。

然而，我们仍然看到了 EVM 的价值，因为我们密切关注以太坊的 layer2 解决方案的最新发展，如 Optimistic Rollups 和 ZK Rollups。在 Web3 基金会资助计划的支持下，我们已经开始在 Plasm 生态系统中实施 Rollups。

我们与Automata达成合作，加强Litentry的身份分数计算层的隐私保护。通过基于可信执行环境Intel SGX的安全硬件方法，Automata确保Litentry的身份计算是在一个隔离和屏蔽的环境中进行的，不受任何恶意行动者的监视。在Automata完成计算后，将使用来自Litentry的请求签名密钥对结果进行加密，并记录在区块链上，外界需要用私钥解密计算结果。这确保外部无法观察到任何计算过程或使用的数据。通过本次合作，我们希望与Automata Network共同防止身份评分生成过程中潜在的数据泄漏。

项目合作：Litentry x Encointer

Litentry和Encointer就DID如何应用于实现这样的系统进行了深入的讨论 —— Litentry的DID解决方案具有高度的Sybil防御性能和匿名性。我们认为将Litentry的DID系统集成到Encointer的协议中将有的应用潜力，能大大简化用户在Encointer dApp上注册和证明身份的使用过程。目前，Encointer正在利用随机的假名密钥签名方来实验性地执行项目。他们的实体出席证明可以用作人格验证，可以用于补充Litentry协议中相关的聚合身份特征。
