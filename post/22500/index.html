为什么需要开放式互联网？

表达的自由极具力量，这也就是为什么表达的自由总被各种框架束缚。有时表达的内容可能是仇恨言论，而有时则可能是错误信息。无论在什么情况下，互联网上信息的自由流通所产生的力量都不可忽视。

前段时间叙利亚的天然气骗局丑闻中，传统媒体联合推动了叙利亚的天然气骗局，引发了另一场毁灭性的战争，导致该地区未来数十年动荡不安的局面。媒体的名声一片狼藉，这一事件引起了公众对传统媒体的警惕。

而开放式互联网对于构建网络信任有重要作用。而拜登政府最近宣布将与私人公司合作，共同监管国内人口，从而规避法律漏洞，更加严格的审查制度发展趋势已经出现。

Web 3.0的出现

从20世纪90年代到今天，万维网经历了两个不同的阶段：

Web1.0——静态页面、简单内容创建、个性化页面。不方便用户使用，所以这主要是只读（read-only）的体验。

Web2.0——此阶段用户参与度增加，从而产生一种新的互联网体验。从YouTube和推特到Reddit和Instagram，用户可以提交内容并进行互动，通过动态页面相互协作，使其成为读写（read-write）体验。

Web3.0仍然是一个理论框架，也被称为语义网，它认为互联网不仅由单个搜索引擎有效管理。Web3.0的关键标志是它依赖人工智能将用户连接到更相关、更有意义的数据。

换句话说，Web3.0应该是机器对自然语言理解的顶峰。此外，语义元数据将允许用户更快速地连接到任何一个连接他们需要的环境和服务。然后，这些内容将无缝地在设备和应用程序之间无缝分发，从而升级Web3.0的体验。

更重要的是，语义网准备将以分布式方式分散用户数据，使其简化并让用户拥自己持有。在当前的Web2.0中，公司会利用你的用户数据，这就是为什么互联网上的大多数服务似乎都是免费的原因。

Web 3.0会是什么样子？

当我们考虑到推动Web3.0或去中心化互联网（DWeb）的进程时，这一领域的先驱Protocol Labs功不可没。你可能听说过，或者拥有他们的Filecoin存储代币。这个项目与作为P2P存储网络协议的IPFS和libp2p互联。后者允许开发人员独立于站点、执行时间和地址服务来运行网络应用程序。

在未来，人们可以根据内容相关性而不是地理位置相关性来访问内容，同时不依赖于可能出现单点故障的单个服务器。

除了Protocol Labs的项目外，其它项目还正在努力替代现有的服务，如：

  •  Textile Photos – Pinterest 或 Instagram的去中心化版本；

  •  DTube –YouTube的去中心化版本；

  •  Graphite Docs – 谷歌文档的去中心化版本；

  •  OpenBazaar – 一个旨在与亚马逊在数字商品上竞争的去中心化市场。

有趣的是，结合了区块链、应用程序和网站的混合替代方案也在崛起。例如，Odysee（以前是Lbry）使用LBRY区块链协议将上传的视频、图像和文档设为不可禁止模式，而网站本身是建立在这个抵制审查协议的基础之上。同样，LBRY也可以用于以微交易的形式奖励内容创造者，消除对广告的需求，防止赞助商腐败现象出现。

实现Web 3.0困难重重，但是柳暗花明又一村

任何服务都可以达到去中心化。尽管困难重重，但技术方面的问题是最不需要担心的。其中更重要的一点是网络效应的重量级效应。换句话说，用最少的努力立即可用的服务，也是最受欢迎的服务。

如果比较Twitter和Gab（美国社交媒体）时，可以发现其在功能上几乎没有区别。然而，不同之处在于两者的受欢迎程度。一些组织正越来越积极地试图扼杀人们的思想和语言，因此，网络效应在去中心化领域可能会出现一个转折点，从而促进DWeb（去中心化互联网）早日面世。

最后，还有一个问题是DWeb（去中心化互联网）不再是中心化形式。解决这个问题的办法可能关键在于DAOs（分布式的自治组织）以及市场的良性竞争。尽管“去平台化”的趋利避害的寒蝉效应（chilling effect）将会取得成功，并为数百万人带来自动审查，但历史的轨迹始终掌握在一小部分企业家手中，Web 3.0的到来将会是互联网历史的转点。
