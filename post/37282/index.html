理解 EVM 等效性

EVM 等效性：完全符合以太坊虚拟机规范

EVM 等效设计理念是生产一种与以太坊“最小差异”的 Optimistic Rollup。

EVM 等效性将以太坊的属性扩展到其 L2。它模糊了以太坊 L1 何时停止以及 L2 Rollup 何时开始之间的界限。

这就是以太坊扩容和以太坊本身扩容的区别。

作为以太坊 EVM 的完美克隆的 Optimistic Rollup 不仅共享以太坊的安全性；他们共享其网络效应的各个方面。

其他 L2 设计结构没有访问以太坊所有网络效应的相同特权，并且总是比它们的 EVM 等效对应物更专业。

EVM 兼容性已死。 要么通过遵循以太坊标准来优化通用性（因此选择与其他人相同的标准），要么构建完全不同的东西，为你的用例高度优化（参见 ZK-rollups）。

为了将以太坊的全部力量完全扩展到 L2，我们需要的不仅仅是 EVM 兼容性。

我们需要 EVM 等效性。

兼容性与等效性

当 Optimism 团队去年引入 EVM 等效性时，他们涵盖了等效性和兼容性之间的技术差异。

Rollups 被誉为我们的扩容救星：“终于，可以在 L2 上运行 Uniswap 了！”

最早的 Rollup 通过完全重新创建 Uniswap 来实现这一点，在自定义 Rollup 之上使用自定义代码。

这还不够。

EVM 的网络效应远不止 Solidity。大量支持工具使以太坊开发者拥有超能力。由于这些工具也在 EVM 标准上运行，因此它们无法用于定制的 Rollup。更不用说协议开发人员为创建与 Solidity 兼容的东西所需的巨大努力！

借助 EVM 等效性，EVM 本身被复制并粘贴到 L2 中。这背后的一切都是一样的。

EVM 是一座城市

David Mihal（他帮助我理解了这一点）给了我这个比喻：

“开源代码就像一座城市。它是由许多发现问题并构建解决方案的开发人员的贡献自下而上紧急创建的。随着时间的推移，这座城市变得优化、强大和高效……支持 EVM 的区块链就像拉斯维加斯版的巴黎；他们正试图人为地复制有机产生的东西。”

开源软件是一种公共产品，由各个社区维护和升级。

使用开源软件的开发者在使用过程中会遇到各种各样的问题；一些琐碎的，一些关键的，以及介于两者之间的一切。一些开发人员花时间解决这些问题，然后游说社区接受他们的意见。如果社区看到了价值，那么贡献就会被合并。创建了一个新标准，并且该软件在实用性和稳健性方面不断增长。

就像一个新兴城市一样，建设者来生产周围社区需要和重视的东西。共享资源和实用程序被生成，并且由于它是代码，它永远不会衰减。这是一条增值的单行道；只要每个人都在同一个基础上工作。

每个开发人员都按照自己的方向进行构建，并发现他们的具体贡献以添加到集体中。随着时间的推移，数千名开发人员的共同贡献会产生高度稳健的公共产品。

Geth

下面是 Geth 的故事。

多年来，Geth 客户端慢慢吸收了以太坊开发人员的有意义的贡献，并开发出一款软件，该软件在以太坊内外拥有一些最强大的网络。

这就是为什么 Geth 已成为该行业很大一部分基础的参考点。已经分叉 Geth 并产生非以太坊区块链的链仍在为以太坊所立足的同一基础做出贡献。

偏离参考版本的 Geth 分叉正在偏离 Web3 中最大的网络效应。一次性副本越往不同的方向发展，就越需要更多的劳动力和资源来跟上核心网络的进步。

EVM Peloton

Pelton 是指（自行车赛中的）主车群，一群或一群公路自行车手。团队中的骑手通过在其他骑手后面进行起草来节省能量。阻力的减少是显着的；骑在一个高度发达的群体中间，阻力可以减少到 5%–10%。对这种潜在节能的利用导致车手和车队之间在比赛战术中非常复杂的合作和竞争互动。

如果你想骑得快一点，就一个人去。想骑得远一点，就一起走。

效率和速度越高，队伍则就越大。空气阻力分布在更广泛的群体中；随着团队规模的扩大，团队的效率也会提高。处于大部队中间则基本上毫不费力。大部队的领导者经常循环担任，因为领队要全力抵抗逆风，并且必须花费额外的能量来设定大部队的步伐。

但是当领队疲倦并且步伐放慢时，有很多新的贡献者一直在大部队内部形成以取代他们的位置。

热带雨林中的毛毛虫在群体中的移动速度要比单独移动的要快。

开源社区的发展速度总是会超过中心化团队的发展速度。这个行业发展如此之快的原因是它是一个协作飞轮。我们以彼此的成功为基础，当我们中的一个人前进到前沿时，他们会把我们所有人都带到他们身边——有点像这些毛毛虫。

EVM 等效性是自下而上的新兴城市类公共产品。另一方面，EVM 兼容性是它的一次性副本。

偏离 EVM 有无数种不同的方法，但只有一种方法可以坚持它。

复制应用层

生成与 EVM 等效的 L2 生态系统对于保持可组合性和互操作性的网络效应至关重要！

EVM 等效汇总允许即时复制 + 粘贴代码库跨链。单个等效于 EVM 的汇总上的开发和创新可以无缝转移到任何其他等效于 EVM 的 ORU 以及以太坊本身。由于一切都是在 EVM 标准上开发的，L1 的网络效应扩展到 L2，L2 上的创新在整个生态系统中回荡。

当部署在 ORU 上时，一行代码意味着更多。如果您是一名开源开发人员并且希望您的代码被广泛使用，那么您自然会想要使用 EVM 等效的 ORU，因为您的代码可以立即与所有其他 EVM 等效的 ORU 兼容。

如果您编写一次代码，但它可以在 1,000 条兼容链上无缝运行，那么您刚刚生成的代码的价值要大得多。

EVM 等效性将 EVM 网络效应提升到一个全新的水平。

非 EVM 等效的 ORU 不会从这些共享网络效应中受益。非 EVM 等效 ORU 中缺乏“最小差异”设计理念破坏了以太坊 + EVM 等效 ORU 网络与非 EVM 等效 L2 之间的关系。

每增加一个 EVM 等效的 ORU，都会加剧以太坊网络效应的巨大浪潮。如果你不在这波冲浪，你将不得不努力游泳才能赶上。

复制协议层

这些 EVM 网络效应不仅仅适用于以太坊的应用层。它也适用于协议层本身；事情真正变得有趣的地方。

因为与 EVM 等效的 ORU 与以太坊的“差异很小”，它们为以太坊提供了一个在实时生产环境中测试新 EIP 的平台。

目前，EIP 都是通过以太坊测试网进行测试。EIP 在测试网上进行了多次测试，以确保在最终集成到以太坊 L1 时不会出现任何问题。

这样做总是有风险的，因为测试网与以太坊并非只有“微乎其微的区别”。在 Goerli 或 Koven 中实现 EIP 与在以太坊 L1 中实现 EIP 是不同的；区别在于以太坊之上经济活动的规模、重要性和性质，这是测试网无法模仿的。在以太坊中实施 EIP 时总会有一些“未知”情况。

EVM 等效性提供了一种解决方案。

当 EIP 在 EVM 等效的 ORU 上成功实现时，它为基础链提供了强有力的保证，即相同的 EIP 可以成功集成到 L1 而没有错误。EVM 等效的 ORU 提供了一个具有真实经济活动和真实资本的实时生产环境，用于测试 EIP。EIP 可以在 ORU 层进行测试，而不会有破坏和影响整个系统其余部分的风险。

当 L2 普遍采用相同的 EIP 时，它会向以太坊 L1 发出信号，表明社区需要它并且可以安全地集成到 L1 中。

EVM 等效的 ORU 使以太坊能够在其 L2 上感知参与者的愿望，使每个 L2 都成为与其用户的愿望和愿望相协调的天线。TradFi 和 Web2 的命令和控制治理被转化为 Web3 的感知和响应范式。

因为每个 ORU 都是自己的主权经济体，它会根据用户的需求独立和异步地实施各种 EIP，与生态系统的其他部分无关。

随着时间的推移，最好的 EIP 将主导 L2 ORU 领域，因为越来越多的独立 ORU 看到了相同 EIP 的价值。当 EIP 在整个 ORU 领域中占据主导地位时，它将向以太坊 L1 发出信号，表明它是一个很好的 EIP，并且在 L1 协议层实施是安全的。

以太坊上将会有许多 Optimism 的分叉！由于它们都接受相同的 EIP，它将向主 L1 发出信号，表明这些 EIP 是可取且安全的。

总结

EVM 等价的意义：以太坊摆脱了 L1 的束缚。“以太坊”能够向外延伸到 L2 中。以太坊 L1 和 EVM 等效 L2 之间的分界线变得非常模糊。以太坊 L1 停止的地方就是 EVM 等效 L2 开始的地方。

所有这些以太坊。

恐怖真空

“大自然厌恶真空”——亚里士多德

大自然真的很擅长填补空白。一个有机体在进化上越适合，它所占据的空间就越多。动物在环境允许的最大范围内消耗食物和繁殖。所有植物都是分形的表达，因为分形是最大化表面积的算法。增加的植物表面积增加了它在叶子上捕获阳光和从根部获取营养的能力。

世界上生物多样性最丰富的地区之一是亚马逊雨林，只有不到 2% 的光到达地面，因为树木在捕捉阳光方面非常高效。

最后一英里问题

最后一英里是指旅程的最后一站，包括人员和货物从交通枢纽到最终目的地的流动。“最后一英里”描述了从枢纽到最终目的地的人员和包裹运输中最困难的最后一部分。最后一英里交付的一些挑战包括最小化成本、确保透明度、提高效率和改进基础设施。

大自然真的很擅长解决最后一英里的问题。这是“适者生存”的自然结果；能够最好地复制和繁殖的有机体来填补它所生活的空白。

即使在单个有机体内部，分形也是增加有机体规模和效率的基本模式。肺负责捕获氧气并分配到血液中；循环系统负责将氧气和其他营养物质分配到有机体的最远范围。

分形是其中每个子结构具有与整体结构相同的属性的结构。分形可以被认为是永无止境的模式，而可复制性和可再现性是分形的根本。

只有可复制和可再现的结构才能有效地填补自然界中发现的空白。加密是一个巨大的空白；还有很多事情要做。但为了用新结构填充所有这些空白空间，我们需要可以复制和再生产的系统。

EVM 等效性产生了产生这些属性所需的基础。

借助与 EVM 等效的 Rollup 生态系统，以太坊可以以极快的速度进行复制和再生产，关注用户的需求并更新其代码以反映这些需求。

每个 L2 都可以朝着自己独特的方向前进，专注于它想专攻的任何事情。成功的 L2 拥有许多用户和大量价值，这将向其他 L2 发出信号，表明它发现了一些有价值的东西。我们都知道加密行业非常好，一旦证明有用，就会复制一些东西。

2013-15 PoW 公平启动，2020 年 DeFi 夏季的收益耕种活动被大量复制，2021 年的 L1 Geth 分叉。阅读：《关于协调与背叛‌》。）

一旦 L2 找到新的价值来源，该价值来源就可以在整个生态系统中复制和共享，并最终带回生态系统的中心。

David Attenborough 的观点：这个应用开发者找到了一些食物！看，EVM 等效性将其带到 L1 以进行共享。这个年轻的开发者肯定会获得无限的资金和一群 Twitter 关注者。

EIP 是新基因

理查德・道金斯 (Richard Dawkins) 的《自私的基因》一书讨论了有机体的每个基因如何成为生命的最小单位，它对自身维持和复制的内在自我利益提供了所有生命赖以生存的基本模式。

好的基因会存活下来。坏基因死亡。

随着生物体的进化和适应，使生物体健康的基因随着时间的推移通过复制和繁殖在整个物种中传播。个体有机体中有益的随机突变有助于它比其他有机体更好地生存和茁壮成长，因此，该基因从仅存在于一个实例中变为存在于所有可能的实例中，因为它是一个很好的基因。

在模块化以太坊的世界中，EIP 是新的基因。

良好的 EIP 传播开来。糟糕的 EIP 会死掉。

以太坊是一个响应式自适应系统，与 EVM 等效的 ORU 允许新基因首先在边缘实施到有机体中，然后在这个新的 EIP 证明其可行性后，EIP 会向外传播到其余的 ORU。如果 EIP 足够好，它将一路回到以太坊跳动的心脏：L1。

以太坊成为一种能够响应和适应其环境的有机体，即使其环境随时间而变化。在生物有机体的基因组从起源固定下来的情况下，以太坊有能力根据需要发明和整合新基因，以跟上世界不断变化的需求。

发现 EVM 等效范式中的价值将该价值转化为可以在整个以太坊生态系统中共享的公共产品，而不仅仅是一个特定的 L2。
