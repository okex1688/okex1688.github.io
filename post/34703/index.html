我最近一直在思考关于 NFT 和 Web 3 社交体验的问题。以下是我的一些初步结论：

1. NFT 将构成我们在线身份的基础

2. 我们购买和赚取的 NFT一起构成了我们的在线身份

3. 策展为王

这篇文章我想更深入地探讨一下关于策展的问题，以下是详细内容。

画廊

与当前的社交媒体内容不同，NFT已经存在于线上。所以你不需要像社交媒体一样去发布新的东西，而只需要对其进行策展。

当今展示 NFT 的最流行的方式是通过画廊。很多 NFT 是数字艺术，所以可以像其他艺术品一样在画廊中展示。

在研究这个问题的同时，我开始着手建立我的 NFT 画廊。这个项目虽然还在进行中，但构建的过程令我非常兴奋。它感觉像是装修房子。以下这是来自 @derekedws 的一个精心策划的画廊的案例（他可能不这么认为，但这确实是事实）。

这个画廊是 Web 3 版的 Instagram， 一个基于你拥有的数字资产和你曾获得的数字凭证而策划的视觉作品集。

我这周回顾了下之前使用 Opensea 的经历，并发现我在“个人资料”页面上，会花相同的时间查看我自己收藏的NFT 以及隐藏的 NFT。老实说，这是我用来查看哪些是我自己拥有的NFT以及哪些是收到的空投NFT的唯一方法。

我认为 Opensea 的“个人资料”页面在该平台上最被低估、且具有最大的发展潜力。如果我是 Opensea 的产品主管，我会花费大量时间来研究如何让用户 (a) 构建他们的个人资料，以及 (b) 查看其他人的个人资料。

画廊最终是一个赌注，但如果能掌握其中的思想，你就可以开始在此基础上去添加具有真正网络效应和竞争优势的功能。

画廊的缺点是缺乏用户参与度。它类似于橱窗商店，用户之间以及他们的 NFT 之间无法产生互动。为了解决这个问题，各大平台都在添加具有社交属性的功能。

社交功能

受 Web 2 社交体验的启发，Context、Zapper 和 Prysm 等类似平台正在添加活动信息、排名及关注他人钱包等功能。它们将 Web 2 社交平台中为大家熟知并喜爱的特点带入到了 Web 3。

这不算是一个重大的飞跃，但它需要以这种方式发生。尽管目前有了新技术 (NFT)，但我们构想出第一波应用程序的灵感来自于我们已有的东西。

毕竟复合创新需要些时间。

元数据

元数据是伴随NFT一起的底层信息。到目前为止，元数据仅限于有关所有者/买方/卖方的信息，你也可以把它当做是产品认证链。与各个NFT交互过的钱包地址和拍卖合约将永远与该资产绑定在一起。

一旦 DeFi 开始接受 NFT 作为抵押品，并且NFT所有者可以将他们的资产投入到市场，那么NFT 所携带的元数据集就会变得丰富，而且NFT 存在的时间越长，其元数据集就越大。

我很高兴看到产品团队可以利用资产的元数据去构造应用程序及体验，这个过程中存在着社交与策展方面的重叠。

代币访问权限

无论 NFT 是艺术品、个人资料图片还是凭证，它对拥有相同 NFT 的人来说，兼作访问和会员资格。NFT是独一无二的，而且大家喜欢这种与众不同的感觉。

我们正在看到 NFT 的所有权赋予了策展人访问权限。在我获得画廊会员卡 NFT 之前，我无法设置我的 Gallery.so 页面，而且会员数量有限。DAO根据NFT所有权对其 Discord频道的访问进行把关的情况也越来越普遍。

在这些情况下，专属性是非常实用的特征，因为它可以留住追随者。

代币访问权限未来将发生很大的变化，并且将有完全基于这种模式的巨型社交平台出现。

角色

我对 Web 3 中角色如何演变感到困惑。

我们今天的在线身份是默认的角色，因为我们共享的所有内容都是提前策展好的。例如，我有意识地选择只在 Twitter 上谈论加密货币的内容，而不谈论关于运动、旅行或任何私人的事情。

这在 Web 3 中更难实现，因为默认情况下，一切都是链上公开的。解决方案是分配多个角色，即用不同的钱包做不同的事情。我在这方面没有做好，这也是我今年想要改进的地方。

总的来说，我认为人们在角色管理方面会逐渐变得更加成熟。我们可以打赌，之后会有一套强大的工具来帮助我们。

但我也意识到人们很懒惰，会尽可能避免麻烦，而管理多个角色是一件繁琐之事。
