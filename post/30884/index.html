本文写于PAKA成立1周年，旨在回顾Polkadot生态在过去一年中经历的重大更新，以及输出PAKA在投资逻辑上的洞见。

我们认为在过去的两年中，Polkadot 已经获得了早期阶段的成功：在 Layer0/1 市场中实现了基础协议的占位；成功验证了 DOT 和 KSM 卡槽经济模型的成立；令人惊喜的是，Kusama 作为试验网络，其成功远超所有早期生态参与者的想象，比如 Kusama 已经如预期的那样，发展出来与 Polkadot 社区相比，更加激进的社区文化和更加活跃的支持者；在 Kusama 上已经有 10 余条平行链平稳运行，并且像 Karura、Moonriver 等智能合约链的 TVL 已经达到了数亿美金。

Polkadot的生态范式是三层结构 ：

  • 最底层 是中继链 （Layer0） ，即 Polkadot 或 Kusama ，提供整个网络的共享安全性，负责平行链的共识保障和基础功能（如跨链消息传递）。

  • 中间层 是平行链和外部桥接的公链（ Layer1） ，平行链包含公益平行链与通过卡槽拍卖接入的平行链，负责业务基础设施，如智能合约基础设施、隐私基础设施，外部公链包括BTC、Ethereum 等价值网络。

  • 最上层 是负责抽象逻辑的协议去中心化应用 （Dapps） ，如基于某条平行链开发的DeFi协议、或者基于公益平行链开发的DAO服务等，还包括未来在冗余平行链上的扩容协议（Layer2） 。

在近一年中，我们注意到了生态的发展已经超越了Gavin博士在白皮书中的预期，涌现了大量新现象和新物种，也衍生出很多新问题和新机会。因此我们将分享 PAKA DAO 的观察给各位读者。

我们发现，Kusama 社区已经成长为一个更加活跃、拥有自己文化特征的社区。从持有数据来看，目前 KSM 的持有者结构相比 DOT 更加分散 。

Kusama 的原生通证  

$KSM 基本上来自于对 $DOT 持有人的空投，并且在2019年主网上线的过程中估值较低，属于典型的社区驱动型项目。这影响了 Kusama 的持有者结构，其持有者大部分是散户，而不是机构。

社区文化上

Kusama 大量鼓励创新和冒险，其社区的 slogan 就是“Expecting Chaos”。并且创新确实发生了：

  • 在 Kusama 上，早期就涌现了“Society”模块的兄弟会文化：Kappa Sigma Mu 是基于 Substrate 社区平台的会员俱乐部。这是一个用社交行为激励的社区组织，旨在激励用户加入一个能够参与规则制定同步协调的社区。财政部通过奖励支付的形式来激励社区成员的加入。目前，Kusama 上只有一个社区，待运行升级后，将有望拥有更多的社区。

  • “治理模块”的快速更新：早期治理模块的“财政库”仅支持提案和批准，然而 Kusama 社区发现需要大量跟进财政库申请的 grant 之后，开发了“bounty”和“tips”，分别对应不同的使用财政库的场景。bounty 针对的是悬赏类的项目，议会可以指定项目经理跟进，并且项目经理在跟进完成后会得到部分奖励；tips是针对于小额奖赏，可以快速拨款给每日社区的激励和报销需求。

  • 出现了像 RMRK 这样的原生创新项目。

社区发展过程中

Kusama的文化性得到了不断正反馈加强：

  • 如果观察 Kusama 的议会，你会发现 Kusama 议会成员的丰富性非常高，并且有大量高影响力的社区领袖（如@DonnieBigBags, @Jam10o)。我们可以基本上判断，Kusama社区已经逐步弱化了和Parity/Web3基金会的关联，成为了一个真正的自治组织。 

  • 宗教性：以 Kusama 的鸟型 logo 为例，通过RMRK的Kanaria或第三方艺术家发布的鸟主题创作空投，社区文化得到了不断强化。你可以从Kusama 推特大V的名称、介绍、语言中印证这一点。

  • 对社区的保卫心态。你会发现，Kusama 社区的自卫心态极强，并且行动力惊人：在 Integritee 社区大战事件中，你会看到社区为了保护自己，具有侵略性的抵制上线了 KSMStarter 的平行链参与卡槽拍卖，甚至发出了提案要求下架该平行链暂时终止该项目的竞拍。

以上这些现象是早期波卡支持者始料未及的，我们可以判断，Kusama 社区将成为与 Polkadot 社区并驾齐驱的驱动能量。并且存在以下可能性：

  • Kusama 生态的创新性可能发展得会更好。

  • Kusama 卡槽拍卖的锁仓量衰减率可能会低于 Polkadot，因为新平行链入场必须赢得 Kusama 社区的支持。

  • 投机性强的资本及该类资本支持的项目，较难赢得Kusama社区的尊重。

  • Kusama 在元宇宙的业务发展可能会远超人们预期。

很有意思的是，这样的现象与 DeFi2.0、Social-Fi 的社区化现象趋同，这意味着资本在背后的能力将会越来越弱。具体体现为：机构无法像之前一样获得大量的廉价筹码、额度越来越少，而需要更激进的加入社区创造之中才有可能成为早期的投资者。

技术探索：公益链、XCM与ink!合约

早期的设计中， Relay Chain 包含了众多区块链基础功能，如资产模块、转账系统、治理模块、小型智能合约（pallet）等，然而在Parity开发演进的过程中，我们发现Relaychain的技术思想已经从基础公链转换为Layer0 ，即仅负责最基础的部分，例如跨链消息路由和共识安全性，而中继链的资产发行、治理等模块，可能都会拆出来成为单独的平行链！

这些平行链将作为“公益平行链”存在 ，即无需拍卖就可以运转的平行链。目前已经上线Kusama 和 Polkadot 的公益平行链是Statemine/ Statemint ，一个基础资产模块，用于发行和管理生态的 token 系统。

公益平行链还可能会开放 ，如：支持ink!合约的平行链 Canvas，与 Moonbeam 的定位类似，但是专注于ink!合约部署。Moonbeam 和 Moonriver 的案例较为特殊：Parity 在开源社区基本实现了 Substrate 的 EVM 全兼容技术 Frontier 之后，由独立的项目方部署。我们认为基于 Canvas 和 Statemint 两个公益平行链实现的去中心化应用将会迎来爆发。

除此以外，第三方团队也可以申请成为公益平行链 。例如 Encointer 团队提出的申请很有可能成为首个第三方团队实现的公益平行链。这也印证了我们对于Kusama社区拥有独特文化的判断。有意思的是，在社区针对此提案的讨论中，除了“公益性”和“利于Kusama”之外的要求包含了“治理重叠”诉求，即公益平行链的议会最好与Kusama议会保持一致。例如 Statemint 就是如此。

我们也发现了一些非常优质的波卡基础设施开发团队 ，他们大多专注于基础设施开发，也暂时没有发行 token 的预期 。这是开源生态进入到更高水平 level 的体现，我们很少在其他生态观察到类似现象，而具备该现象的生态都迎来了基本面的突变，如Arvware。值得关注的波卡基础设施建造商包括：Patract Labs，ChainSafe，Onfanality、zCloak。

在2021年Q3，Gavin Wood 博士连续发表了三篇对于XCM（跨链消息传递）的技术思想阐述。这也是今年中Parity最重要的技术思想升级：

XCM: The Cross-Consensus Message Format

摘要：XCM不是一种消息传输协议，而是一个通用的消息格式，其应用范围不局限于跨链，可拓展至跨共识。

XCM Part II: Versioning and Compatibility

摘要：XCM作为一种通用的消息格式，就像一门语言，突然改变语言会妨碍人们使用它交流，但一成不变的语言无法适应变化的世界，因此XCM需要一个版本管理工具，以实现其有序升级，该工具可以让通信双方互相确认目前使用的XCM版本。

XCM Part III: Execution and Error Management

摘要：XCM的执行环境XCVM 是一个非常高级的、非图灵完备的虚拟机。它是基于寄存器的（而不是基于堆栈的）并且有几个专用寄存器，其中大部分保存高度结构化的数据。XCM可以实现命令的有效执行与错误管理。

我们认为其中比较值得重点关注的是，XCM将不仅仅服务于资产跨链，数据消息传递的想象力可能远超大多数人的认知 。

此外，ink!类型合约的价值被低估了：ink!合约可能会成为首个支持跨链合约级别通信的案例！ 请针对以下场景展开想象：

  • Sushi Swap 在 A 链和 B 链均部署了同一套合约，分别称为A合约和B合约。

  • A资产仅在A链，B资产仅在B链。

  • 目前，如果我希望用A资产换B资产，只能通过跨链桥的方式先铸造B链的A’资产，再完成交换。并且AB合约之间完全没有调用能力（EVM的限制）。

  • 但是在ink!合约中，A 合约可以通过 xcm直接调用 B 合约，资产不需要映射/铸造就可以完成交换。

这仅仅是我们想象力最匮乏阶段的一个案例，实际上跨链合约直接调用的组合性可能会对现有 DeFi 生态带来指数级的效率和丰富度提升 。例如 Phala Network 在实践 ink! 的案例中，将 WASM 部署在可信硬件内部执行跨链通信，该合约甚至可以发送 Http 请求！Phala 对该类合约定位为"胖合约"，即补充传统智能合约不具备的特性，如低延迟、高性能要求计算、隐私性等。我们相信，ink! 合约的能力远不止于此。

我们也观察了目前各个平行链的技术推进情况，PAKA DAO 中的成员也经常交流开发问题。除了繁荣的开发者生态外，我们也识别到一些值得注意的技术问题：

  • XCM 标准在快速迭代中，会对下游的平行链更新和应用部署带来比较多的对接成本。

  • Relaychain 的版本升级比较频繁，同上，平行链的开发占用压力比较大，会导致业务更新的技术资源不足。

  • 平行链平均上线速度比较慢。以 Kusama Slot Auction 为例，从拍到卡槽到平行链部署真正的业务（不仅仅是shell chain），大概需要 30天-60天，即技术冷启动，就会占用了 15% 的卡槽使用时间。

整合型应用的崛起

在波卡生态的前2年，大多数创业项目都会选择成为平行链。然而我们观察到最近事情发生了变化。RMRK 和 Zenlink的崛起给我们带来了新的思路：你是否真的需要平行链？

我们甚至认为，整合类去中心化应用将会为波卡生态带来下一阶段的繁荣 ，并且这类创业方向将成为生态明星！

在波卡官网的wiki文档中，我们看到了智能合约与平行链的对比 ：

  • 当您用智能合约去实现某个功能时，您编写的智能合约必须部署于特定链，且与该链上的某个地址相关联，相比之下，基于平行链开发的 Runtime 或是 Module 以实现您想要的功能时，您创建的是一个独立的状态机。

  • 智能合约必须有意识地实现可升级性，而平行链将能够通过根命令或治理 Pallet 完全换出其代码，升级更加方便。 

  • 当你构建一个智能合约时，它最终会被部署到一个有自己环境的目标链上。平行链允许开发者声明自己链的环境，甚至允许其他人为其编写智能合约。

我们认为在波卡生态中，已经出现了超越这种范式的应用类型： 整合型去中心化应用 。以 RMRK 和 Zenlink 为代表，我们总结了这种范式应用的多种特征：

  • 应用本身的后端逻辑，是去中心化的。

  • 用户与应用的交互，实际上是与链交互。

  • 应用并不是一条平行链，而是一个可插拔模块 ，一个可以（但不是必须）部署在平行链的 pallet。

  • 应用的不同后端模块可能部署在不同的链上。如 Zenlink 的 swap pallet 部署在多条平行链上。

  • 可升级性：应用的升级是灵活的。

  • 应用的 token 发行在 statemint 上。

这类应用不同于EVM链常见的智能合约（如DeFi项目），整合型应用不仅保持了去中心化和智能合约的特性，在技术上也十分灵活 。他们可以通过部署多个 pallet 在不同的平行链上，使用多条平行链的最佳特性能力，整合为体验良好、创新性强的新DAPP。RMRK、Zenlink就是这类应用的先锋者。

我们认为整合型应用将带来XCM的最佳实践，并且会创造出其他公链无法复制的繁荣生态 。

制约因素：XCM版本更新频繁与资产碎片化

在Kusama的头号玩家 Karura 和 Moonriver 上线后，我们注意到了他们的TVL并没有达到最佳预期。

截止11月11日，Moonriver的TVL约为377M，Karura的TVL约为$91M，在所有公链的TVL排行榜中分别排行第18和第34，未能进入Top10。（数据来源：https://defillama.com/chains）

我们推断的一个原因是，波卡DeFi生态产生了流动性不足（除去协议本身的通证外）：

  • Karura 的主要流动性来源为 KSM 的金库抵押，质押可以铸造 KUSD。因此参与铸造 KUSD 的 KSM 数量决定了Karura 的真正流动性程度。而 KUSD 的使用场景，又决定了铸造KUSD的意愿。

Karura TVL分布

  • Moonriver 的流动性来源为 EVM 桥 ，如以太坊、BSC、Fantom等，而目前主要来源桥是Anyswap。

MoonBeam TVL 分布

但是很奇怪，尽管 Moonriver 已经有非常多的 fork 项目诞生，并且社区 fomo 情绪不错，其TVL仍然只有几亿美金。问题出在哪里？

我们认为，这是多个原因叠加因素导致的：

  • 因为 XCM 的版本更新频繁问题，导致多条平行链在互相确认版本、对接调试 XCM 的进展比较慢。这导致了Kusama 的10条运行平行链，只有 Karura 和 Bifrost 链接了起来，更多的平行链还处于孤岛状态。因此平行链的组合性还没发挥充分。这导致了 Moon 生态与 Acala 生态的 DeFi 还没有实现互操作，也导致了Moon桥接的资产/Karura铸造的资产无法在其他平行链使用，因此社区铸造资产的意愿趋于变缓。

  • 资产碎片化：Karura平行链已经有两种铸造的KSM资产（lKSM，vsKSM），而Moonriver上的资产更加繁杂。我们认为这个现象导致了 Polkadot 生态的DeFi项目流动性被切割成了N个碎片，并且因为能够充分利用 XCM 进行资产聚合的应用还未出现，导致流动性分散在各种碎片化资产中，资本效率没有充分发挥。

范式变迁

总结上述观点，我们认为波卡在发生最重要的范式变迁中：

  • 社区：Kusama社区处于强势的驱动一方，甚至已经形成了话语主流。

  • 技术：Relaychain将更加专注于Layer0，新的公益平行链将成为新势力。

  • XCM：XCM更加完备。ink!合约会成为生态利器和黑马。

  • 平行链：上线业务的进度较慢，但是充分上线平行链业务之后，会带来基本面大幅提升。

  • 应用：整合型应用已经出现其他公链生态不具备的特性，并且在爆发前夜。

  • 流动性不足：XCM进度和资产碎片化问题比较突出，但是在Polkadot Defi上线后、XCM版本共识加强后，Polkadot的Defi将赢来新的爆发。

重点 机会

PAKA在近期和未来将会重点关注以下重点机会（不代表投资建议）：

  • 整合型应用：我们会推荐一些平行链项目转型为“整合型去中心化应用”。并且以RMRK、Zenlink作为标杆去探索实践，这将是下一阶段PAKA关注的重点方向，参考案例：SubDAO、SubSocial、OpenSquare。

  • 各个平行链生态孵化的头部应用，以及与Kusama社区文化紧密结合的项目。

  • 能够解决波卡生态流动性不足的协议：例如桥协议，或能够聚合同质化资产流动性的协议。

  • 使用ink!合约技术的应用或开发商。

  • 基础设施：如Subquery、Zeitgeist、zCloak、Manta Network。

  • 元宇宙：如bit.country。
