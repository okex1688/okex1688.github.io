凭证存在于人们的日常生活中，而链上凭证最好的一个例子是用户可以从 AAVE 和 Compound 当中借钱但从未留下被清算的记录，不仅如此，用户在协议首次启动时对流动性池的贡献，或者链上投票历史都会算是一种凭证。

更不用说，过去在 Web2 中的所有行为数据都会被计入凭据里，许多互联网公司在人们的凭据上运行演算法，以便企业端可以发送精准地广告给人们，这就是凭证重要的关键因素，因为它提供了很完整的用户资讯。

但随着 Web3 和 DAO 的进步与崛起，新用户产生的行为数据将呈爆炸式的增长，这也将之提供必要的基础设施以帮助这些新用户建立至关重要的数字凭证，而世界上最大的 Web3 凭证数据网络  Project Galaxy，它建立在开放和协作的基础架构之上，可以帮助 Web3 开发人员和项目利用凭证数据来构建更完整的产品和社区。

今日的凭证数据都被隔离在由应用程序、政府组织以及金融和信贷机构拥有和许可的封闭资料库当中，这就导致了一个分裂的格局，可以对拥有该数据的用户或需要它的应用程序不开放。Project Galaxy 的出现就是要完整开放和协作凭证数据网路愿景，在未来会帮助 Web3 开发人员利用凭证数据来构建更好的产品和更具吸引力的社区。

  1. 项目概述

Project Galaxy 等于构建了一个基于 Web3 DID（Decentralized ID 去中心化身份）系统简单来说，即是我们当前的身份系统是分裂的，不同中心化网站上有不同的身份系统，用户每使用一个新的网站后都需要注册一个账户（如微信、淘宝）。

而随着身份的逐渐增加，也给用户带来许多不便，即使目前有许多网站已经可以提供第三方登录，但是用户的身份系统还未统一，等于说我们无法只凭借一个账号、只用一个身份系统，来登录所有网站，汇集所有资料与凭证。

因此，Project Galaxy 则要通过链上、链下两个维度的多个数据源搜集凭证数据，进行管理，旨在构建一个开放和协作的凭证数据网路，Web3 中心的所有开发人员都可以访问该网络，平台会为社区成员提供基础设施，以便为数据网路管理和贡献数字凭证。

且平台的基础架构支持通过多个数据源管理凭据。对于链上凭证，发行者可以提供子图查询或静态快照，而对于链下凭证，则与 Snapshot.org、Twitter 和 Github 等数据源集成。

通过 Galaxy 凭证为数据网路做出贡献，当凭证用于 Galaxy 的应用程序模块、凭证 Oracle 引擎和凭证 API 时，发行者则会获得奖励，因为在生态系统中有用户消费（付费使用）数据，这就会激励更多用户来管理数据网路。

简单举几个例子：

  • Ａ 项目方想开一个 Ｂ 线上会议，为了鼓励用户积极参与，会让参加这次 Ｂ 会议的人都有资格去 Project Galaxy 领取一个活动纪念 NFT，以增加用户参与率。

  • C 项目方为了奖励那些使用过 C 交易所的用户，在 Project Galaxy 平台制作了 C 勋章。当用户交易金额达到 1000 美金之后，就可以去 Project Galaxy 领取一个 Ｃ 勋章。

  • Ｄ 项目方为了鼓励大家在 Opensea 上使用 Ｄ 网络进行交易，在 Project Galaxy 上面举办了一项活动：凡是交易过五次的用户，可以在 Project Galaxy 上面领取一个神秘宝盒 NFT，交易次数越多可以领取的宝盒越多，最后这些宝盒还可以在 Project Galaxy 上面合成更高级的 Ｄ Genesis Creator NFT。

也就是说，Project Galaxy 就是一个链上凭证基础平台，所有区块链项目都可以加入。这些项目为了鼓励自己的用户多多参与自家项目，就会在 Project Galaxy 平台上面为用户创建一些奖励 NFT，而拥有这些 NFT 的用户则可以被认为是活跃用户。

  2. 应用场景

Galaxy ID：

用户若拥有 Galaxy ID 相当于通用的用户名，只要通过 Galaxy ID 将凭证串联起来，就会形成一个凭证数据网络，用户将其钱包链接到官网后即可生成 Galaxy ID，在此 ID 上会根据用户的历史行为，自动生成“Uniswap V3 交易员”、“OpenSea 交易员”等标签，且用户还可通过 Galaxy ID 显示出在 Web3 世界中所收集的所有凭据，例如 NFT 徽章。

所有流程都可以由用户控制自己的身份，无论是进行认证、授权，而不是由中心化机构介入控制。

一般情况下用户需要支付 Gas 费才能申领和铸造 NFT，不过 Galaxy 为了支持活动发行者代表用户支付交易费用，从而实现无 Gas 的铸造。当凭证用于 Galaxy 的应用程序模块、凭证 Oracle 引擎和凭证 API 时，也就是说数据被用户使用后，也会给予开发者一定的奖励。

开发者可以集成 Galaxy ID 工具包，并根据用户的链上凭证为用户提供专属的功能。目前 Cyberconnect、Unipass 和 Salad Venture 等项目已在其产品中进行使用。

构建和分发 NFT 徽章：

Project Galaxy 为开发者提供应用程序模块、凭证 Oracle 引擎和凭证 API，开发人员可以利用凭证数据来构建更好的产品和社区。简单来说，就是其他项目方可通过 Project Galaxy 的 NFT 基础设施与链上凭证数据网络构建和分发 NFT 徽章，来对社区成员进行管理。

官方称之 Galaxy OAT（链上成就代币），这记录了用户的所有成就，举例来说，Salad Ventures 通过与 Project Galaxy 进行合作，让 Scholar 每达到一种成就可以铸造新 NFT 徽章，再通过 GalaxyID 所展示，让用户的 Gamefi 身份贯穿其整个 Web3 生涯。

若应用在游戏当中，每当用户达到了一定的胜率，即可铸造一种新徽章，若应用在交易上，就可铸造作为 Uniswap V3 交易员的徽章，或者在现实生活中曾购买哪场门票、参加过什么活动，都可以铸造成 NFT 徽章所展示出来，Galaxy 表示 Galaxy OAT 是用户一生所有成就的数字凭证。

而开发者创建流程也不困难，活动开发者只需要提交参数和徽章设计就能得到创建活动和凭证，供给用户参与，所有 OAT NFT 的数据都将存储在由 IPFS 和 Filecoin 提供支持的 NFT.Storage 上。

  7. 链茶点评

目前该项目支持 Ethereum、Polygon、Fantom、Solana、BNB Chain、Arbitrum 和 Avalanche 七条公链，若用官方数据来看，迄今为止已有 100 多个合作伙伴在 Project Galaxy 上发起了 500 多个活动。

当今的 DID 项目如 Civic、uPort，都注重匿名验证以及安全性，应用场景也在物理世界中居多，无论是机器进行匿名年龄验证，或对用户进行 KYC 验证等都是注重用户安全性的最佳代表。

而 Project Galaxy 带给用户的体验大多倾向于身份认证与 NFT 的结合，再加上 Project Galaxy 许多应用模块，给链上世界提供了更完整的基础设施，让许多开发者都可以进行 Galaxy OATs、NFT 忠诚度计划、定制治理等策略，以用于管理社区。
