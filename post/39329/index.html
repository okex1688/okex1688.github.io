dYdX「逃离」以太坊

6 月 23 日，dYdX 宣布将转移至 Cosmos 生态，将基于 Cosmos SDK 开发定制应用链，并在即将到来的 dYdX V4 版本中实现迁移。这可能是第一次出现，一个业内知名的以太坊原生 DeFi 应用选择逃离。

创始人 Antonio 的一席话意味深长：「当选择一条链的时候，我认为人们不应该思考这条链上有怎样的用户，反而应思考该链能实现怎样的产品体验。」

应用层苦以太坊久矣，早在去年 11 月，dYdX 创始人 Antonio 便曾在推特上公开质疑以太坊的提升效率，言谈中已经显露出其正在考虑将 dYdX 转移到其他链。

Antonio 认为，多年间，以太坊在技术落地上缺少作为，而 dYdX 的目标是成为「十倍于当前规模、并能在体验上与中心化交易平台一较高下」的 DEX。

一直以来，Antonio 在行业中以产品经理的面貌示人，dYdX 作为最早期的 DeFi 项目之一，长期致力于开发移动端应用，并力图让去中心化应用（Dapp）走进大众市场。迁移公告中，dYdX 提到，团队不满意于目前 10 笔/秒 的交易速度和 1000 次/秒的下单/取消性能，而扩容需要构建中心化的链下订单撮合系统，这与 dYdX 去中心化交易的定位不匹配。因此，在 V4 中 dYdX 将构建去中心化的链下订单系统，兼顾性能与理念。

但仅仅是因为以太坊生态真的不能满足 dYdX 的需求吗？社区认为 Antonio 决策背后的原因也许更加复杂而多面。

加密 KOL polynya 表示，从技术上讲，dYdX V4 的所需的链性能，Rollups 都可以实现。因此其做出此决策或许是出于链主权和时机的考虑。首先，rollup 技术为时已晚，无法为黄金时段做好准备；其次，行业没有足够的需求来要求 rollup。另外，使用 dYdX 作为生态 Token 可能会存在链的安全性问题和可持续性问题。

对此，Antonio 回应道，迁移确实有链主权的考量，dYdX 独占一条链将使协议更容易从漏洞中恢复（可分叉）。安全性方面，新链初始阶段将控制节点数量以保证安全性。

针对 rollup 技术，Antonio 表示对于 rollup 未来扩展性提升持悲观态度，「dYdX 是现在 rollup 上最大的 dapp，太了解 rollup 了。」

加密分析师 @EffortCapital 表示，dYdX 作为 DeFi OG 选择 Cosmos 这一事件的影响不应低估。其他加密 KOL 对这个决策不看好是因为他们并不了解 Cosmos 生态。

此外，Chainlink 社区大使 @ChainLinkGod 和 Cosomos 生态开发者 Zaki 等人均对迁移表示欢迎及期待。

Solidity 开发者 0xTomoyo 则从技术角度深入分析了迁移原因。当前 dYdX 订单簿是在链下的中心化服务器上运行，L2 仅用于结算匹配交易。理论上他们可以在 L2 上构建带有内存的订单簿，但目前这样的工具并不存在。因此，dYdX 选择在 Cosmos 上构建一条新链，每个验证者都运行一个内存订单簿。这样进一步实现协议的去中心化。

StarkNet 联合创始人 @TobbyKitty 直言，迁移到 Cosmos 的最大原因是为了让 dYdX Token 在新链上运行验证节点，锁定协议价值，但在 L2 上就不行。

最后，针对社区普遍质疑的该决策是否应该交由 dYdX DAO 进行投票表决，Antonio 回应，「DAO 将对该决策进行投票。这只是公司正在开发的软件，协议可以选择使用或不使用。」

协议层将开发者友好的工作量推给 Layer2，Layer2 旁边又有 Infura、InfStones 等大量的 API 提供商、节点服务商，开发工作在经年的生态堆彻中，到底是变得越来越容易，还是越来越难了？不难感受到，Antonio 对于一再地跟随以太坊的节奏已经感到心累。

是生态孕育应用，还是应用反哺生态，本就不是东风压倒西风的事，不同时期矛盾的不同方面引领了事物的发展。

只是 dYdX 的事情告诉我们，在当下的时间，比起争论谁是 Layer2 的老大，也许我们更该听听产品经理们的声音。
