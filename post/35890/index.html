去年，零知识证明(ZKP)在加密领域和Web3中扮演了重要角色，它在可扩展性和用户许可隐私方面都发挥着重要作用。这是令人兴奋的，因为它为ZK应用程序在整个加密生态系统中的推出奠定了基础，并成为Web3的杀手级功能。

Web3

Web3是一个经常出现的术语，但它有点模糊。乐观地说，它的最佳定义是，它将提供Web1的去中心化和Web2的丰富性。并在这个过程当中，解决了Web2的许多问题。

Web2有很多问题——中心化允许黑客攻击、数据泄露、无情地从用户身上赚钱。这些都是它的系统性问题。

Web3带来了以用户为中心的价值观、去中心化和加密技术，这些技术有望解决其中的一些问题。

抛开乐观的最终定义，我想说今天的 Web3 实际上只是从 Web 访问加密的用户体验，以及围绕它构建的生态系统，这开启了许多令人兴奋的可能性；DeFi，NFT，DAO等等。

但是，它缺少发展其更广泛影响力所需要的一些关键部分。

Web3和隐私

对于Web2，我们已经习惯了联合隐私模型。在这种体系结构中，我们保留了个人信息的隐私，不让网络上的普通参与者知道，而把隐私留给与我们交互的特定实体。

虽然这为使用私有数据打开了设计空间，但它引入了巨大的交易对手风险。数据泄漏、私有数据货币化和泄漏数据的永久可用性都是该体系结构的主要缺陷。

另一方面，Web3没有私有数据的功能。这意味着它不需要处理Web2隐私的缺点，但到目前为止还没有这个组件。使用Web3，我们不是与一个中心化的团体进行交互，而是与一个去中心化的网络进行交互。虽然这确实消除了共享数据和交易对手风险的负面影响，但只有去中心的实体才能与之交互，这也带来了如何在新环境中实现信任的挑战。

如果Web3想要在广泛的应用中具有竞争力，那么改变这一现状并引入隐私功能将是必要的。它要么在许多用例中不可行，比如银行账户、在线购物、身份、社交网络和企业，要么在使用加密货币进行其他改进的过程中，创建了比Web2更糟糕的隐私环境。

Web3和安全

Web2 还存在其他明显的中心化问题，除了失去隐私外，还有许多其他对用户不利的结果。从理论上讲，Web3有很大的机会解决类似问题，用去中心化代码来声明其系统的规则。

然而，如今的加密货币却没有这种潜力。如果我们正在运行一个完整节点，那么我们将获得由加密货币实现的完整的、数学上有保证的加密安全性。然而，对于大多数加密货币或用户来说，运行一个完整节点并不可行。它需要昂贵的硬件，并下载和保持与100千兆字节的大型区块链的连接。

这种范式在Web3中尤其成问题。在浏览器中运行的加密客户端根本无法处理这类需求。

因此，Web3今天的工作依赖于可信的中心化中介体，该中介体将连接中继到Web3后面的去中心化网络。这是值得关注的，因为它复制了许多已经存在于Web2中的相关动态。在今年的开始，Moxie Marlinspike详细地描述了这一点——在这个领域中，为数不多的作为中介的中心化服务正在复制与Web2相同的动态，只是表面积要大得多。结合上述的隐私问题，这些实体拥有超级ISP的能力，可以读取和控制对我们所有数据的访问，这对用户主权来说是一场灾难。

这些实体，如Infura和Alchemy，是Web3去中心化机会的弱点。委内瑞拉的用户最近发现他们访问Infura的权限受到了审查。

ZKP的影响

这些安全和隐私问题都是技术问题。幸运的是，ZKP巧妙地解决了这个问题。在隐私方面，ZKP允许用户私下向去中心化的网络共享信息，同时为网络提供安全保证，并且该保证的数据经过了身份验证。在安全方面——它们向用户保证他们从网络接收到的数据是真实的，而不需要信任中间接入点。

这开启了我们在Web2中常用的范例，即与应用程序来回共享可信信息。但是 Web3 的去中心化同时消除了在 Web2 的中心化世界中常见的应用程序背叛信任的可能性。

在隐私方面，考虑身份。假设我们想创建一个NFT集合，其中每个人最多只能拥有该集合中的一个NFT。如果这样做，用户就需要公开自己的身份。然而，使用ZKP，用户可以证明其身份的唯一性，而不需要具体说明他们是谁。在Web2环境中，这将需要一个中心化的实体来跟踪用户并确保惟一性。在一般的Web3中，这将要求用户公开他们的身份和相应的NFT。在带有ZKP的Web3中，这可以完全私密地完成，并且有人们想要从这个系统中得到的所有保证。

以安全为例，假设金融状态的很大一部分是通过web访问的加密货币。随着ZKP证明了链状态，DeFi用户将确保他们在网上看到的账户状态与链上状态真正匹配，保证了其安全性。从用户的角度来看，这类似于从HTTP过渡到HTTPS，减少了审查的风险。

不过，ZKP的新功能并不仅仅适用于将Web2功能开放到Web3。它也将极大地扩展Web3的一系列可能性。对于可能性的一个简短预览，用ZKP增强的应用程序可以这样构建:

  • 允许每个人只能拥有一组 NFT 的 NFT

  • 让用户在不透露他们拥有的特定 NFT 的情况下证明对 NFT 子集的所有权

  • 生成不可转让的 NFT 以识别现实世界或数字成就（即；我是开源项目的主要贡献者）

  • 允许Twitter用户为其关注者创建DAO

  • 解锁DAO的匿名投票

  • 将现有金融数据连接到加密货币，以帮助引导DeFi

ZKP提供的大量增强功能将为Web3打开一个巨大的大门。它将让Web3提供比Web2更好的体验。Web2在处理敏感用户数据和隐私方面遇到了困难，而Web3将凭借ZKP胜出。它将解决用户在中心化网络方面最大的痛点之一，并帮助向Web3的迁移加速，超越了我们在DeFi、NFT或其他带来更多兴趣的浪潮中所看到的速度。

缩小范围，如果我们想创建一个新的、去中心化的互联网，我们将需要安全、规模和隐私——没有比零知识更好的技术了。

为什么是2022

这就留下了一个问题，为什么是现在?其中一个变量是Web3在过去12-18个月的巨大增长，为进一步的实验和开发创造了环境。

这也是ZK的好时机，因为它们现在已经准备好应用于这个领域了。2020年的技术改进使零知识在2021年得到更广泛的推广，使ZK-rollup成为主要的扩展工具。在2021年，ZKP还看到了验证和证明性能的进一步技术改进。

随着这些加密技术的发展，ZK在2021年见证了ZK可编程性的大量产品开发。ZK可编程性意味着使ZK超越可扩展性、zkRollup或zkevm。下一步将带来更强大的dapp，充分利用ZK的核心功能，或者我们称之为zkApp。

2021年，O(1) Labs发布了Mina的zkApp的早期版本。这是开发者第一次能够在像Typescript这样广泛使用的语言中编写ZK代码。O(1)刚刚在2021年12月为Mina的ZK智能合约举行了第一次研讨会和黑客马拉松，今年还会有更多的活动。在Mina，我们看到了一个巨大的机会，可以通过ZK可编程性释放ZKP为开发人员提供的全部潜力。

随着SnarkyJS支持简单的ZK编程，开发人员已经用它来构建，Mina到 Ethereum的zkBridge正在成型，我们很快就会看到上面提到的所有ZK应用的例子部署在Web3上。

无论是Mina推出的第一波应用，还是更多的行业更广泛地接受ZK编程，ZK将在2022年进入Web3 dapp。

对2022年及以后的预测

加密领域的ZK时代正在开始，它将对Web3产生巨大的影响。以下是我认为今年还会发生的一些事情:

  • 随着Web3的起飞，我们将看到对隐私和安全的需求越来越清晰。ZKP将成为最有前途的工具，让用户能够控制自己的数据，并有选择地分享个人信息。这将导致下一次用户大量迁移到Web3，并进一步削弱基于web2的网络。

  • ZK应用将首先在最明显的领域推出：投票和私有身份管理，以及在这一开发中扮演重要角色的rollup。

  • 至少有一个zkEVM将在年底前发布测试版

  • 至少有一家大型科技公司会宣布与ZKP有关的重大事情——现在推出产品还为时过早，但会形成一个团队或主要的研究成果。

在所有这些之间，我们也看到ZKP推出的时间表成为焦点。当我们回头看时，我预测我们将会看到它按照如下的时间线展开:

  • 到2020年，技术基础已经奠定

  • 2020年的今天，底层密码学的功能达到了一个临界点

  • 2021年- ZKP得到了自己的发展

  • 2022年——随着ZKP开始成为差异化、规模化和其他方面的重要组成部分，颠覆开始

  • 2023/2024年——随着ZKP成为产品增长的主要组成部分，颠覆全面展开

  • 2024/2025年——利用ZKP和ZKP平台的应用的主导地位

  • 2025/2026 - ZKP和ZKP驱动平台的标准化

总结

随着ZKP和加密技术成为构建可扩展的、私有的、安全的Web3的主流，将有机会构建真正增强用户能力的技术。

幸运的是，我们在Web3内部看到了以价值为目标来执行产品的意愿。我们已经看到加密技术在这方面实现了从工作证明到权益证明的转变。随着加密货币开始影响更广泛的世界，看看我们还能实现什么令人兴奋的事情。
