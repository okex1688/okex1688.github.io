至今没有一个像 Polygon 一样推出数量如此多的扩容方案。

Polygon 以 2.5 亿美元收购 Hermez 可能是迄今为止第二大的区块链原生协议并购案 ，以太坊扩容方案 Polygon 也因此补全了目前难度最高、基于零知识证明_（ZK）_的扩容技术，朝他们设想中提供的「全栈」扩容解决方案又近了一步。

由于 Polygon 在以太坊扩容领域提供了多方面的解决方案，因此团队曾自比「以太坊扩容瑞士军刀」。其中就包括最核心的 Polygon PoS 链 ，两套成熟解决方案Polygon SDK 和 Polygon Plasma ，以及不久前公开的试图搭建专用的数据可用层的 Polygon Avail 。

而 Hermez 的加入进一步使 Polygon 进一步补齐 Rollup 类的方案，而且是直接一步到位的 ZK Rollup 技术。

Hermez 团队之前是去中心化身份系统 iden3 的开发团队，他们是在开发身份系统中逐渐萌生了探索 ZK Rollup 扩容方案的想法，此前还得到过以太坊基金会 的支持。

随着这一合并的完成，StarkWare 和 zkSync 已经成为了为数不多的能够在提供全套扩容解决方案上挑战 Polygon 的竞争者，这两个团队都提供了两档的解决方案，用户可以从「成本更低」和「更安全」中自由选择。尚不清楚 Optimistic Rollup 类的扩容方案 Arbitrum 和 Optimism 是否会尝试除了 Rollup 之外的更低成本解决方案，毕竟 Rollup 的扩容效率是存在上限的，除非以太坊可以真正部署分片技术。

区块链网络的首次全面合并？

今年 3 月，两个专注于隐私技术的区块链底层协议 Keep Network 和 NuCypher 团队选择进行网络合并，合并后的网络被称为 KEANU 。这也是加密货币行业内第一次提出了合并的概念，所以双方在提出概念后，分别在各自社区进行了治理和投票，逐渐推进网络的合并。

Keep Network 是 Thesis 公司旗下的一个项目，其隐私协议层被用来开发了比特币资产跨链协议 tBTC。Keep Network 本来是为以太坊提供隐私层的方案，利用链下的容器存放隐私数据，而 tBTC 利用了 KEEP 代币和其多签名机制。

而 NuCypher 则专注于全同态加密_（FHE）_技术，他们提供的代码库可以使外部节点对加密数据进行安全、私密的计算。

在提出合并概念后，双方也在随后的几个月逐渐推进了合并的具体细节，两者推出的新代币被称为 T，总量为100 亿，其中 NU 和 KEEP 代币持有者可以分别获得 45%，剩下的分配至 KEANU DAO，由社区治理。而双方的开发团队将保持独立。

相比之下，Polygon 和 Hermez 虽说是代币合并，但其实本质是收购 ，Hermez 的 HEZ 将被吸收到 Polygon 网络中，也最终将换成 MATIC 代币，所以 Polygon 团队表示：「据我们所知，这是区块链网络的首次全面合并」。** **

Polygon 和 Hermez 如何合并？

对于区块链协议来说，合并至少包含三个部分：代币、团队、技术 。

由于 Polygon 是完全收购 Hermez，所以 Polygon 的生态中将只会有 MATIC 这一种代币，Hermez 的 HEZ 将可以通过 1 HEZ = 3.5 MATIC 这一确定的比例兑换成 MATIC。HEZ 代币将在未来的某个具体日期后无法使用，具体细节会逐渐公布。

之前 Hermez 推出的 ZK Rollup 二层网络将被更名为 Polygon Hermez，治理代币也会换为 MATIC 而不是 HEZ。而 Hermez 项目的所有组成部分也将全方位被整合到 Polygon 生态系统中，包括技术、已上线的解决方案、团队等。

Hermez 的核心技术：zkEVM

Hermez 脱胎于以太坊生态的去中心化身份系统 iden3 团队 ，该团队之前希望将 ZK Rollup 技术用在身份证明等高频场景中。后来随着技术的演进，他们计划将这套特定场景的解决方案逐步升级为一个通用解决方案，基于零知识证明技术兼容 EVM，或称为 zkEVM 。在此之前，Hermez 的二层网络于今年 3 月正式上线后，仅支持转账这一特定场景。

iden3 还开发过 zkSNARKS 的电路编译器 Circom，该工具被不少零知识证明相关的团队使用，也曾被以太坊基金会资助支持。

就在上个月举办的 EthCC 4 以太坊社区会议中，Hermez 团队正式公开了 zkEVM 方案，与其他 zkEVM 团队方案不同的是，他们希望通过零知识证明电路实现所有 EVM 的操作码。目前还在探索 zkEVM 的团队有以太坊基金会自己建立的 zkEVM 团队以及 zkSync 团队。

Hermez 团队表示将在本季度对 zkEVM 进行概念验证，第四季度部署测试网，明年二季度启动主网。按照这个进度来看，zkSync 的 zkEVM 方案可能会早于 Hermez 团队实现 。

Polygon 有 4 个扩容方案？远不止

Polygon 在宣布收购 Hermez 团队后，在社区发布了这样一张截图，包括了他们四个扩容方案 Polygon SDK、Polygon PoS、Polygon Avail 和 Polygon Hermez 。

但其实根据他们官网显示，他们还在规划中的扩容解决方案远不止这些。不过上图列出的应该是他们目前最重要的四个：

  • Polygon PoS 就是目前以太坊社区用的最多的侧链扩容网络，有时候会被简单地称为 Polygon；

  • Polygon SDK 是一个模块化和可扩展框架的以太坊兼容区块链网络；

  • Polygon Avail 是一个扩容方案中的数据可用性方案，属于比较新的技术路线之一；

  • Polygon Hermez 是一个基于 ZK Rollup 技术的扩容方案**，** 目前已上线；

除此之外，Polygon 还计划推出三种新的方案：Optimistic Rollup 方案 、独立的区块链方案 、共享安全性的链 ，以及一个已经上线但是被社区逐渐抛弃的扩容方案 Polygon Plasma 。

通过这些方案，Polygon 可以覆盖到数据可用性各异、安全性各异、成本各异的扩容方案，总有一款适合不同的用户。

Polygon Avail 涉嫌抄袭？

前不久在 Polygon 宣布推出通用的可扩展数据可用性层解决方案 Avail 之后，Celestia _（原名为 LazyLedger）的联合创始人 Mustafa Al-Bassam 在社交网络（https://twitter.com/musalbas/status/1409581373329866755）_中称Polygon Avail 的介绍几乎逐字复制了他在 2019 年介绍 LazyLedger 的文章。

LazyLedger 是一个新颖的扩容方向，构建独立的数据可用性层，允许任何人可以快速部署去中心化区块链而不用担心如何启动共识网络。但是目前仍处于早期阶段，团队将进展也都公开在了 GitHub 上，距离主网上线应该还有不少的距离。

针对 Mustafa Al-Bassam 关于 Polygon Avail 涉嫌抄袭的指责，Polygon 团队回应称，介绍文字的雷同是营销和内容团队的失误，实际上 Avail 与 Celestia 是两个不同的架构，主要区别在于 Avail 基于 Kate 承诺，而 Celestia 基于欺诈证明，因此技术层面上存在实质性的区别，不过仍因营销和内容团队的失误对 Celestia 及其社区表示歉意。

技术是否抄袭的确需要专业的来判断，每个人的衡量标准也都不同，虽然对于区块链这样的一个开放社区而言，对借用代码_（Fork）_的行为相对包容，但是大家还是更愿意支持更创新和原创的团队，也有人称之为「正统性」。

Polygon 团队关于 Avail 在营销内容上抄袭的声明虽然暂时解决了这场风波，但是对于 Polygon 来说，也许只有让 Avail 产品尽快落地，才能彻底「自证清白」。

还有谁能提供全栈解决方案？

虽然没有团队能像 Polygon 一样推出数量如此多的扩容方案，但是以太坊社区中的 ZK Rollup 团队 StarkWare 和 Matter Labs 也能为用户提供至少两个解决方案，用户可以从「成本更低」和「更安全」中自由选择。

StarkWare 早就上线了基于 StarkEx 技术的扩容方案，dYdX 就是其中最典型的一个应用，该方案没有将数据可用性放在以太坊区块链上，所以相比之下成本更低，虽然可能会比另一个方案略微降低一些安全性。

而 StarkWare 计划在今年末之前上线 StarkNet 扩容方案 ，可以支持将数据可用性放在以太坊区块之中，这种方案成本会高于 StarkEx，但是安全性可以认为是几乎等同于以太坊主网的。

Matter Labs即将推出的 zkSync 2.0 也是这种结构，虽然技术实现上和 StarkWare 不同。其中 zkSync 账户可以提供等同于以太坊主网的安全性，成本略高，而 zkPorter 账户可以提供极低的交易成本。

而目前仅支持一种类型扩容方案的 Arbitrum 和 Optimism 都是 Optimistic Rollup 方案，他们都是选择将数据可用性放在了以太坊的区块中，然后通过欺诈证明确保二层网络的安全性。

虽然 Arbitrum 和 Optimism 可能是最快上线的通用 Rollup 扩容方案，也就是这几个月的时间，但是他们目前尚未支持更低成本的扩容方案，Polygon 的这次收购可能会让他们很被动。所以不清楚他们是否会考虑通过收购或者开发一套更低成本的扩容方案来满足更多用户，或者对于 Optimism 还有一个更容易的办法，就是拾起他们作为 Plasma Group 期间的 Plasma 解决方案，将数据可用性放到链外，和 zkPorter 和 StarkEx 一样。

特别是在游戏和 DeFi 等多场景应用的爆发后，用户的需求是非常多样化的，或许没有一套方案会满足所有用户。
