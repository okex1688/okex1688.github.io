如今，Discord 在加密市场的影响力与日俱增，成为加密项目和社区必备的工具之一。

而在 Discord 中，Mod 成为普通社区用户与项目开发团队之间最重要的连接器，直接影响项目治理落地实施。

对项目组织者来说，如何挑选合适的 Mod 不仅是一件技术活，也考验运气；而对 Mod 来说，既往工作经历与成就也同样难以证明。二者之间，目前仅靠信任连接，但这显然不够。

如果加密市场有一套类型领英（LinkedIn）的去中心化身份证明系统（DID），通过社区信任背书，可以证明 Mod 既往经历，显然可以解决上述困境。

一、Mod影响社区存亡

不熟悉 Discord 的人，可能对于 Mod 有些陌生。Mod 是 moderator 的缩写，即主持人、仲裁者，中文社区更习惯称之为「版主」，主要负责社区频道管理与日常维护。

一般来说，项目组织者是服务器的创建者和管理者，同时会挑选社区成员作为 Mod 管理子频道。如果我们做一个形象的比喻，项目组织者就是社区的董事长，而 Mod 就是各个分公司的总经理，可见权柄之大。

而在过去一年，随着 NFT 的发展，Mod 的权力进一步扩大，并且到了失控的边缘。

早期，一些用户为了获得白名单资格，乘着项目早期，规则并不完善，发送大量信息水群以提升等级；但在社区发布禁令后，这些用户开始被清算，遭到 Mod 禁言、踢出社群，基本是常规操作。一些用户只是在社区抱怨了几句，同样惨遭 Mod 处罚，处罚来得如此随意。后期，个别项目社区 Mod 更是滥用权柄，利用一些用户对白名单的推崇，诱导用户发送涉黄信息，以此满足私欲。

如果说，上面 Mod 的行为只是影响项目的品牌价值，那下面不靠谱的 Mod 将直接影响项目的社区治理，造成社区成员沟通不畅。

此前，Uniswap Discord 中的社区电话通常每两周举办一次，他们将感兴趣的成员聚集在一起，了解与 Uniswap 相关的项目、工具、初创公司，并对相关问题进行讨论，有些与项目治理有关。但是，在我跟进的数次电话会中，几乎没有听众被邀请进行语音提问，整个活动基本都是演讲性质，长达数个小时，然而社区成员很多关心的问题却得不到讨论和解决。并且，社区成员关心的议题，甚至都很难保证会被纳入议程，有时甚至延迟几周才会得到重视。

而这些，都是 Mod 管理不善，导致社区讨论与治理沦为形式主义，渐渐地社区成员互动欲望越来越低。我并非有意批评 Uniswap，但不可否认，他们的社区 Mod 真的做得很差。

而类似 Uniswap 社区 Mod 这群人，在整个加密社区比比皆是。这些人尸位素餐，职业能力低下，有的甚至有着严重的品德问题，但却管控着社区，造成 Web3 项目创建者与社区成员割裂开来，最终推动整个项目陷入濒死边缘。

二、Web3身份DID，为Mod打分证明

Mod 的存在，直接影响 Wbe3 项目社区的未来，因此选择必须慎之又慎。

目前，项目组织者挑选 Mod 主要有两种方式：一是从社群中挑选活跃用户培养、培训，使之成为合格的 Mod 管理员；二是直接从外部选择有经验的社区经理直接担任 Mod。两种方案各有优劣，前者的优势在于用户忠诚度较高，对项目认知度较高，但缺点是培训时间可能很长，而且培训培训效果因人而异；后者优势是上手较快，但对社区认知度较低。

但无论是哪种方案，其核心点在于：项目组织者应该对 Mod 的过往经历有着清晰的认知，并且得到验证，而不是仅仅依靠 Mod 自述，难以佐证。

如果将挑选 Mod 的过程看作是招聘，传统的招聘过程中，可以通过第三方软件，例如领英（LinkedIn）查求职者的过往职业经历，并且可以得到验证，从而大大提升履历的真实性。而在 Web3 的世界，类似「链上领英」的产品虽然也有，但受限于种种条件逐渐弃用。目前，针对 Web3 Mod 管理者的验证产品尚且处于缺位状态。

比如，国内的升谱科技曾做过“职场通行证”的平台，希望用区块链的方式解决履历造假问题。职场通行证是一款基于区块链技术的个人职场诚信平台，提供包括离职证明、工作证明、收入证明、职场诚信证明等服务；该平台主要面向 B 端企业用户，由负责相关事宜的 HR 录入信息，但受限于该产品的客户群体较小以及落地可操作较大等原因，最终逐渐销声匿迹。

在我的设想中，这样一款面向 Web3 社区 Mod 的产品，是以链上地址作为载体的去中心化身份证明系统（DID）。

具体来说，某位用户成为 A 社区 Mod，其任职期间竭力完成管理工作，表现优异，A 社区可以给其地址发放专属于 Mod 的 POAP 作为激励。当该用户前往 B 社区谋求 Mod 职位时，B 社区可以点击该用户地址的 DID，查看用户过往经历，POAP 奖章可以给该用户证明，同时 A 社区也为其进行信任背书，甚至 B 社区还可以联系 A 社区链上地址进行验证。

上述场景，在 Web3 社区平台 MOJOR 都能实现，其为 Web3 社区提供了一个方便灵活的提供激励的身份识别设施，Mojor赋予社区管理者绝对的权限，可以自行发行任意的属于社区的POAP用来证明Mod甚至每一位社区用户的角色身份，能够无缝高效地解决社区问题，提升社区管理者的治理效率。
