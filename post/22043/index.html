信标链

当前已有超过13.2万有效验证器，质押以太币数量达到了420万 ETH（截至目前价值接近160亿美元），并且最近新存款的比率有所上升。

自两周前的小震荡以来，一切都进展顺利。这是Prysmatic Labs对这次事件的详细回顾报告。

不幸的是，在经历了整整两个月的运行而没有任何罚没情况之后，上个月发生了三起孤立事件，对于事件的起因，我没有任何的信息。

Staking

对于这一点我确实很高兴：看到Adrian Sutton在大约一分钟内从头开始同步Teku。尽管你可以使用任何已知同步的信标节点，也可以使用已下载的包含该状态的文件，但该演示是直接从Infura下载初始状态。这是staking服务和个人质押者的游戏改变者。当同步信标节点的速度如此之快时（几秒而不是几小时），你基本上可以忘记维护持久性存储及冗余设置。

如果你想检查你的验证器如何很好地执行其验证职责，那么可以看看Paul Hauner的电子表格工具‌和说明。与Beaconcha.in提供的单个“有效”数字相比，他提供了更多的见解。

你知道秘密共享验证器吗？如果没有，现在是赶上的好时机。Alon Muroch已撰写了最近的SSV 阶段1测试轮‌。

Rayonism

Rayonism‌，也被称为一种俄罗斯抽象艺术风格，这项正在进行中的努力，与Eth1/Eth2合并测试网以及分片研发工作在同步进行。它是在EthGlobal以太坊扩容黑客松活动上出现的，不过肯定会扩展到更广的范围。

Nethermind团队的Lukasz Rozmej制作了一个非常好的视频教程，演示了如何使用Nethermind为Eth1/执行端以及为Eth2/共识端的Teku建立一个合并测试网。

大新闻是，第一个开发者测试网Steklo，在4月30日（星期五）就存活了几个小时的时间。

Steklo在俄语中是“玻璃杯”的意思，这代表它会很脆弱，而事实也确实证明了这一点。公平地讲，所有团队都在没有准备任何测试向量的情况下潜入了该网络，因此这是第一次盲尝试。 Decrypt撰写了一篇关于Steklo 的文章，总共有十二个客户端组合：Nethermind、Besu和Geth/Catalyst分别与Teku、Nimbus、Lighthouse和Prysm配对。

Lukasz提供了一个关于Rayonism Discord频道的总结：

  1. Lighthouse有共识问题，并且从一开始就出现了分叉（状态根问题），但是在这个分叉上，所有执行引擎均能很好地工作。
  2. Prysm遇到了一个问题（一般问题或共识问题，没有确认），并在genesis创始阶段卡住了；
  3. Besu或Teku有时没有和其他客户端一致，但最终它们达成了共识。当拆分时，拆分为集合（Teku-Geth，Teku-Nethermind）和（Teku-Besu，Nimbus-Geth，Nimbus-Besu，Nimbus-Nethermind）
  4. Nimbus与Nethermind的对话可能在Nimbus端的传输层有一些问题，但它需要Nimbus和Nethermind团队进行更多的调查，最终它在大部分的时间里都能工作-可能需要进行一些重启。

简而言之，发生了很多事情，并且很多事情都进行的很好，总的来说，整个演习非常令人鼓舞。经过一些修复后，Lighthouse后来与Teku以及Nimbus达成了共识。

上周已经准备了一些常见的测试向量，这样客户端可以在再次聚集在一起之前分别进行调试。计划是在下周早些时候建立一个寿命更长的合并开发者网络。

Altair

Altair是计划在年中发布的相对较小的信标链更新。

根据最近的开发者电话会议，各大客户端团队在实施Altair规范方面的进展相当顺利。我们借此机会草拟了一个规划时间表（未作承诺）：

  1. Altair规范将在5月21日左右冻结。
  2. 6月初上线寿命较短的联合测试网（不测试分叉转换，只测试Altair规范）
  3. 试着在6月底之前分叉当前的测试网。我想Pyrmont将是第一个，所以最好尽快迁移到Prater，以避免不稳定。
  4. 七月底八月初部署到信标链。

在冻结规范之前，还有一些事情需要解决。特别是，如何处理这个问题‌（在升级过程中丢失了一个epoch的奖励）。

合并

目前，大多数合并活动都集中在Rayonism上。

尽管如此，在Rayonism之后，仍然还有很多工作要做。Mikhail Kalinin 在Rayonism的Discord频道帖子中总结了公开的话题：

首先，是过渡过程（又称对接）。我们需要弄清楚，在test/dev网络中编写代码并进行尝试。第二，同步算法；设计工作已经在进行，然后进行实施和测试。区块提议优化技术，通信通道协议，无论是JSON-RPC还是REST；用户的JSON-RPC修改（添加最终确定区块方法）。编写描述执行层修改的具体EIP。解决BLOCKHASH随机性问题。多轮各种测试。链工具，找到它们有用的证据。基础设施更新，主要问题是区块浏览器会是什么样。

我们有很多事情要做，但对于年底前完成合并，这是高度乐观的看法，更为谨慎的看法是在2022年第一季度，但没有人会预计比这个时间点晚。
