8月27日消息，以太坊L2扩容方案Optimism推出了新的网关接口，允许任意的ERC20代币在Optimism上进行存取款操作。

该桥接方式取决于Optimism团队设计的新的代币桥接系统，当L1的地址向Optimism转账时，会在Optimism上对应的地址上生成L2代币完成跨链。

Optimism向开发者提供了一个CLI工具来部署新的ERC20代币，开发者可以在生成的地址中输入网关实现与Optimism的交互。此外，该系统的前端采用了Uniswap的代币列表模式来管理受信任的L1、L2代币对。

在Optimism的官方介绍文档里可以查看到，关于代币传输标准桥的描述。

为了在两个网络之间传输 ETH 和 ERC20 代币，Optimism构建了“标准桥”，以尽可能轻松地在 L1 和 L2 之间移动这些资产。

标准桥由两个主要合同组成OVM_L1StandardBridge （对于第 1 层）和 OVM_L2StandardBridge （对于第 2 层）。

存款和取款

#存入ETH

ETH 存入 L2 可以通过depositETH或触发depositETHTo。或者，也可以简单地将 ETH 直接发送到桥接合约。OE（Optimism Ethereum网络)有一个内置的 ERC20 兼容的WETH代币合约，一旦存款在 L2 上完成，就会发行。在 L1 上存入的任何 ETH 都将保持锁定在存款合约中，直到取款为止。

#存入 ERC20 代币

ERC20 存入 L2 可以通过depositERC20和depositERC20To函数触发。存入的代币金额需要经过标准桥的批准才能转移。

#提取资产

ETH 和 ERC20 的提款是相同的，因为 ETH 在 L2 上与 ERC20 兼容。使用withdraw和withdrawTo方法OVM_L2StandardBridge启动提款。

#向标准桥添加 ERC20 代币

任何人都可以向标准桥添加新的 ERC20 代币。但必须在 L1 和 L2 上都有一个代币合约。 L2 代币合约还必须实施IL2StandardERC20 界面。Optimism 提供了该接口的标准实现作为L2StandardERC20合约。

如果L2StandardERC20实现不足以满足要求，可以部署替代实现，只要它符合IL2StandardERC20接口即可。可以自由地将提议的实现部署到 Optimistic Kovan 测试网。准备好经过测试的 kovan 部署后，可以通过请求审核，Optimism将考虑将部署者地址列入 Optimistic Ethereum 主网上的白名单。
