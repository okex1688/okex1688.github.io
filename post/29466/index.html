风险投资家 Chris Dixon 于9月27日在 Twitter 上引发了一场关于“为什么 Web 3 很重要”的讨论。Andreessen Horowitz (a16z) 的合伙人 Dixon 坚信“下一波计算创新将由去中心化技术驱动。 ” 

根据 Dixon 的说法，Web3 是“建立在通证之上的，由建设者和用户拥有的互联网。” 他将这个新时代定位为 Web 2.0（2005-2020）的一种解毒剂，Web 2.0 被 Facebook、谷歌和亚马逊等中心化平台所主导。他说，在 Web3 中，“所有权和控制权是分散的。”

毫不意外，Dixon 的说法引来了热议。有不少人赞同他的观点，但是，也有不少人对此表示异议。主要原因是认为区块链成本太高、性能不够，而且所谓的元宇宙不过是概念，是泡沫，还远远称不上 Web。

这一个方面说明 Web3 处于非常早期，就像其他任何新生事物一样，一开始总是难以被人所理解，受到各方面的打压和排挤，一个新生事物必须自己向世界证明，你是更高效的，能够解决问题的，有更好的应用场景的，或者说是对的，然后才能成长。但是，一些能够提前看到趋势的人，更能够获得早期发展的红利，并随着新技术的发展和采用迅速发展壮大。

那些对 Web3 持怀疑和否定态度的人，还远远没有感受到去中心化所带来的巨大威力，以及新技术发展的指数效应。一方面，Web3 的技术底层正是解决现在 Internet 存在的问题，另一方面，它也需要经历 Web1.0 和 Web2.0 的发展过程。 

Web3 打破互联网企业模式，突破成长困境

集中式平台遵循可预测的生命周期。这个也就是常说的 S 曲线。一个企业的成长，或者产品的生命周期，都遵循 S 曲线。在 S 曲线的上升期，一个产品吸引越来越多的用户，与更多的人合作，出现一个用户、合作者、产品共赢的局面；但是，当体量到一定的规模，合作就变成了竞争，垄断更为重要，系统不具有开放性，这就到了 S 曲线的平台期，也开始进入产品或组织的终结。这是因为达到一定规模后，用户利益和产品方的利益产生了冲突，而用户对产品并没有话语权，产品在利益最大化的驱使下，走向末路，被新的更有生命力的产品逐步积累用户所取代。 

而在一个去中心化的网络中，由通证提供良好的流动性和权益，用户、开发者、投资人是一个利益共同体，他们必须共同寻找办法来完善和做大系统，从而最大化自身的利益。Web3 网络平台和应用有一个特点，就是没有具有绝对权威的公司来管理网络，大家依靠既定的规则行事，对规则的改变必须获得大多数权益人（也是用户）的同意。这就使得系统的演进符合大多数人的利益，是一个完善和提升的过程。

在区块链的整个技术的演进过程中，我们看到了这种发展历程，比特币作为价值储存网络是一个开始，而后，隐私交易网络的出现，智能合约平台的出现，去中心化存储网络的加入，都是Web3基础设施的完善和演进的过程。同时，对单个平台而言，比如 Ethereum，Filecoin，不断地通过提案来进行改进。提案来自社区，来自开发者，同时提案的最终落地必须得到大多数网络维护者的同意才会生效。

这种把现有利益的冲突各方变成利益关联方，产生的巨大威力将使得网络的发展速度前所未有，这一点可以从区块链的价值反映看出来，极少有公司能够向比特币网络或以太坊网络这么快速成长，并在市场上体现极大的价值。

公开和透明性带来前所未有的自由

这里所提到的自由，并不是说为所欲为。世上没有绝对的自由，Web3 的世界也不会有。但是，Web3 天然是反垄断的。垄断可是当前互联网的难题，超级大公司制定规则，把握数据，掌握流量。在这样的世界里，小公司或新技术如果不抱大腿几乎没有生存空间。

而 Web3 是开放的，代码开源，治理公开，交易可查。这不是说没有隐私，相反，Web3 通过加密技术保护隐私，把隐私交给所有人，是更安全的隐私保护。但技术是公开的。这是这种公开和透明性，带来了前所未有的高速发展。任何一个创新都可能被后来者采用。在一个没有围墙的世界里，沟通无障碍，发展当然更快。

Web3 是一个共同网络，各种不同的技术之间更多的是相互融合和协作，因为，协作越多，价值越大。这与Web2.0 存在本质的不同。Web2.0 不同公司的系统是分离的，技术方案是不同的，技术是保密的，是隔离的网络。

作为一个共同网络，一个最有意思的现象就是，产品之间是可以无障碍沟通的，NFT能够被其他平台使用，一个应用的产品可以成为另一个应用的创作材料，不同的平台可以融合在一起成为更大的平台。这得感谢区块链和通证设计，一个应用或其上的产品被使用越多，其价值越大，所以这天然就是鼓励流通和协作，而不是鼓励垄断。自由的流通和链接不再是一个问题。

指数级增长迎来裂变

完全可以理解一些人对 Web3 的到来认识不足，因为在目前确实大家所看到的是成本，能源消耗以及性能问题。前面提到过，任何新技术在早期都碰到类似的问题，汽车的出现让人困惑，这个比马车慢、对道路要求高、维修麻烦、成本不菲的东西到底有什么用？当计算机出现的时候，大家不知道它能用来干什么，曾有专家预测，全球只需要4台计算机就够了。不少人还记得比尔盖茨20多年前解释互联网是什么是一脸的无奈。

但是，这些困惑和不解并没能够阻碍技术的发展和广泛采用。IT 行业的摩尔定律能够解释这个问题。摩尔定律不仅适用于芯片行业，其他在多数的技术行业都实用，简单来说，在一个技术发展的逐步成熟的发展阶段，成本呈现指数式下降，而性能/容量呈现指数式上升，两者综合起来，促进技术的迅猛发展。即使新技术与现有技术相比，成本高很多，性能也不足，但是，其指数级数的发展，将在某一个时刻迎来平衡点，这个时间可能会在很多人意料不到的时候出现，大多数人都没有做好准备，只能是被动追赶。

裂变由于突破音速的音爆时刻，就在那么一瞬间，一旦突破，后续就是势不可挡。

在 Web 3 之前，用户和建设者不得不在 Web 1 的有限功能或 Web 2 的企业集中式模型之间做出选择。Web 3 提供了一种全新方式，结合了以前时代的优势，现在是这场运动的早期阶段，也是参与的好时机。IT的从业者们，是考虑进入 Web3 的时候了。
