前一篇文章中，我们对 Web3 做了基本科普，总结起来就是，Web3 是基于区块链的去中心化生态网络，它的核心特点是用户在使用 Web3 服务和应用的同时，可以拥有和控制自己的资产和数据。

为了让大家能够对 Web3 有一个更清晰和直观的认知，小编搜罗了各大区块链平台和研究机构/媒体总结的 Web3 生态全景图，供大家参考和理解。

1. Alchemy：Web3 开发 堆栈

Alchemy 作为重要的区块链基础设施，对底层技术很了解，他们从技术堆栈角度总结了 Web3 开发堆栈全景图。

Web3 网络层 ：Web3 技术栈的基础是区块链网络层。虽然 Web2 应用程序依赖于集中式数据库，但 Web3 应用程序构建在区块链架构之上，可实现无信任和无许可访问。

区块链交互层 ：选择区块链网络后，Web3 堆栈的下一步是区块链交互层。该层允许开发人员和用户在区块链中读取和写入数据。在考虑 Web3 框架内的数据访问层时，请记住，链上存储是标准，因为它本质上是不可变的，并且允许任何公共个人查看/验证它。

表示层 ：在许多方面，Web3 表示层密切反映了 Web2 开发的许多方面。Web2 和 Web3 前端堆栈不同的一个领域是用于与区块链数据交互的特定库。

Web3 应用层 ：应用层主要允许公共消费者轻松地与直观的前端交互。最终，用户友好的前端使公众能够在日常生活中利用去中心化的互联网。

应用层涵盖 DeFi、NFT、身份和认证、数据和分析以及许多其他令人兴奋的 dApp 类别。总之，这些分散的 Web3 应用程序包括一些最大的原生 Web3 公司，如 OpenSea、Uniswap 和 Aave。

2. Coinbase：web3 堆栈指南

在 Coinbase，我们通常认为 Web3 作为无需信任、无需许可和去中心化的互联网利用区块链技术。

协议层 ：在堆栈的底部，我们有协议层。这由底层区块链架构组成，其他一切都是在其之上构建的。

基础设施层 ：位于协议层之上，由可互操作的构建块组成，这些构建块在执行特定任务时非常可靠。这是一个密集且多样化的层，项目构建了从智能合约审计软件、数据存储、通信协议、数据分析平台、DAO 治理工具、身份解决方案、金融原语等所有内容。

用例层 ：在协议层和基础设施层之上是用例层，所有这些都聚集在一起。

接入层 ：在堆栈的顶端是访问层——作为各种 Web3 活动的入口点的应用程序。您首先需要一个钱包，它是大多数 Web3 应用程序的主要入口点。Moonpay 、Wyre等法定货币入口或 Coinbase 等交易所可帮助用户将他们的法定货币换成加密货币以便开始交易。

3. Arcane research：Web3——用户控制的互联网

Arcane Research是一家研究机构，建立了自己的 Web3 Database，跟踪了近500个 Web3 项目。Arcane 并没有从技术堆栈角度来概括 Web3 全景图，而是从基础设施、可体验 Web3 应用和新应用探索三个层面来展示 Web3 生态全景图。

Arcane认为，Web3 协议可以以 Web2 公司无法做到的方式为社区提供动力。可以通过编纂和执行规则以正确管理共享资源，从而使社区更强大并更快地扩展。更具体地说，Web3 协议可以调整社区的目标、定义边界并创建透明度。

Web3 不仅仅与技术有关，也是一场文化运动。其中许多应用程序可以建立在现有技术的基础上，对最终用户几乎没有影响。Web3 运动之所以如此引人入胜，是因为它之上所体现的信念、理想、模因和情感。社区和共同价值观是每个人类社会和运动的核心。这也是为什么由技术原始人领导的大规模运动如此迅速地在我们的文化中传播的原因。 

4. APIS：以太坊 Web3 全景图

APIS是一种用于读和写的区块链索引协议，自身定位是 Web3 基础设施和开发工具。APIS对 Web3 生态全景图的概括主要是围绕加密资产展开的，一次划分为：稳定币层、服务层、基础设施和开发层、资产管理工具层和去中心化交易平台层。
