作为 NFT 爱好者，每当有知名 IP 爆款或新品上线 Mint 时，我和我的小伙伴总是第一时间冲在前线。Mint 后，大多 NFT 都会以盲盒形式存在数天才会开盒。

在这段时间，我们总抱着一种幻想：万一我的盲盒是稀有款，会不会原地起飞？ 但每次开盒后，结果却是出奇一致：NFT 属性不佳，排名基本垫底，能开到排名靠近腰部都属“珍品”。就我身边的参与者（极小的样本范围）来说，还从未有人开出过排名前 100 的 NFT。

有没有一种可能：经历上述情况的 NFT 玩家并不在少数？毕竟，我们也没少见到玩家在社交媒体晒出自己开出排名第一的 NFT 盲盒。

这些稀有款盲盒究竟去了哪里？在等待开盒的数天时间里，我们的 NFT 盲盒究竟有没有被调换？为什么盲盒尚未开盒时，一些特定编号的盲盒却有人进行高价竞拍？这是否意味着我们的盲盒对一些群体来说是「透明」的？幸运者又真的是靠运气吗？

针对上述问题，Odaily星球日报与多位业内技术人士进行了探讨。

NFTSCAN 联合创始人拾二认为，NFT 盲盒对于项目发行者而言是透明的，所谓的「盲盒」状态只是对于玩家而言是不确定的。

举个例子，玩家在网站 Mint 时，会随机分配盲盒编号（假设 1-10000）；但特定编号的盲盒，在创建时背后对应的图片/视频时确定的、已知的；因此玩家在 Mint 结束时，其盲盒状态对于发行者来说是确定的。

至于为什么总是抽不中稀有款，可能一些项目发行者将随机数进行了调整。假设最稀有的 NFT 盲盒编号是 888，普通玩家 mint 时的随机数范围将跳过这一编号。

即便是项目方将全部盲盒编号都放了出来，你也抽不中稀有款，因为他们理论上可以改元数据，使得你的稀有款变成普通款。

“ 如果 Token URI 的 Jason 是存储在中心化服务器，项目方可以随便改，想什么时候改，就什么时候改。如果 token URI 的 Jason 是存储在 IPFS 的，那就不能改了。 ”拾二表示。（注：Jason 字符串，里面包括了 NFT 对应的图片的存储路径，以及各种属性和参数。）

当然，这并不意味着发行者有着无限修改的机会。有技术人士表示，如果 NFT 上传至 Opensea，会对其状态进行冻结锁定，发行者将无法再此修改数据。

只不过，对于大多 NFT 发行者而言，由于缺少审计，前期的可操作空间较大，且盲盒属于「透明可见」状态，其中留有较大的操作余地。在这种情况下，开盲盒的游戏，从一开始玩家就输了。
