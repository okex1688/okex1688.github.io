MetaMask 的历史可以追溯到有史以来第一次以太坊开发者大会。作为加密领域使用最为广泛的钱包产品，标志性的“小狐狸” MetaMask 将数千万用户带领到了 web3 世界之中 ，与此同时，它的产品设计也遭到不少用户的诟病。 

这是一家富有争议的公司，人们对它有着截然不同的观点。支持者们会强调 MetaMask 令人钦佩的普适性，并认为 MetaMask 才刚刚起步，自然还有很大的优化空间；质疑者们则将其比作 web3 的雅虎，等待着“谷歌”的出现让其黯然失色、退出历史舞台。  

在今天的文章中，我们将解析 MetaMask 在加密领域的地位，概述其发展历史、核心功能和重要性，我们还将揭开其明显的缺陷，思考可能的下一步行动，并列举那些会对 Metamask 产生竞争的玩家。主要包含以下内容： 

  • 加密钱包不再是小众市场 ：加密钱包的用户数在三年前是 3,100 万，而今天这个数据将近 8,000 万。虽然 web3 真正意义上到来还需要一段时间，但像 MetaMask 这样的产品已经开始在大众生活中开始渗透，单就 MetaMask 这一款产品就拥有 2,100 万用户。

  • MetaMask 的收入相当可观 ：2021 年，公司收入超过 2 亿美元。这要归功于其“Swaps”功能，让用户可以在产品内部进行不同加密货币的交换。 

  • 不同类型消费者的期望不尽相同 ：MetaMask 面向技术极客们进行开发，考虑到加密货币的起源，这不足为奇。但随着技术背景偏弱的大众用户开始涉足 web3，MetaMask 将在前端付出更高的设计成本。

  • 加密钱包要建立护城河将非常困难 ：因为数据存储在区块链上，所以加密钱包无法真正绑定用户。用户可以在几分钟内就从一个钱包提供商切换到另一个，并且不会存在数据丢失，这意味着提供商必须找到其他方法来建立防御能力。 

  • 市场竞争正在加剧 ：如今，MetaMask 是这个领域的王者，但还有很多竞争对手正不断涌现，Coinbase 、Block (fka Square) 、 Rainbow、Phantom 以及 Argent 等都不容小觑。 并且，我们正处于加密革命的早期阶段，像 MetaMask 这样的钱包将不得不适应技术和社会的变化。

起源：Vapor

迈阿密

2014 年 1 月，一个相当不起眼的团体在迈阿密的海滨别墅里相聚，这些人为了参加北美比特币会议来到迈阿密，但聚在这间别墅则是为了讨论另外一个完全不同的加密项目：以太坊。

就在几个月之前，当时年仅 19 岁的 V 神发布了以太坊“白皮书”，并在其中概述了他所看到的“世界计算机”的可能性，以太坊很快在加密爱好者群体中引起讨论。 这些聚集在海滨别墅中的人就是其中的一部分。除了 V神外，这次聚会的其他人还包括，Gavin Wood 和 Charles Hoskinson 两位开发者，以及Anthony Di Iorio 和 Joseph Lubin 两位企业家。 

在这些人当中，Joseph Lubin 看起来有些格格不入，除了他比 V 神大将近 30 岁之外，从装束上看他也更像个金融从业者而非技术极客。事实上，Lubin 的确有过一段华尔街的工作经历，从普林斯顿大学毕业并获得电气工程和计算机科学学位后，Lubin 先做了 7 年的数据科学家，随后加入高盛。值得一提的是，在普林斯顿读书期间，Lubin 的室友是如今著名的加密货币投资者 Michael Novogratz（Galaxy Digital 的创始人及 CEO）。

但 Lubin 并不是大众刻板印象中严肃而保守的金融人士，相反，如果了解他的背景，就会发现 Lubin 极具冒险精神和开放心态。大学毕业后 Lubin 的第一份工作志愿是成为一名职业壁球运动员， 2008 年经济危机之后，他还考虑成为全职军人、甚至打算在秘鲁购买一块不起眼的土地作为世界末日来临时的避风港。后来，他还创办了自己的唱片公司。

借助这次聚会，Lubin 不仅确保了他在以太坊革命中先行者的地位，同时在以太坊的核心人群中建立了自己“具有商业思维”的鲜明形象， 从整个以太坊的发展上，这个身份和 V 神形成了很好的平衡，这些都为 Lubin 后来的职业生涯选择以及以太坊历史上的决定性时刻打下了基础。 

Devcon 0 ：Vapor 的诞生

11 个月后，这些人又在 Devcon 0 上相遇了。Devcon 0 是以太坊的第一个官方论坛，在柏林举行，在这场活动中， V神、Wood 以及其他以太坊的早期核心骨干都发表了演讲。 

在 Devcon 0 的论坛上，以太坊团队做出了一个关键决定：以太坊将保持自己的非盈利性。 这一选择使以太坊的领导团队分裂，一部分人继续追随 V 神，另外一些人，包括 Lubin 在内，走向了更加商业化的道路。

在 Devcon 0 期间，Lubin 开始了一个新的冒险：ConsenSys。该公司的核心目标是为建立在以太坊上的企业提供资金、资源和行业人脉等一系列项目孵化服务。

与此同时，在这次柏林聚会的另外一间会议室里，如今 ConsenSys 旗下一项明星资产的前身诞生了。 

Joel Dietz 是以太坊的早期拥护者之一，当时已经在行业中建立了一定影响力，除了在媒体上发表自己对加密货币未来的思考，还在硅谷积极地举办了一系列以太坊爱好者聚会。 此外，Dietz 自己同时运营着多个区块链的项目，其中一个比较著名的是基于比特币的众筹平台 Swarm。此外，他还致力于一系列加密浏览器扩展开发。Dietz 就会议上讨论的一个想法开始行动，他这样回忆道：

“在 Devcon 0 论坛上有很多关于是否需要 JavaScript 客户端和一些 web3 接口的讨论，这些接口在标准的 web 应用程序世界和以太坊客户端世界之间架起了桥梁。”

在会议的其中一个房间里，Dietz 给 V 神、 Gavin Wood 详细讲述了自己的项目计划：创建一个基于 JavaScript 的浏览器扩展项目，试图获得以太坊开发者资助计划的资助。 

虽然并没有被 Dietz 的项目打动，但 V 神还是送了 Vapor 这个名字给项目。

虽然没有获得资金支持，但 Dietz 还是收获了 Vapor 这个“新项目”。回到加利福尼亚后，他开始利用自己人脉网络开展工作。他试图和加州“所有在以太坊客户端上、进行过任何形式演示的 JavaScript 开发人员”交流，在一次项目演示后，Dietz 遇到了一群 Apple 员工，其中就有“JavaScript 黑客” Aaron Davis 。 

Aaron Davis  在加密领域中以其笔名“Kumavis”而闻名，和 Dietz 一样，他很早就意识到了区块链的潜力，并早就开展了一些尝试。在一次众筹活动后，Kumavis 意识到他需要一种“传播链接”，于是得出了一个“以太坊钱包”的构想。

Dietz 和 Kumavis 决定联合打造 Vapor。Kumavis 对 Vapor 很感兴趣，但在 Vapor 拿到启动资金之前，Kumavis 不会离开苹果公司，而是以兼职的方式参与，团队的第三名成员是 Martin Becze ，他 一直住在 Dietz 运营和协助以太坊基金会的联合办公楼中。 

为了项目的融资，三个人报名参加了 Y Combinator，不过在 YC 的面试中，团队并没有将他们想要做的产品定义为“钱包”，而将其描述为 “浏览器与区块链 “的结合，这也和当时整个区块链的发展有关，在当时“钱包”并没有今天这样的语义。 虽然这个项目足够新奇并引起了 YC 的好奇，但在面试后，YC 没有选择投资 Vapor，YC 的合伙人 Blackwell 也在邮件中具体阐释了自己对于项目的观点。

Joel Dietz 提供YC的邮件

Blackwell 在邮件中表示，自己对于 Vapor 最主要的 concern （担忧）在于不确定产品是否能够实现很大规模的用户体量。在他看来，一个智能合约想要大众接受，必须需要建立起非常大规模的信任。之前被大众所广泛接受的东西，比如法币、普通法合约，他们都是拥有了大批信誉良好的早期用户（政府，垄断企业），并以此建立起可以被信任的网络。

虽然事后看来 YC 对于 Vapor 的判断太过于悲观，但 Blackwell 的立场确实也代表了当时大多数人的观点。

Post-YC：Vapor 的分裂和 MetaMask 的诞生

即使没有 YC 的支持，Vapor 的创始团队们也会持续推动项目的发展，但团队成员之间开始出现间隙。

在被 YC 拒绝后，Kumavis 继续进行 Vapor 的开发，但他很快发现，他是三人中唯一做出贡献的人：Becze 专注于其他与以太坊相关的活动，而 Dietz 同时还在参与其他项目， Kumavis 坚称自己完成了 所有工作。 

Dietz 本人也给了 Kumavis 很高的评价，并证实他“完成了大约 90% 的编码和浏览器扩展工作”，Dietz 自己则在一开始“为早期开发提供了资金”，比如支付办公空间和团队餐费。不过，Kumavis 明确表示 Dietz “从未支付过任何费用”，并要求他提供收据。 

在以太坊基金会的赠款问题上，Dietz 和 Kumavis 两人也分别有自己的逻辑和判断，并且问题过于复杂以至于外人无从得知真相。最终，Dietz 和 Kumavis 两人之间的矛盾爆发并导致了团队的分裂。

Kumavis 从 Vapor 的 GitHub 和 Slack 群组中将 Dietz 删除，Vapor 也被更名为“MetaMask”。对于团队的分裂，Dietz 认为自己是被迫离开，并因此受到了不公平的待遇。而站在  Kumavis 的立场，Vapor 在 Dietz 的管理期间实质上并没有得到任何发展，而且他对合伙人缺乏努力感到沮丧，所以他决定和 Dietz 结束合作，并重新申请了新的项目资助。

Kumavis 向以太坊基金会申请了新的资助，申请通过后，Kumavis 把得到的 3 万美元用在了他的新项目 MetaMask 的开发上，当这笔钱花光后，他又继续独自工作了一段时间，出于薪水和更大的发展机会考虑，他最终将项目交给了 Lubin 的 ConsenSys。

加入 ConsenSys 后，Kumavis 又说服了 Apple 的前同事 Dan Finlay 加入 MetaMask。Finlay 的加入对项目的影响立竿见影，最重要的是，Finlay 为该项目带来了一个狐狸标志，如今它已经是 MetaMask 的代称。 Kumavis 强调这一行为带来了全新的面貌，以及它不依赖 Vapor 的代码，他说：“由于我们刚刚从头开始，我们在任何工作功能之前就拥有了狐狸。”

如今，Vapor 已经不存在了，MetaMask 成为了 ConsenSys 的成员项目，Kumavis、Dan Finlay 和 Joe Lubin 三个人是整个项目的核心灵魂。

历史： ConsenSys 的多面性

要了解 MetaMask ，我们还需要对其母公司 ConsenSys 进行了解。在其七年的发展过程中，ConsenSys 也进行了多次的改变以适应剧变的市场。 

ConsenSys 的首席经济学家 Lex Sokolin 对公司发展的各个阶段进行了解释，并总结出了三个不同的转变。我们将研究这些不同的变化如何影响 MetaMask 的发展。

阶段 1：加密货币的“火人节”

ConsenSys 最早试图成为一家加密领域初创公司的孵化器，它将杰出的技术专家们聚集在一起，为他们提供资金支持，并给予他们绝对的项目自主权，和 YC 不同，ConsenSys 的孵化是有宏观目标的，它希望通过自己的孵化来完整构建整个加密生态。 

虽然这是一个很好的想法，但松散的孵化导致了整个 ConsenSys 内部缺少一种明确的组织结构，Rainbow 钱包的联合创始人 Mike Demarais 对这一时期的 ConsenSys 有一个十分生动的形容：“这里有点像有薪水的火人节。” Demarais 密切关注着 ConsenSys 的发展，并且也雇佣了曾经在 ConsenSys 赞助下工作的人。 

这种组织形态非常适合，但同时也阻碍了 MetaMask 的发展。 

一方面，它让 Kumavis 和 Finlay 能够继续推进项目。Sokolin说：“MetaMask 以一种很好的方式，变成了永无止境、不断发展的项目。” 如果有更严格和明确的项目截止日期或者项目截止目标，MetaMask 可能不会变得如此强大。 

另一方面，ConsenSys 的松散意味着该项目几乎没有明确的企业愿景和目标，尽管 Kumavis 和 Finlay 非常有才华，但他们也只是将 MetaMask 视为一个技术开源项目，而非产品。Phantom 钱包的首席执行官 Brandon Millman 曾经也与 MetaMask 团队有过合作，他解释了这种偏好及其背后的逻辑：

“很明显，该产品（MetaMask）是由开发人员为其他技术人员开发构建的，这在当时很合理，因为以太坊的大多数早期用户也都是开发人员。”

尽管在 2014 年，将项目专注面向开发人员是合理的，但缺乏用户视角和对应的规划也阻碍了 MetaMask 的发展，在今天，许多用户对于 MetaMask 产品设计上的抱怨，其实从项目一开始就埋下了伏笔。

阶段 2：收缩和生存

在繁荣的 2017 年后，加密货币进入了长达 2 年的低谷期，比特币从 19,000 美元暴跌至近 3,000 美元，以太坊从近 1400 美元跌至 84 美元。

不可避免地，ConsenSys 的发展也受到了重创。除了公司持有的资产价值暴跌，团队面向加密货币未来努力的信心也在消退。与此同时，市场上对于区块链潜力的讨论开始盖过加密货币。ConsenSys 于是转向服务于这种伪怀疑论者。

ConsenSys 在这一时期专注于为一些大型机构提供知识服务以探索区块链革命。在几年里，ConsenSys 的咨询服务已经覆盖了银行、媒体公司和政府等多种类型的客户。Sokolin 表示，公司已与十家中央银行合作，为数字货币项目提供建议。

然而，ConsenSys 的财务状况并不健康。2018 年底，该公司解雇了 13% 的员工，以改善财务状况。MetaMask 在这一时期幸存下来，标志着它已经成为基础设施的核心部分。

而在 ConsenSys 没有关注加密货币的时间里，公司早期孵化的十家初创公司在没有人关注的领域默默发展，就在 ConsenSys 大规模裁员的 2018 年，MetaMask 的下载量超过了 100 万。  

阶段 3：拆分和精简

从 2020 年开始，ConsenSys 开始重组。

为了筹集资金，2020 年 9 月，公司将其主体拆分为两个独立的实体：“ConsenSys Mesh” 和“ConsenSys Software”，分别对应不同的业务发展规划。 

  • ConsenSys Software ：一家综合软件公司，通过把公司拥有的领先技术资产进行组合形成了一个更清晰的产品矩阵，该架构下包括了 Infura（为以太坊和 IPFS 提供 API）、Truffle（以太坊智能合约开发工具）、Codefi（为商业和金融提供区块链应用套件）、Diligence（为以太坊智能合约提供安全审计） 、 MetaMask（以太坊钱包）， 以及 Quorum（商业服务的开源区块链平台）等六个项目；

  • ConsenSys Mesh ：独立于 ConsenSys 软件业务的补充，将此前的投资和投资组合管理活动从软件业务中分离出来，为致力于发展以太坊和其他去中心化协议的开发者、研究人员、创始人、投资者和社区的网络而运作；

为了促进变革，ConsenSys 公司再次裁员 14%。而这些变革也得到了回报，2021 年 4 月，ConsenSys Software 宣布了一轮 6500 万美元的融资，随后在 11 月又获得了 2 亿美元的投资。 ConsenSys Software 在资本市场上受到欢迎显然和日趋白热化的加密货币市场有关，但来自 Third Point 和 Marshall Wace 等的支持也表明了行业内对 ConsenSys 新结构的信心。 

对于 MetaMask 来说，ConsenSys 的重组则带来了一种新的发展逻辑。 

一方面，ConsenSys 似乎热衷于促进旗下六个项目之间的协同作用，这延续了公司创立之初就设定的愿景，并且由于目前要关注的产品更少，“协同”更能够真正实现。 

同样重要的是，MetaMask 在这个过程中获得了更多的发展资金，重组后的 ConsenSys 除了在融资方面更具优势，在产品之间的资金上也比早期要有更大空间，并且作为产品组合中仅次于 Infura 的明星项目，MetaMask 在资金分配上也具有优势。

MetaMask 在 2021 年迅速发展为世界上最受欢迎的区块链应用之一，ConsenSys 更需要证明它是一个合适的“管家”。 

定义： 是什么造就了加密钱包？

在讨论 MetaMask 的发展前景之前，我们先来看看用户从加密钱包的功能中可以得到什么。 

有关 web3 领域的解析是一个很复杂的问题，因为单一类比很少能捕捉到全貌。考虑到这一点，我们可以从五个角度来类比加密钱包，从而更具体地了解它们的功能、在生态系统中的位置和重要性。 

  1. 作为一个“钱包”

  2. 作为一个银行账户  

  3. 作为一本护照

  4. 作为一个浏览器 

  5. 是一个神奇的“仓库”

作为一个“钱包”

这是个很容易得出的结论，加密钱包因其功能与普通钱包一样而得名。它们旨在保护你的资金并使其可用于消费。就像你可能会在咖啡店拿出自己的皮夹一样，当你准备在 OpenSea 上购买 NFT 时，你需要登录你的 MetaMask。 

作为一个银行账户（或金融科技应用程序）

仅仅用“钱包”来看待加密钱包并不全面，主要有两个原因：

  1. 加密钱包实际上并不保管你的资产。

  2. 加密钱包的功能不仅仅局限于物理钱包。 

让我们来看看以下这些情况。 

首先，传统的钱包确实可以容纳你的钱，但加密钱包却不会，你的比特币或以太坊等资产并未真正存储在 MetaMask 上。相反，这些资产存在于区块链上，你的加密钱包只是管理着一个“私钥”，在这方面，它与银行账户没有太大区别。当你启动自己的银行 app、输入密码，然后查看余额时，你实际上并不是在观察钱本身，而是在看自己的资产。

其次，与实体钱包相比，你可以在银行从事更多种类的活动。

物理钱包只允许用户有效地操作两个命令：加法和减法，即你只能将钱放入（加）或取出（减）。 但加密钱包在交互上要更丰富，更类似于银行账户。除了增加或减少资金，你还可以使用加密钱包，通过抵押、发放贷款、为朋友支付或进行任何不同的操作来赚取利息。虽然这些不在钱包中执行，但是钱包是促成这些关系形成的中介。  

正如你可以使用 Chase 账户获得抵押贷款一样，你可以使用 MetaMask 账户获得加密贷款。 

作为一本“护照”

web3 是一个在线平行宇宙。这个宏观时间中并行着不同的术语、不同的文化规范和不同的货币。 虽然你可以在没有像 MetaMask 这样的钱包的情况下参与部分加密经济，但如果要参与这个领域中的大部分活动，持有一个加密钱包就变得十分重要了。

例如，要使用像 Sushi 这样的去中心化交易所，就需要有一个可以与之交互的钱包。如果想获取 Olympus 提供的惊人的且可能不可持续的利息，也需要这样一个钱包，在 OpenSea 上购买 NFT、玩加密游戏 Axie Infinity 同理。像 MetaMask 这样的公司提供了这样的接口。

虽然你可以选择匿名（或假名）在去中心化网络上冲浪，但同样也可以为自己附加一些身份标志性信息，例如 NFT 或以太坊地址，来标记你的身份。

钱包是一本护照，代表一种身份感，开启一个新世界。

作为一个“浏览器”

得益于在操纵资产方面的实用性，钱包通常是加密活动开始的地方 。例如，如果我想用以太坊交易 Luna，我可以通过 MetaMask 进行。同样，如果我想给朋友汇款，我可能会从钱包而不是 Coinbase （交易所）开始。 

这给了钱包一个优势地位，就像网络浏览器一样。加密钱包作为起点调节着我们与区块链的体验和互动，介于我们和我们想要采取的行动之间。

钱包是神奇的“仓库”

ConsenSys 的首席经济学家 Lex Sokolin 给了数字钱包一个抽象的概念描述：一个神奇的仓库。这是因为它可以包含很多东西，不仅仅是货币，还显示了你的金融、社会、艺术方面的资产和社交关系。

例如，在我的钱包中，你可以找到：

  • 一些加密货币 。主要是以太坊，它的功能最接近法定货币。我用它来购买其他货币和 NFT，或支付使用以太坊网络产生的交易费。

  • 一个以太坊域名 。我通过加密域名提供商 Ethereum Name Services 购买了 mariog.eth，用作链上身份。

  • 一些视觉 NFT 。显然，我拥有一只 Philosophical Fox，以及其他几个 NFT。

  • Generalist 创建的令牌 。今年早些时候，我们铸造了 GENERALIST 代币，作为众筹、多人媒体和 NFT 实验的一部分。

  • 来自 DAO 的一些代币 。我加入了一些 DAO 以对其进行更好的研究，即拥有了象征会员资格和治理权的代币。

这是一个令人眼花缭乱的混合体，它显示了与现实世界中的钱包相比，加密钱包的功能有多么强大。 为了更好地理解这些组合是如何结合在一起的，让我们来看看 MetaMask 的产品。 

产品： 功能齐全但有缺陷

尽管 MetaMask 以其用户服务而闻名，但它还有另外两个利益相关者：开发者和机构。 

用户

目前有超过 2100 万个账户使用 MetaMask，其中绝大多数是个人用户，对于这些用户，MetaMask 实现了四个核心功能： 

  1. 管理： 用户可以看到他们持有的代币并管理他们的私钥。 

  2. 转让： MetaMask 为用户提供了一个接口，可以将代币发送给他人或在自己的账户中转移代币。 

  3. 购买： 用户可以使用他们的 MetaMask 钱包购买代币或 NFT。 

  4. Swap： 通过 Swap ， MetaMask 允许用户通过其他交易所远程交换代币，这项功能被证明是 MetaMak 一个巨大的收入来源。 

  5. 签名： 通过 MetaMask，你可以进行各种交易或链上行为的验证，例如加入 DAO 或铸造 NFT。

MetaMask 以两种形式扩展此功能：浏览器扩展和移动 App，后者于 2020 年秋季发布。

下图就是 MetaMask 在浏览器中的样子。狐狸图标位于角落，一旦登录，用户就可以访问一个微型门户，上面显示着自己的钱包余额，主界面提供了三个关键功能的快速访问入口：购买（Buy）、发送（Send）以及交换（Swap）。

当需要验证某种链上交易时，MetaMask 也会被启动。例如，当你到 OpenSea 上尝试购买 NFT 时，MetaMask 会弹出，要求你“签名”并确认操作。 

OpenSea 和 MetaMask

尽管功能强大，但在使用上， MetaMask 却显得有些冗杂，尽管 MetaMask 试图简化交易流程，但即使对于有一定经验的加密货币投资者，其加密术语、启动时间和 gas 信息的混乱也令人困扰，并且功能也不齐全，例如用户在产品的任何地方都看不到他们所购买的 NFT。

需要注意的是，上面只是 MetaMask 产品的一个方面。虽然它的前端可能存在缺陷，但 MetaMask 的后端却非常出色，因为公司在其架构中投入了大量精力，努力使其成为最安全的产品，这些使 MetaMask 成为了开发者们的最爱。

开发者

MetaMask 令人迷惑的消费者体验可能不是重点，因为这是一款开发者优先的产品。 Lex Sokolin 强调了这一点。他指出团队的首要任务是改进核心产品和加强安全性，紧接着他指出“团队优先考虑的第二件事是开发人员。”

正如他所描述的那样，MetaMask 的存在很大程度上是为了给加密货币开发人员提供一个用户访问接口。而这一想法可以形成一个良性的飞轮：随着越来越多的开发人员使用 MetaMask，产品变得更有价值，吸引更多的消费者，从而吸引下一波开发人员。 

为了自己在开发者端的吸引力，MetaMask 进行了开源，使开发者可以轻松调用产品。这是 MetaMask 与该领域其他产品的真正差别所在，Phantom 的 CTO Francesco Agosti 提到了 MetaMask 这一策略的有效性： 

“他们一直擅长以开放的方式构建他们的开发人员 API，作为标准而不是专有接口。他们非常重视安全性，并将他们用于安全性的许多工具开源，我对此表示赞赏。” 

机构

今年早些时候，MetaMask 宣布其下一个重大举措：为机构客户提供支持，包括基金、做市商和交易员。为此，MetaMask 后台再次强化了自己的安全、托管和交易功能。 

在安全方面，MetaMask 也和 ConsenSys 旗下的另外一家企业 Codefi 进行协同，Codefi 的“可信和自动化合规性”为 MetaMask Institutionnal  (MMI) 提供了支持。托管方面，MetaMask 与提供商 BitGo、Cactus 和 Qredo 进行合作。

MetaMask Institutionnal 在推广时强调了交易流程的简化和更低的交易费用。  

MetaMask

综上所述，这似乎是为大型企业提供服务的强有力的第一次尝试，而且它依赖于 MetaMask 的优势。ConsenSys 一直在深思熟虑，在大型机构中建立了良好的声誉和网络；它应该能够利用这些关系使 MMI 受益。

MetaMask 的高速发展带来的摩擦

MetaMask产品的弱点也与它的背景有关。

整个团队并没有一个完整的组织结构做支撑，而在过去的一年里，MetaMask 又快速出圈、经历了规模化的增长。 在过去一年多的时间里，MetaMask 的月活跃用户增长了 21 倍，从 100 万增加到超过 2100 万。 这些好消息也给 MetaMask 的系统和运维团队带了极大的压力。 在评论 MetaMask 的不足时，Sokolin 表示，“这对团队来说是一种难以置信的压力。” 

MetaMask 的快速增长除了体现在使用人数上，还体现在收入上。2020 年 10 月，该团队推出了“Swaps”功能。如前所述，Swaps 允许用户在 MetaMask 内部进行货币交换，MetaMask 会收取0.875% 的费用。 

尽管被一些人认为费用收取过高，但 MetaMask 也成功地进行了自己的商业化。研究公司 Delphi Digital 的报告称，MetaMask 的 Swap 功能收入在 2021 年达到 2 亿美元。相比之下，像 Sushi 这样的 DeFi 热门产品同期仅赚了 7000 万美元。 

Delphi 对 MetaMask 收入的分析表示： 

“Metamask 的客户获取成本 (CAC) 是一个巨大的零。Metamask 没有代币激励，因此他们的收入几乎没有成本，他们的利润率接近100%。更大的想象空间在于，如果 MetaMask 发行了自己的代币呢？”

这样的建议并非不合时宜。MetaMask 在其历史上多次尝试推出代币。最近，Joe Lubin 在推特上发布了一条关于 $MASK 的建议： 

代币化对 MetaMask 有帮助吗？

似乎发币本身并不能让公司走得更远。

首先，MetaMask 到目前还没有公开阐明一个明确的想法，即自己的代币将起到什么功能。过去的讨论中被简单提及到的是，这种代币允许用户对潜在的新功能进行投票，根据 Demarais 的说法，对于这样一个项目来说，这是一个“最弱的价值主张”。

另外一个风险是，代币只是给项目增加了短期的热度。在我们对 OpenSea 的讨论中，我们概述了 Rarible 的 $RARI 代币如何短暂地让他们在 NFT 市场上占据主导地位，但这并不持久。在没有代币的情况下，OpenSea 仍旧能够以其强大、专注的产品更好地满足消费者的需求，并超越了 Rarible 。Demarais 指出，“拥有代币可以推动你进步，并不一定能让你赢。”  

经过 18 个月的飞速发展，MetaMask 看起来不像是一家需要振作起来的公司，而更像是一家需要重置的公司。Phantom 首席执行官 Millman 对于可能出现的 $MASK 代币的态度是：

“MetaMask 现在需要做的是专注于改进他们的核心产品。将更广泛的「社区」带入一些代币治理过程，只会减缓开发和迭代速度。我想每个人都会同意，我们需要更多的时间去将钱包产品做正确。”

总的来说，MetaMask 在 2021 年的超速发展中幸存下来并进行了有效的商业化，是值得称赞的，同时，为了保持自己的领先地位，MetaMask 还需要适应新的挑战。  

未来： 钱包的发展将走向何方

没有哪个行业比加密货币发展得更快。这个领域以天为单位向前快速发展，这个过程伴随着新公司、技术和金融家的不断涌现。当下，我们还只处于比赛第一局的第一分钟，未来还有比之前更多的野蛮发展。

这意味着，加密钱包还只是处于起步阶段，它们甚至不是半成型的东西，加密钱包设计的服务所面向的场景还在不断发生变化，所依赖的技术也有可能会被更新的技术所替代。

这种波动既带来风险，也带来机遇。人们可以期待 web3 领域的四个直接影响钱包的变迁的因素，其中每一个都会影响 MetaMask 的未来：

  1. 更激烈的竞争

  2. 更大规模的应用

  3. 不断增加的跨链需求  

  4. 更多的应用  

更激烈的竞争

在 MetaMask 正式推出的 2016 年，它的优势之一是极少的市场竞争，而在接下来的几年里，越来越多的人加入了这场竞争，虽然尚未有谁颠覆 MetaMask，但随着新玩家伴随老玩家从多个维度涌入钱包服务，MetaMask 的领先地位或许不会是永恒的常态。

Coinbase 是加密货币存量玩家切入钱包市场的代表。除了既有的交易所业务外，Coinbase 也推出了一款钱包产品，Coinbase 的钱包刚刚推出了一项功能更新，将 NFT 引入了钱包。虽然 Coinbase 足够用户友好，但感觉 Coinbase 不太可能成为该领域的获胜者，这和它的本质角色有关。例如 Swap 功能， MetaMask 汇总了不同的交易所并积极为用户找到最优惠的价格，但 Coinbase 几乎没有动力这样做，Coinbase 的核心业务限制了它在做钱包这件事上的开放性。 Block （原 Square ）也可能面临类似的问题。

去中心化交易所和其他加密项目也在积极涌入钱包领域。

根据 Rainbow 的 Demarais 的说法，Uniswap 和 Aave 也都可能会推出自己的移动钱包产品，但他们在钱包产品上也会面临 Coinbase 和 Block 同样的问题，即期待的钱包产品更多被设计为支持自己的核心业务的存在，而不是作为一个真正独立的项目。 

当然在钱包产品中也有一些新兴项目崭露头角，Rainbow 是最突出的那个，整个团队的理念和成功逻辑也很简单：制作一个普通消费者可以使用的钱包，并提供新体验值得的教育和支持。 

Phantom 也是一个用户友好型钱包，但不同的是，它选择从 Solana 生态切入。随着 Solana 生态的崛起甚至对以太坊形成挑战，Phantom 团队想借用这个新生态的发展红利来发展自己。目前，Phantom 已经是 Solana 用户的首选，最新数据表明，Phantom 每月有 150 万活跃用户，每周有 10 万次新下载安装。  

Argent 专注于以太坊生态系统，强调易用性，用户不仅不需要自己记忆私钥或助记词，钱包内的转账、交易无需手续费（使用元交易的方式将交易费转移到了 Argent），此外，钱包内集成了 DeFi 应用，让用户更轻松就能接入 DeFi 应用。从技术角度来看，Argent 具有强大的安全功能和稳健的架构。 Argent 已经从 Paradigm、Hummingbird 和 Index 等领先公司获得了 1620 万美元的融资。

还有其他钱包项目值得在另一篇文章中进行更多讨论。Trust、Pillar、Dharma、Frame、Balance、Torus、WalletConnect （一个 SDK）等等在生态系统中扮演着重要的角色。除此之外，还有几个未宣布的项目已经引起了轰动。 

加密领域是一个增长得足够快、并且可以容纳多个赢家的市场，这意味着 MetaMask 可以不用太担心竞争，但不能被忽略的是，由于所有数据都存储在链上，“钱包”只是对私钥的管理，所以用户在不同钱包之间的迁移成本极低。

总而言之，加密领域的快速发展使得 MetaMask 面临着大量的竞争对手，以及并不怎么忠诚的消费人群。  

更广泛的应用

MetaMask 将面临如此众多竞争对手的很大一部分原因是因为“加密”正在进入主流领域，一波新用户涌向了这里，包括普通用户、机构和开发人员。  

我们先来看用户规模，三年前，估计有加密钱包用户数量为 3100 万，今天，这一数字逼近 8000 万。 

8000 万的用户体量和其他技术的应用相比仍然相形见绌。2018 年，世界银行统计估计，有 38 亿人在使用传统银行服务，目前活跃互联网用户有 46 亿，我们能看到加密钱包实现类似规模的渗透吗？  

无论能否达到这样的高度，加密钱包产品都在吸引更广泛的受众。MetaMask 在这方面确实有自己的先发优势，虽然产品并不完美，但因为发展足够长、用户规模大，MetaMask 可算作默认的钱包服务供应商。 Rainbow 的 Demarais 指出，“它已成为代名词，已成为品牌，这是我们这些新玩家们的挑战。”

为了取代 MetaMask，Rainbow 也积极地采取了相反的定位，MetaMask 的产品设计晦涩，Rainbow 则主打自己的用户友好和包容性。Rainbow 的社区负责人 Jackson Dame说： 

“Web3 的未来以及接下来 1 亿用户的加入将需要更加用户友好且直观的体验 。我认为生态系统中的许多人并不会使用这些词来描述 MetaMask。”

对于 MetaMask 来说，要赶上“用户友好”的风潮也并不晚，只需要在设计架构上进行调整，但最大的挑战可能是 MetaMask 的底层产品精神针对开发者，这一点根深蒂固，而开发者和用户需求并不完全统一，所以也不可能对 MetaMask 进行真正的改造。  

根据 Demarais 的说法，“MetaMask 一定不会成为我弟弟手机里的 App。” 

但 MetaMask 在机构方面 的定位似乎要好得多，公司的悠久历史和过去的客户积累都是很好的背书，推出 MetaMask Institution 是一个明智的行为。

MetaMask 过去持续地关注开发者 也会发挥作用。WalletConnect 等产品使 dapp 的开发人员可以更轻松地同时实现和许多不同钱包的集成，在这一过程中，MetaMask 通常是第一个调用端口。即便目前团队似乎没有特别准备好迎接“消费者革命”，但 Kumavis 和 Finlay 在了解“工具开发人员下一步可能想要什么”这一方面可能是世界上最好的。 

不过，Phantom 的 Agosti 认为，仅仅赢得开发者的青睐是不够的，他的看法是： 

“钱包确实具有良性循环和各种网络效应。开发人员的使用是一个间接让 Metamask 等钱包快速增长的途径，但用户并不是因为要使用 Metamask 而下载 Metamask；他们下载 Metamask 是因为他们想铸造或购买 NFT。 谁能完全赢得用户和开发者的心，谁就能从这个良性循环中获益最多。”

不断增加的跨链需求

一年前，Solana 的交易价格为 1.80 美元，而在最近，它的价格已经接近 200 美元，激动人心的价格上涨是该项目在短时间内崛起的标志之一。在未来几年里，我们可能会看到更多其他公链项目重复 Solana 的路径，也就是说，我们正在走向一个多链、多网络的未来。例如，一个人可以同时选择以太坊、比特币、Solana 和 Terra 作为交互方式。  

这种扩散对 MetaMask 和其他钱包制造商产生了影响。这些公司是否应该接受多网络现实并扩展其产品？或者加倍努力和专业化，为一个网络构建丰富的功能集？没有人知道正确的举动该怎么做。

增加对某一条新链的支持需要大量工作， 它不仅增加了技术复杂性，而且增加了用户使用时混淆的可能性。根据交易类型在不同网络之间进行交换需要一定程度的知识，而这种教育需要时间。 

但是如果用户不断在不同的链之间交换，那么缺少这样的功能对于产品的发展则影响重大。Phantom 的 CTO 、联合创始人 Francesco Agosti 巧妙地总结了这一困境： 

“如果有几条市场份额相当（用户数量，交易数量）的链，并且这些链服务于相似类型的应用程序，那么我认为多链钱包是一种更好的体验。如果链的工作方式、链的用户群或这些链提供的使用场景之间存在显著差异……那么答案就不清楚了。多链钱包将难以与专门针对特定链上特定利基市场的钱包竞争。”

目前，MetaMask 主要支持以太坊及其生态系统，就目前而言，它还不需要做太多其他事情，但如果像 Solana 和 Terra 这样的项目继续繁荣，它可能就要开始考虑对不同链的支持。 

更多的运用场景

通用钱包的概念会慢慢消失。

随着越来越多的交易和互动发生在区块链世界，也许会带来服务特定交易场景的垂直产品机会，并且已经有一些钱包在一些利基市场中站住了脚。

例如 Gnosis Safe 是面向 DAO 的资产托管产品，它的多签功能使很适合 DAO 的公共金融结构。游戏领域也出现了类似的趋势，Wombat 就是一个面向区块链游戏的钱包产品，已经吸引了接近 100 万用户。 随着时间的推移，会有越来越多的细分行业需要自己的钱包产品。

Agosti 再次很好地概括了这种潜力： 

“接下来的 5 年中可能出现的大规模加密浪潮会将钱包的市场扩大一个数量级，从这场新浪潮中受益最多、或自己创造浪潮的人会是最终的赢家。”

MetaMask 不应该被期待抓住其中所有的机会，但它至少要确保自己不会作为前浪被冲向沙滩。目前来看，MetaMask 还不是对一系列变化做出最快响应的那个。 

MetaMask 是加密领域的传奇英雄。过去六年多，它的团队一直致力于打造一种虽然不完美、但足够有效的产品，同时它对于开发者而言也是一个值得信任的原始程序。 通过这些，MetaMask 帮助数千万用户和数千名开发人员打开了 web3 世界。Demarais 认为：“如果没有 MetaMask，我们就都不会在这里。” 

如此重要的贡献值得尊重，并不意味着舆论的漩涡会绕过 MetaMask。随着越来越多的大众用户涌入加密领域，他们会对 MetaMask 的前端提出更高的设计水平要求，在这些差距被填平之前，各种各样的抱怨将继续下去。 

MetaMask 自身或许并不介意，毕竟它已经经受住了大量的考验，经受住了母公司不断的变化和加密市场的残酷与变幻莫测。即使不受欢迎，也仍然有用，这就是 MetaMask。
