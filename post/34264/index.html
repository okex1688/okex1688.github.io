Web 1.0，是我们当前Web 2.0立足的一个传统概念。世界目前正在从 Web2 过渡到Web3 ，也因此出现了许多关于 Web3 会是什么样子的问题。

虽然本文没有深入探讨 Web 的起源及其第一个演进阶段，但它概述了建立 Web2 的基础以及即将到来的 Web3 的基础——自由和语义 Web 的未来。语义网，有时也称为 Web 3.0，是通过万维网联盟制定的标准对万维网的扩展，旨在使互联网数据具有机器可读性。

请注意，本文交替使用“Web 3.0”和“Web3”等术语。

前言

我们今天所用的互联网很大程度上是基于Web 2.0的，那么Web 2.0到底是什么呢？

对于许多人来说，Web2.0主要是社交媒体，比如说Facebook，但这并不是很准确。确实，社交媒体是Web2演变的一大部分，并且Facebook也的确是这个时代最重要的贡献者。Facebook创建了Messenger，并提供了第一个全局采用的Web平台，允许向Web添加另一个应用程序，为他人开辟许多可能性。这些创新动态地改变了Web环境。但是Facebook不完全是Web2的定义。相反，Facebook和其他媒体组织是首批资本化Web2的基础技术，因为他们的出现，使得互联网作为可以构建应用程序的多层平台的基础架构。他们是Web2的先驱，但Web2并不是他们，就比如说，爱迪生的灯泡是一种电力的创新，但电力不是爱迪生的灯泡一样。

Web 3.0 概念从 Web2 开始停滞的地方开始。其主要目标是通过使用区块链和零知识证明 (ZKP) 技术来改善 Web2 的缺陷（缺陷主要集中在数据保护和隐私领域）。 Web3 并不等同于以太坊网络或建立在特定区块链上的 Web，也不等同于浏览器中的 MetaMask 扩展，即使它们可以用作通往Web3的接口。Web3可以通过无限的创造力来改善您的Web体验。但是无论Web3会发展成什么样子，我们都不应该忘记这个即将到来的时代最重要的方面：对您的隐私和自由意志的主权保护。

在 Web 1.0 时代之前

互联网，从万维网（WWW）开始发展。互联网一词封装了使用物理布线和网络连接协议来管理计算机之间的通信方式。互联网在 60 年代和 70 年代被设想为一个学术和军事项目，在 90 年代开始用于商业服务。

在本文中，您将大量地阅读到Web这个词，它来自一个相互关联(蜘蛛的)网络的隐喻，反映了Internet被用于在Web浏览器中显示相互关联的Web页面和应用程序。

Web 1.0——只读的静态页面的生成(大约1989-2005年)

在Web1时代建立的网站在今天看来非常简单。它们通常由文本和低分辨率的图像组成，因为当时互联网的连接速度很慢。Web1网站通常是一个只读的静态网站——这意味着需要网站管理员在线发布他们的内容，并且用户只能被动地从屏幕上获取信息。用户能够与之互动的内容并不多; 今天标准的常见功能 - 比如点“赞”，上传数据，或留下评论在当时来看都是比较少见的。

Web1.0的关键特征

  1. 内容从服务器文件系统提供

  2. 文件只以“一个WebForm和一个文档”的形式存在

  3. 具有有限用户交互的只读Web，用户无需登录或注册

  4. 通过可点击的超链接，将静态页面连接到互联网

  5. 服务器端使用通用网关接口 (CGI) 构建页面

  6. 框架和表格用于定位和对齐页面上的元素

  7. 基于关键字的搜索

  8. 缺乏网络标准

  9. Flash 和 Java 的首次出现

  10. 少量的横幅广告

Web 2.0——动态网站的生成(大约2002年至今)

Web2通过异步模式增加了Web1的能力，允许用户以同步请求的方式查询web和服务器——因此可以以不同的延迟来回答，而不会冻结或延迟通信。- JulioMoros (Sovryn)

我们今天交互的所有页面都是 Web 2.0 的产物。让我们看看 Web2 与 Web1 的不同之处：

  1. 来自Web页面的信息与 1.0 一样，但Web也从我们这里获取信息；这些Web页面拥有增强的功能和更好的外观；由于具有更高的数据吞吐量能力，所以一切皆有可能

  2. 就可用内容的数量而言，Web 的内容数量高于所有其他媒体

  3. 网页可以和程序链接在一起，形成了功能丰富的动态应用程序，可以在多个设备上使用，而不是简单的页面之间的超链接

  4. 可以通过关键字搜索和内容分类查找信息，另外还可以通过给用户添加“标签”（通常是一个词或两个词的简短描述）来促进搜索

  5. 使用聚合技术(如RSS提要)通知用户内容更改

  6. 使用来自用户的信息（例如位置）并根据它调整 Web 内容

  7. 使用用户数据并根据用户需求、偏好提供内容或服务的Web

  8. Web用户行为跟踪

  9. 自由分类的信息，允许用户检索和分类信息

  10. 通过在线评论，信息在网站所有者和网站用户之间流动

  11. 开发允许自用的API，例如软件应用程序

  12. 更多互动广告

自 2002 年以来，Web2 逐渐兴起，其主要功能之一——MSN Messenger，于 1999 年推出——在 2003 年左右开始流行。70 年代和 80 年代出生的人经常使用它！不久之后，在 2004 年，Facebook 出现并重塑了我们对社交媒体的看法。

虽然 Web 1.0 主要是文本和小图像，但 2.0 版本包含大视频和任何其他可以通过高吞吐量共享和传播的数据。 Web 2.0 既是一个构建创新技术的平台，也是一个从 Web 开发的角度来看，用户被视为一个流对象的空间。网上银行、网上购物、Uber、Airbnb 等应用程序都连接到 Web2 并在其上创建特定服务。

从 Web1.0 到 Web2.0 的主要飞跃在于使用网站进行交流和响应——最初是以用户在文章底部留下评论的形式。这项创新首次允许读者将他们的内容添加到他们正在阅读的页面中。这也是用户相互交流的第一步。第二步，允许用户创建个人资料并将他们的数据上传到网络。最后，他们能够创建群组、与他人的内容互动，并扮演管理员或版主的关键角色。

Web2 的这种动态交互特性通常被描述为读写网络。

Web2.0 阶段也迎来了信息热潮，因为您可以从各种来源搜索并找到问题的答案。尽管社交媒体经常被引用为 Web 2.0 的定义，但它只是其中的一部分。由 Facebook 推动的最近趋势是将 Web 2.0 视为支持其他应用程序的平台。这是通过打开 API 并允许用户将应用程序添加到他们的帐户并与应用程序共享一些信息（例如与他们的社会动态交互）来实现的。

因此，Web 2.0 成为了博客、论坛、wiki 和社交媒体的时代，WordPress、维基百科、Twitter、Facebook、Instagram 等就是证明。

然而，也许最重要的是，Web2 还彻底改变了网络浏览器的“问答”交互方式，它现在为可能的支出提供建议，例如餐馆或购物的小费。此外，看似无处不在的定向广告受到您在线活动的严重影响。例如，如果您在人们讨论特定视频游戏的线程中单击“喜欢”，您可以预期相同或相似的内容也会显示在您的社交空间中。

您可能已经注意到，提供给您的网络广告、产品和建议与提供给您的朋友的不同。这是因为 Web 收集有关您的在线活动的信息，例如您的消费行为，并使用它来定制 Web 上的内容以符合您的喜好。虽然 Web2 的早期版本允许您购买特定商品而无需单独吸引您的兴趣，但近年来，您的特定行为已成为数据收集者的目标。

简而言之，除非您在在线隐私措施上投入大量精力，否则您必然会受到网络的监视。

正是这种对隐私的需求以及同时保持自由使用互联网的各种功能的需求（以任何形式表达自己，使用互联网服务、游戏或简单地消费数据），产生了正是 Web3 需要解决的关键挑战。区块链技术的粉丝可能现在已经开始了解，但其他人可能还没有看到这项技术提供的机会。在进入本文的最后一章之前，我们应该熟悉 Web2 的底层技术和站点特性。

Web2 的一些最重要的网站功能包括：

  1. 用户作为系统中的一级实体，具有突出的个人资料页面，包括年龄、性别、位置、推荐或其他用户对用户的评论等特征

  2. 可以通过链接到其他用户的“朋友”、各种“组”的成员资格以及来自其他用户的“更新”订阅或 RSS 提要，拥有在用户之间建立联系的能力

  3. 能够以多种形式发布内容:照片、视频、博客、评论、对其他用户的内容进行评级、为自己或他人的内容添加标签，还能够控制隐私，并赋予用户自己创建网络内容的权力

  4. 其他更具技术性的功能包括允许第三方增强和“混搭”的公共 API，嵌入各种丰富的内容类型（例如 Flash 视频），以及通过内部电子邮件或 IM 系统与其他用户通信

  5. 跨设备通信

启动 Web 2.0 的关键底层技术

  • Ajax   代表异步 javascript 和 XML，是流行的 Web 2.0 技术中的关键可见构建块之一，异步的定义：“异步通信是任何类型的通信，其中一个人提供信息，然后在接收者接收信息并提供响应之前存在时间延迟。”

  1. Ajax 是多种技术的混合体，这些技术集成了 Web 页面表示、客户端和服务器之间的交互式数据交换、客户端脚本以及服务器响应的异步更新。 Ajax 中介位于客户端，向服务器发送请求并异步更新页面。基于开放标准的 Ajax 的一个关键组件是称为 XMLHttpRequest (XHR) 的应用程序编程接口，脚本语言使用该接口在客户端和 Web 服务器之间交换数据。许多流行的动态 Web 应用程序，例如地图，都使用 XHR。

  2. Ajax 的主要目的是让脚本充当 HTTP（或 HTTPS）客户端，并使用各种常见的 HTTP 方法（当前支持 GET、HEAD、POST、PUT、DELETE 和 OPTIONS）从 Web 服务器发送/接收数据。因此，Ajax 可用于网页的动态布局和重新格式化，通过仅发送一小部分请求来减少所需的重新加载量，并按需与服务器交互。来自服务器的响应由浏览器异步处理。

  • Flash  (由Macromedia创建，现在属于Adobe) Flash对象可以提供类似的功能，一旦下载，他们可以与服务器异步通信。因此，例如，YouTube视频可以在整部电影被接收到之前就开始播放。用户下载一个紧凑的flash对象，该对象下载视频的一个小前缀并开始播放，同时异步获取视频的其余部分。目前，Flash主要用于呈现丰富的嵌入式对象，如视频、音频和游戏。

  1. 曾经有一段时间，Flash 在技术方面处于领先地位。但随着时间的推移，越来越多的网站和移动应用程序不再使用 Flash。制作电影、音频和其他交互式媒体的更好方法是使用 jQuery 或 HTML5。

  2. Flash 于 2017 年弃用，并于 2020 年完全停产。

到目前为止，我们已经描述了一个纯粹从服务器向用户提供服务的 Web。简单和静态的 Web1 变身为一个增强的、动态的、更快的版本，具有大量的社交功能的Web2.

这些服务器有一个集中的所有者，并且在大多数情况下，还有一个单点故障数据库，可以被政府没收、被盗、滥用或关闭。这意味着两件事：

没收的内容可以用来对付用户，因为社交媒体网站拥有您个人数据的副本

集中服务可以随时关闭，所以任何时候巨头都可以有理由或无理由地禁止用户

在某些国家/地区，Facebook、Google、YouTube 和许多其他 Web2 平台受到完全限制。

在某些国家，人们不允许访问互联网。

Web 3.0 概念 - 为数据消费而设计的个性化 Web，同时保护隐私

Web 发展的第三阶段应该增强我们所知道的 Web2 的一切。这个版本将邀请我们进入虚拟现实世界，让我们能够在家中舒适地参加博物馆、音乐会或异国他乡。在一个我们可以应用个性化设置和偏好来塑造我们现实的世界中，我们可以将我们的创造力超越当前的限制。但是，目前来说，Web3 的隐私和安全元素更为重要。

关于在语义 Web 中的创建……“在 Web2 中，应用程序被设计为可交互的。但是这种交互性不能由用户进一步定义——除了使用的字体、颜色主题或语言。相比之下，想象一下您自己的偏好塑造了您的网络体验。与只读的 Web 1 应用程序相比，Web 2 使我们能够拥有一种身临其境且可交互的数字媒体。它让我们拥有了长达数十年的引人入胜的内容时代，旨在为用户提供大量自由。优酷、脸书、推特、亚马逊。在互联网概念出现之前的 20 年，所有这些应用都以大多数人无法想象的方式丰富或重塑了我们的生活。现在，在 Web2 全面展开大约 20 多年后，我们正在考虑再次构建互联网的想法。 Web2 正在设计应用程序，以便开发人员可以与用户交互，并受到他们的工具和互联网边界的限制。现在我们只能想象如果 Web 3 突破这些想象的边界并为 Web3 提供一种为每个用户单独塑造互联网体验的方式会发生什么。”- Unikum (Sovryn）

关于主权和自由……来自区块链的 Web 技术为用户提供了强有力的、可验证的保证，包括他们收到的信息、他们放弃了什么信息、他们为什么支付以及他们收到什么回报。 Web3 运动旨在创建一个去中心化的网络并使所有区块链能够进行通信。它是一组包容性协议，为应用程序开发人员提供构建块。这些构建块提供了一种创建应用程序的全新方式。将 Web 3.0 视为一份可执行的大宪章——“个人自由对抗暴君专横权威的基础”。

Gavin Wood, 2018 (Ethereum, Polkadot, Kusama)

虽然 Web2 被描述为读写 Web，但 Web3 可以被描述为读写信任 Web。信任是通过在核心层建立可验证性来实现的。这是通过将区块链添加到 Web2 中来实现的，在 Web2 中，区块链带来了新的可能性，就像比特币和加密货币增强了金融世界一样。

代理的使用将是另一个发展重点

用户的区块链地址（十六进制代码）充当唯一别名，因为您可以跟踪它们，但您不能立即将它们与某个人匹配。

Web3 是一组相关技术，旨在使 Web 和互联网更加分散、可验证和安全。它旨在为我们提供在互联网上自由互动和创作的能力，同时在我们的隐私甚至生命安全可能受到威胁时使用代理或 ZKP 在数字空间中保护我们。

除了 ZKP 能够使用户匿名的能力之外，ZKP 还有一个更强大的王牌。它可以处理无限量的交易，几乎没有延迟，而且费用非常低。虽然存储大量数据的可能性超出了当前的技术水平，但 ZK 已经允许匿名传输大数据块，并且无论在何处，都不会留下痕迹。

该技术首先与闪电网络一起使用，这是一个使用区块链中智能合约功能的去中心化网络，能够以去中心化的方式传输视频并接受所有可能的付款。闪电网络的主要重点是使用支付渠道来提高比特币的可扩展性并加速比特币交易（TX）。

未来越来越多的去中心化应用程序将跟随比特币的脚步自主运行，无需某方维护。仍然需要开发和维护，但这同样可以由免费的开源社区来处理，而不是由 CEO 领导的公司来处理。这种自主性是一项非常重要的属性，是任何技术都没有的，而比特币之所以具有这种特性，是因为它具有非常强大的经济激励机制。

Web3 的目标包括：

  • 使用智能合约构建无需信任的基础设施——您可以尝试在 Web 上做所有事情，而不必担心技术缺陷或欺诈会对您不利。

  1. 在选举中投票

  2. 向别人借钱或借钱给别人

  3. 参与NFT市场

  4. 匿名广播

  • 无需中介，用户可以在全球范围内自由交互。

  • 赋予用户对其数据、身份、安全和交易的权力和所有权。

  • 授予审查阻力 - 即授予用户发布任何内容的能力，确保任何权威机构都无法将其删除（除非该机构禁止整个互联网）。

Web3 运动包括：

  1. 区块链和去中心化网络（dweb）项目和链接数据工作

  2. 为安全链接数据和程序、加密可验证性、事务处理、P2P 连接和无需信任的互操作性添加功能和功能的技术

  3. 去中心化计算和存储，支持完全自主的应用程序（dapps）

Web3 的关键方面：

  • 将网页和程序直接相互链接，同时绕过中介组织，去除中间商，并获得公开可验证性

  • 将集中式应用程序转变为使用更安全但更难构建的分散式协议

  • 改进现有结构并带来新功能：

  1. 更好的移动应用程序和网络同步

  2. 多个应用程序可访问的内容，每台设备都连接到网络，服务可以在任何地方使用

  • 语义网的使用改进了网络技术的需求，通过搜索和分析基于理解单词含义而不是关键字或数字的能力来创建、共享和连接内容

  • 能够使用将平台连接在一起的平台, 即使用在平台之间进行交互或共享来自多个平台的空间平台

  • 增强现实和虚拟现实

  • 专为在其中花费大量时间而设计的数字 3D 空间；博物馆、音乐会、电子游戏

  • 机器学习、自动化和人工智能；计算机可以像人类一样区分信息，以提供更快、更相关的结果，并变得更智能以满足用户的要求

  • DeFi 和点对点交互

Web3 作为区块链增强的 Web

另一种理解 Web3 的方式是，Web3作为 Web2 的增强版本，具有额外的区块链功能，例如 NFT、DeFi，以及目前大肆宣传的流行语，它自 1990 年之后就一直存在——元宇宙。在 Web3 中，MetaMask 等区块链插件将成为一个标准工具，可以作为通往上述领域的门户。人们将能够：

  1. 交易他们的物品，其所有权与他们持有的 NFT 代币绑定。

  2. 接受 DeFi 作为其银行系统的另一种选择。

  3. 去中心化金融 (DeFi) 等应用程序将在 Web3 中受益于流动性的增加以及构建跨社区相互交互的服务网络的能力，从而增加其用户群并扩大可用资源。

  4. 使用 WormHole 之类的区块链桥接器（或其他由于人为因素而没有被黑客入侵的桥接平台，例如 RelayChain - WormHole 黑客攻击发生在写这句话的中间；部分问题源于 Solana 智能合约代码) 将他们的资产转移到区块链上。

  5. 随着区块链技术的成熟，一些项目正在通过在网络之间建立“桥梁”来解决这个问题。转向区块链和系统可互操作的世界将允许应用程序建立在彼此的服务和优势之上。随着新的、分散的和可互操作的互联网开始形成，这可能会对广泛的服务产生重大影响。

  6. 参与将结合多个游戏甚至整个游戏平台（如 PlayStation 和 Xbox）的虚拟现实世界。

游戏、平台、图形界面；当一项技术本身存在于 Web3 中时，这一切都将不再是一个界限，它能够将你的头像从一个游戏中转换出来，让它成为另一个游戏的一部分，你甚至可以花另一个游戏的钱。

Web3 和个性化偏好

从可搜索性以及从用户的角度来看搜索结果返回给用户的方式，Web3 与 Web2 并没有太大区别。

在 Web2 中，用户搜索答案并获得其他用户搜索最多的结果。在 Web 2 的当前迭代中，结果也将根据您的个人偏好进行过滤和排序，或根据从跟踪您的在线状态和行为模式中获得的信息进行调整。您可以在车内使用语音直接搜索网络，找到您附近的一家日本餐厅，该餐厅目前正在营业，用户评分最高，并且与您之前搜索的偏好相匹配。网络搜索将根据您的智能手机提供的位置找到最近的餐厅，并根据您在个人资料中提供的信息或您的订单历史订购搜索结果。然后，Web 将询问您是否要查看菜单预览并为您阅读。

然而，这个 Web2 进程与 Web3 中的进程之间的主要区别在于匿名性。这意味着，例如，您将能够在不让任何人知道您的身份、位置或您的个人数据集中的任何其他信息的情况下执行搜索。

现在Web3如果要与 Google 竞争，虽然google他们的搜索超出了 Web3 的能力。但是使用 Web3，用户可以通过 Google 搜索某些内容，而 Google 和其他任何人都无法判断搜索的来源。鉴于谷歌的审查历史，这在今天尤其有价值——隐藏或改变它认为令人讨厌的网页的“真实”排名，无论是在“错误信息”或“危险”的幌子下……还是仅仅因为某些页面对 Google 或其合作伙伴构成某种竞争。

因此，如您所见，Web 3.0 应该解决的挑战是互联网上的隐私问题。

为什么区块链将当前的数据君主制转变为数据民主制

TLDR - 因为它保证匿名。

正如我们已经提到的，只要我们使用网络服务，我们就会失去对数据的控制和所有权。这是因为发送和接收数据都会在两台机器上创建一个本地副本。这不仅是一个隐私问题。这也是商品和服务供应链中电子商务运营后端的一个大问题，因为该后端流程中的文档处理和数据管理繁琐、成本高昂且效率低下。当我们通过 Internet 发送数据时，我们无法控制数据发生的情况，因为我们无法看到其他人拥有的服务器的数据。

区块链引入了一种全新的方式来保存和处理数据，同时建立在消除中间人的点对点 (P2P) 技术的理念之上。 P2P 技术并不新鲜，但区块链正在将它们提升到一个新的水平。使用区块链，节点网络中的所有计算机都具有相同级别的信息，所有数据对网络中的每台计算机都是透明的，并且通过密码学保证隐私。通过这种方式，网络上的用户可以从数据透明度中受益，同时维护所有相关参与者的隐私。

区块链通过零知识（ZK）技术的新进展使这成为可能。

首字母缩略词 zk-SNARK 代表“零知识简洁非交互式知识论证”，指的是一种证明结构，可以证明拥有某些信息，例如一个秘密密钥，不透露该信息，并且证明者和验证者之间没有任何交互。

“零知识证明”（ZKP）允许一方（证明者）向另一方（验证者）证明一个陈述是真实的，而不会透露任何超出陈述本身有效性的信息。例如，给定一个随机数的散列，证明者可以说服验证者确实存在具有该散列值的数字，而无需透露它是什么

  2. 验证具有有效投票权的人在无法访问选民偏好的情况下进行了投票。

  3. 确保投票与任何其他投票者具有相同的价值和权重。

在一些国家，这仍然是不可能的，但它可以通过 ZKP 的预期 Web3 实现来改变。

Web3 中数据民主的核心概念：

  1. 不可审查的匿名和 P2P 商务

  2. 不可审查和匿名的社交媒体

  3. 治理系统：“让所有用户能够参与，并且将任何项目的预算分配给我们想要的任何人。”
