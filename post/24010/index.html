6月9日，由金色财经主办的第63期线上版：“Web3.0-从信息交互到万物互联还要走多久？”顺利举办，本期金色沙龙邀请

Phala Network创始人 CEO  Marvin Tong 、Conflux研究总监 杨光、科银中国COO  刘耕华、Bandot CMO Cora、CoFix ETH Zurich Research Fellow / NEST Chief Adviso  Yuyi5位嘉宾，本期沙龙设置圆桌1个环节。

圆桌问题一：从各位的角度，是怎样理解web3.0的结构的，可能包含组织架构或技术架构？和以太坊、波卡等网络构建的生态结构有什么异同？

Marvin Tong：这个是web3基金会对web3.0栈的一个设想，当然不同的机构有不同的图/PPT，实际上大家也大同小异。最底层一般叫作layer0，指的是网络基础设施类的模块，比如meta协议，通信协议，语言等等。波卡认为自己是这一层的，即管理区块链的区块链，L1是指0信任或低信任的协议，比如比特币、以太坊、波卡平行链，其特点是每条链有自己适用的场景，链直接面向业务（如金融、dapp）。另外就是数据存储协议和消息传递、数据转移协议，到这一层还是以协议为主。L2一般包括了状态通道、分层分片系统、加密存储、计算、密钥管理和预言机，虽然还是协议，但是已经过渡到了中间件状态，对应的比如以太坊L2项目、chainlink、phala的TEE worker、filecoin的矿工，其实在L2。L3基本上就是基于L2抽象出来的“中台”产品，链接协议层和应用端的插件，也是中间件，一般来说像去中心化的数据集api，wasm工具脚手架，加密前端框架，属于这一层，最后：L4就是应用端了，比如常见的defi、dapp和brave这种产品化的对象，是面向用户的服务。总结一下，纯从架构上说，波卡生态目前是星状结构的，两个恒星是波卡和kusama，每个恒星都有一些行星，行星还有卫星，而以太坊目前更像是恒星-星云的雾状结构的，更加去中心化，但是二层的格局现在还不是很明朗。

杨光：技术方面的内容Marvin已经讲得非常详尽，我对 Web 3.0 研究不多，就不再重复了。简单说一下我对 Web 3.0 组织架构的理解吧！Web 3.0 应该是一个跨区块链的生态。就像互联网是联通了很多小的局域网、城域网形成的广域网络，Web 3.0 应该也是将各种区块链作为底层基础设施，实现一种范围更广的架构，如同我们现在访问一个网站的时候不会很在意网站的服务器在哪个城市、哪个国家。Web 3.0 时代用户访问内容的时候也不需要特别在乎呈现出来的信息到底是存在哪条链或者哪个网络上的。Web 3.0 的前端会把所有链上的合约和信息聚合以后呈现给用户，要说以太坊和波卡的话，他们生态结构还是相对封闭的，而且他们本身的层级还不够高，有很大的局限性。就像对于互联网来说，公司也好用户也好，其实不会特别在意用服务器、数据库、编程语言的各种门户之争，能把活干好才是最重要的！Web 3.0 的生态也一定是横跨各个区块链社区的开放生态，而不是哪一家就能构建出来的，互联网已经是开放的生态了，Web 3.0 总不能越发展越倒退吧！

Cora：目前很多人会将 Web 3.0 当成是基于区块链技术的去中心化 Web 应用，但这样的定义太过于狭隘，事实上 Web 3.0 的概念版图极其庞大，它是一种广义性的架构或者未来互联网的底层协议。这点美国加州两位科学给出了一个全新的概念：Spatial Web空间互联网，Web 2.0 后期我们几乎陷入到了仅限于数据、信息的单维度网络，而 Spatial Web 更像是集合了价值网络、信息数据网络、物理显示网络多维度“空间”的网络。分布式计算、去中心化数据、人工智能，虚拟现实、物联网以及基于外界环境的持续边缘计算等技术的每一次进步都会进一步扩展我们周围的空间，我们可以从三个层次（交互层、逻辑层、数据层），四项主要技术空间（增强现实、虚拟现实和混合现实）、物理（物联网、可穿戴设备和机器人）、认知（机器学习、人工智能）和分布式计算技术（区块链、边缘计算）去定义这个概念。

Yuyi：正如Cora 说的。这是一个很宽泛的问题，首先“什么是web3.0”是分歧很大的，有人认为“万物互联”最重要，有人认为“语义网络”这一点更重要。当然今天我们在这里讨论的，应该更关注从中心化走向去中心化。即便如此，目前来讲对 Web 3.0 也没有更确切的解读，我们只能从过往 1.0、当前2.0阶段，以及哪些突破2.0规范的地方进行总结和构思，组织架构上，我认为与2.0差异并不大，只是在开放程度上会更加宽泛，这需要一个长期的实践和发展来总结具体的范式标准。当前可预见的有所突破的方向是开源文化的共识增强和DAO组织形式的发展，技术架构包含很多，除了2.0已有的技术以外，更多的将价值转移以及公平分发、隐私等涵盖进去。涉及到存储、开放的数据库、普通个人价值的体现等等。若以当前区块链作为基础设施构建3.0的话，主要体现在金融交互协议的技术构建上，以太坊和波卡以及其他公链在交互上并无大的区别，更多的体现在细节和安全性上。相同点是都涵盖自己的开源和开放社区，可自由进行交互，大的区别在于去中心化的程度以及分布式系统的开发上。

圆桌问题二：如果将区块链网络作为Web3.0基础设施，需要搭建哪些基础协议？哪些应用协议？请各位从自身项目或理解角度出发。

Yuyi：以当前区块链主要体现在金融交互上去看，我认为的基础协议有：开放的数据库、DAO 治理协议、数据分析工具、更加完善的底层公链甚至是多链结合的更宽泛的底层等等，前提需要更加去中心化来保证底层的安全性。应用协议上比如确保金融交互的底层价格协议–预言机、构建在预言机之上的各类金融产品等，结合我所服务的项目来说，NEST是个预言机，NEST生态正在打造的Cofix, Parasset和Fort等等就是构建在预言机之上的金融产品。简单来说，Cofix用于交易；Parasset是基于预言机的价值再造，打造抵押代币的终极解决方案；Fort提供了基于信息流的交易模型。

Cora：DeFi 是 Web 3.0 的价值层协议。DeFi 相当于 Http，它本身具有一定的应用价值，但主要是个基础设施，是用来支持上层应用的协议。只有 DeFi 这一步夯实了之后，上面才会出现大量的去中心化的抖音、推特、亚马逊和淘宝等应用，像 Uniswap、Compound 和 Maker 这类协议，可以称之为 Web3 DeFi 协议。Web3 DeFi 协议能够在应对攻击的长期争斗中会存活下来、获得反脆弱性，使得他们能够突破 Web2 应用程序的规模限制。我们Bandot认为DeFi的未来是一片蓝海，还有很多版块未被开发出来，Bandot做的是无抵押借贷协议，建立无抵押借贷仅仅是一个开始，Bandot无抵押借贷协议的愿景是建立一个去中心化的信贷平台。下面是无抵押借贷协议的三阶段计划：信用担保借贷–信用保险池借贷系统–去中心无抵押借贷平台。

杨光：跨链协议是必备的，信息和资产需要在不同的区块链网络之间流动。其实跨链分很多个层次，信息跨链、资产跨链、智能合约的跨链互操作等等，难度是依次递增的。与之配套的还有各种数据格式、接口、通讯协议等等，甚至是统一的智能合约语言和编程范式。关于最后一点可以再多说一些：好的编程范式可以让智能合约更容易被形式化验证，从而更安全；也可以让合约在执行时候更容易实现并行化，优化性能和成本。这也是我们现在正在进行的研究。

圆桌问题三：除了底层技术，数字货币是否也将在万物互联的体系中发挥重要的作用？

刘耕华：“区块链+物联网”的结合的一个基础要求是，通过模组或更复杂的芯片技术对物联网设备引入唯一ID，即数字身份（公私钥）在硬件底层实现不可篡改。物联网设备的ID广泛应用在物联网数据记录、上云、上链以及数字货币交易等方面。这样，物联网中的数据存储、传输和挖掘以及价值交互就能以可信方式进行。

圆桌问题四：如果将区块链网络作为Web3.0基础设施，性能上能否满足需求？例如layer1的TPS，layer2或链下（本地）的处理能力，加密，安全性等等。

杨光：首先，Web 3.0 自己无所谓需求不需求的，就像拨号上网时代的互联网也是互联网，不需要非得 4G，5G。对性能有需求的是各种应用。以现在的区块链技术，性能肯定是无法满足所有应用场景需求的，只能满足很小的一部分。无非是性能好的可以满足需求的场景多一些，性能差的满足的就少一些。Layer2 的处理能力，很多时候水分比较大，看着指标很高，实际上不解决问题打个比方吧，2020 年全国人均住房面积 40.5 平米，但是一二线城市的住房状况大家都懂。多的那些面积，比如鹤岗一套房子可能不到 10 万，还不好卖。最后大家还是都往大城市挤，买不起核心区的房子也要买个进城方便的，每天早高峰时候的那个体验。如果指望靠 layer2 解决性能问题，先想想早高峰进城的地铁。所以说，在 layer1 提升 TPS 是一个长期的需求，这也是我们 Conflux 一直以来努力的方向。我觉得 Layer2 和链下处理技术最大的作用不在于增加整个系统的 TPS，而是在于通过缩小共识的范围，降低成本，提高效率。在保密性、安全性方面也有优势。对于很多场景，其实也没必要把所有逻辑都放到公链上做，用 layer2 就挺合适的。

Marvin Tong：我同意杨光老师的说法，区块链作为“低信任结算层”是可以的，但是作为计算处理层差得比较远。都2021年了，我们先不说TPS，实际上多层任务分层处理已经成为共识了。不管是以太坊还是波卡，分层都是共识。举个最简单的例子，前几天美国输油管道的黑客好不容易骗到了BTC，还被FBI抓起来。为什么？因为他用的是中心化云，这也太扯了。这能侧面说明至少在云服务领域，web3.0还是概念和故事层面。所以我们也在反思自己，我们要提供链的服务还是web3云的服务？目前来看，Phala这样的奇葩项目，更适合提供去中心化的云服务，来补上这一环。至少别在让咱们几千个上万个区块链项目的应用部署在阿里云、GCP、AWS了，给行业挽尊一下。web3堆栈要补齐的东西太多了，其实根本不用内卷，大家各自认领擅长的部分去干就好了！

圆桌问题五：在Web3.0的世界里，哪些场景需求是强烈的？需要尽快实现的？如何实现？

Cora：DeFi的需求长期以来都是非常强烈的，同时也会持续发展下去，DeFi涵盖范围非常广，抵押与无抵押借贷以及各类金融衍生品等都是在2021年需要尽快实现的几个方向。同时，应用型的需求也是非常强烈，我觉得需要尽快实现的还有应用型项目。现在以太坊Gas费那么高，完全没有散户的交易量了，大家已经参与不到DeFi的项目中去了，目前有两个解决方案。第一个方案是以太坊的Layer2，第二个方案是波卡。我深度体验过Layer2，总结下来，他主要有两个优点：它实现了中心化交易所的体验，同时又保证了去中心化交易所的自由。这是让以太坊在网络上的应用可以迅速提高使用场景的解决方案。第二个是波卡。我认为波卡是把Layer2做的更加极致化的分片网络，它能让平行链之间，让应用层之间更具组合性。Layer2最大的问题是可组合性不足，未来波卡可以很好解决这个问题。

Yuyi：在金融科技的应用中，当前最繁荣的生态无疑是以太坊生态，在保证高度去中心化的同时，也在逐步完善低的交易费用以及向世界级的流动性金融交互上突破。对于超大额资金流动性的交互更注重的一定是安全性、自由开放，而不是交易速度，那么要完善世界级的交易场景，最重要的就是价格数据的安全和稳定。当前我们看到价格数据的调用的增多也在逐渐凸显大规模交易对价格的需求，这类预言机或者以非合作博弈来构建的底层价格协议需要尽快突出重围。当前 NEST Protocol 在这以太坊上开发和预言机方面已经深入研究了近三年，并且已经构建了基于底层价格协议的应用产品。主要实现方式是，双向期权报价，引入验证者机制，拿真金白银进行谢林点价格博弈，这种方式的好处是足够去中心化和安全。能承载的市场体量不会受合作博弈那种节点的方式，也就是相对中心化因素影响会比较弱，缺点就是价格数据产出速度会慢，不过这一点会随着市场需求和价格博弈的共识增强而改善。

圆桌问题六：在万物互联的时代，加密市场的生态结构与市场格局是否会发生改变？

刘耕华：会有更多的实体行业，通过物联网形式，进入区块链世界。相较于当前物联网的客户端-服务器模型，去中心化的区块链技术的好处要多的多。区块链技术的去中心化特性能够很好的保护用户的身份及安全，目前，物联网设备是通过云服务器来识别用户，他们将识别数据保存在这些云服务器中。这样一来，这些数据就很容易被泄露、窃取或抄袭，基于这些数据的应用程序也会受到很大的安全威胁，而去中心化的区块链则可利用非对称密码算法和安全哈希算法来保护这些用户身份。设备可以根据对应的区块链地址进行注册，保证身份的唯一不可篡改性。因此，这是现有中心化基础设施无法达到的安全和防篡改识别高度。区块链也正在增加其可扩展性，比如比特币现金 ABC 增加区块大小，以太坊通过分片技术扩展链的容量。同时，随着闪电网络、以太坊 Palsma 和 Blockstream 的 Liquid 侧链出现，可扩展性已经得到了稳定的发展。

圆桌问题七：对于用户来说，身处Web2.0或3.0,或者感受已经进展到了哪个阶段，几乎是“没有概念”或者“无感”的，所以请各位聊一聊如何感知Web3.0？或者如何感知Web2.X？

杨光：可以感受到同时部署在多条链上的项目越来越多，但是并没有一个明显的里程碑说现在是“Web3.0”或者“Web2.5”什么的。可能比较有直观感觉的还是负责跟各种项目对接的小伙伴，越来越多项目不再局限于某个链的社区了。

Cora：DeFi逐渐成为全球性的公共物品，目前，区块链比特币等关键词成为了全球热门搜索关键词，我们也惊喜的发现使用DeFi协议的用户越来越多。同时我们发现类似Bandot这样的支持不同平行链之间的价值互操作性的，以及其他DeFi协议层项目，逐渐显示出其“非竞争性”、“非排他性”、“不可分割性”、“逐渐下沉”的性质和趋势。DeFi项目在使用体验方面会更接近一种经济学意义上的公共物品，为更多的用户提供融资理财便利，同时随着波卡的发展与以太坊的日趋完善。协议层项目会逐渐形成一个协议池，使得系统内部所有混乱、模糊的个体单元收敛为一个可靠、可预测结果的单一有序模式，从而bring order to the chaos。
