原文标题：《 章鱼网络刘毅：Web3.0 是 Web2.0 的延续，并非要去平台化丨 2021 世界区块链大会 》

在 25 日下午的「Web3.0 论坛」上，章鱼网络创始人刘毅出席并发表主题演讲《开发应用链，实现 Web3.0》。刘毅在演讲中讲到，Web3.0 是 Web2.0 的延续，Web2.0 的经济特征是平台经济，Web 3.0 并非要去平台化，能让互联网变得更开放、更公平且更安全。

Web3.0 并非脱离互联网，Web3.0 还是 Web 应用。Web3.0 的时代要来临，必须要有一批好的 Web3.0 应用在各个市场去取代现有的 Web2.0 或传统中介。

以下是整理部分演讲内容：

Web3.0 是 Web2.0 的延续，Web2.0 的经济特征是平台经济。「平台经济」一词的定义就是平台是依靠促进或撮合不同类型的参与者之间的交易或互动来创造经济价值的业务类型，简言之就是中介。

Web2.0 的所有巨头，无论是中国老的 BAT，还是现在的这些美团、滴滴的新巨头，包括头条等，还是美国的五个互联网巨头公司，他们都是平台。

B2C 电商，是电商的商家与电商买家之间的平台；网约车的平台，是司机与乘客之间的一个中介；外卖平台，是餐馆及外卖消费者之间的一个中介。

为什么平台这么重要，成为一个经济现象，或是有人说平台在吞噬世界？因为这个平台有一个魔法——网络效应。网络效应是指随着平台的参与者越来越多，提供的服务会越来越好的现象，最典型的是微信，微信是发信息的人和看信息的人之间的中介。

当平台规模大到一定程度时，就像一个「黑洞」一样，将市场两端的参与者都吸了进去，而且粘在这个平台上，我们无法离开它，这是 Web2.0 经历的特征。

为什么要搞 Web3.0？是因为 Web2.0 的平台虽然将我们的生活变得更好，更有效率了，但它的弊端也正在显现，最核心的弊端是平台都是由某一个公司来运营的，公司经营的天职是创造利润，创造尽量大的股东回报，并不是要给我们的这些顾客，其运营职能是为股东利益服务。

平台规模大到一定程度时，当服务提供与服务使用者都离不开它时，它就要扩大自己在经营交易中的分成，我们在经济学上称为「寻租」，寻租不是一个好词，你在经济交易当中的贡献未变，但因你的地位特殊，所以你可以去寻求更大的一块利润，这就是寻租。

其实，现在的互联网巨头已经处在这么一个阶段了。

现在大家也看到了平台监管的问题，政府已经意识到平台经济发展到了一个临界点，它对这个社会效率的提升，包括科技的创新，和它对贫富分化所谓的内卷贡献，和竞争的意志已经到了一个临界点，其负面贡献已经超过其正面贡献，所以要对平台进行抑制。

我们要换一种经济形态，不是要将平台给去掉，因为经济中介是去不掉的，经济中介是因为交易双方有巨大的信息不对称，我们要将中介形态由公司拥有的平台变成社区拥有的加密协议，社区拥有的加密协议就是我们所说的去中心化，这个加密协议不是听命于某一个人，或是一小撮人，而是为这个社区所拥有。

什么是社区？如果有一个打车的加密协议，其社区主角是网约车司机，有一些辅助角色，开发者、早期投资者以及网约车用户，这些网约车用户推广这个协议，也变成这个社区的活跃成员。一个外卖社区核心角色就是这些餐馆，电商的加密协议的主角/控制者就应该是这些电商卖家。

社区拥有的加密协议去取代公司拥有的平台就会进入 Web3.0 阶段，因为它能让互联网变得更开放、更公平且更安全。为什么更开放？它没有准入了，你要做网约车不需要滴滴去审核，只要你符合社区规范。

更公平是指协议本身并不追求自身利益的最大化，他要将利润让渡给服务的提供者，并将价格降下来，让服务变得更具竞争力，这就是我们说的 Web3.0。

更安全，因为我们都是用匿名的 ID、地址，DID，数据都处于经过加密的区块链上，这就是我们说的通过这样的一个革命，将互联网进入到一个新的阶段。Web3.0 的时代要来临，必须要有一批好的 Web3.0 应用在各个市场去取代现有的 Web2.0 或传统中介。

我们要有优秀的 Web3.0 应用，首先它是一个优秀的 Web 应用，这是很多创业团队忽略的。如果不懂互联网，区块链技术能否做 Web3.0 创业？其实不能，我们不能将根给忘了，因为 Web3.0 还是 Web 应用。

同时我们要建立一个去中心化的权益与权力结构，这个协议怎么分配收益，怎么分配权利，需要在一个新的去中心化账本的新形态的平台上实现，同时保证它具有去中心化的属性，不会被个别人所控制，这就是区块链要解决的问题，它应该具有高安全、低费用、高性能、钱包易用、可升级，具有正统性的升级，在具有社区达成共识之后进行升级。不要只说扩容 TPS 高等，这些东西都做好才能形成一个去中心化的权益架构去支撑一个社区拥有的加密协议。

我们说到了应用链，应用链是另外一种，或是新的一种实现 Web3.0 应用的方式，原来的 Web3.0 应用都是通过智能合约运行在一个公链上的，那它的问题是它只能在整个区块链技术栈里，只有应用逻辑是能修改的，其他的这些共识协议、执行环境以及钱包交互、治理都是跟着大公链一起走的，这就造成了用户理念下降太多，无法与 Web2.0 竞争，以太坊从 1.0，2015 年上线到现在六年时间了，没有什么应用能争取到主流的互联网用户。

去年夏天 DeFi 开始爆发，到现在 DeFi 日活才一万的级别，比主流的互联网应用，比如微信，低了 5、6 数量级，这是一个特别好的东西。

绕是如此，这已是很大的一个成就，主要是智能合约的用户体验太差了，一个新用户用智能合约要安装 MetaMask，买 ETH 充到自己的钱包中，还得知道什么叫 Gas 才能参与去使用这些 Dapp，绝大部分的用户都被挡在外面了。

应用链是用区块链的框架为应用做一条专用的链，这给了你很多好处，一个是性能极快，因为你是独享一条链的处理能力。另外是共识成本非常低，用户用起来甚至可以不花钱。

可定制性强，所有的参数都可以调整，你可以修改执行环境，修改钱包，修改共识环境，去为这个应用所优化。

最近一个很好的例子就是大火的游戏，Axie Infinity，它从 4 月上线了自己的定制以太坊策略之后迎来了爆发，因为它将交易成本降低了五个数量级，速度飞快，且在钱包中集成了法币支付，使得整个协议迎来了一个爆发，这就是应用链的优势。另外应用链可以有链上治理，你可以自己去社区内达成共识，升级协议，否则他快速地进化适应新需求，或是排除软件缺陷等。

应用链也有不好的地方，虽然能力非常强，但需要更高的投入与知识，好在现在已经有工具将开发应用链的成本降低到一个与智能合约相当的水平。

开发一条应用链更方便，但运行起一条应用链是耗时很长，且需要巨大投入的事。因为一条链不仅要开发，还需要有一个链的安全性，需要有跨链能力，需要有除了链以外，还要有区块链浏览器等系列的基础设施，而且要有社区关注这条应用链，你获得早期用户以及获得早期投资者的支持。章鱼网络就是为了部署运行启动应用链的服务网络，我们是建立在 NEAR  Protocol 上的二层网络，Substrate 网络通过加入章鱼网络能获得与 NEAR、以太坊之间的无信任跨链，我们已经做了一套完整的基础设施给应用链使用，而且我们这个社区就是积极支持服务于应用链的社区。

Octopus 上线之后形成的结构，我们早在六年之前 Cosmos 设想的区块链互联网的一部分，章鱼网络的应用链除了可以与 NEAR 跨链进行对接之外，通过彩虹桥与以太坊跨链，可以通过自主决定，通过 IBC 协议跟所有 Cosmos 链互操作，另外我们也可以与所有的 Polkadot 平行链形成互操作，形成这个多链网络。

我们的经济模型是应用链，除开发以外，其他的基础设施都由我们包，应用链的 Token 将 QOS 这部分功能让给我们，章鱼网络的 Token 是唯一用来做应用链安全租用的 Token。

假设一个链在我们这里边付 50 万美元的安全租金，它大概能得到 1000 万美元级别的安全性，同时每一条新链能创造出 1000 万美元对抵押品 OCT 币的需求。

社区工作除了要将 WEB 开发者以及智能合约开发者转化成 Substrate 开发者，当然我们是辅助 Parity 在做。

同时我们通过章鱼加速器去支持这些应用链团队，现在已经有了很多生态项目，我们也得到了一些资本的支持，包括 NEAR 的战略投资。
