为什么OpenSea和Magic Eden的用户体验基本相同（首页是发现页，藏品页左侧有简明线性索引条，相同的搜索展示等等）？

或者，换句话说，为什么OpenSea或Magic Eden的首页没有活动提要（如Fractal或Hawku）、直播入口（如Twitch），或NFT相关的赛事结果等？

答案是，历史路径决定的。

OpenSea成立于2018年，当时只有少数NFT系列，如CryptoPunks。CryptoPunks是一个由10,000个像素化的NFT的集合，具有一些简单且可筛选的属性。Bored Ape Yacht Club（BAYC）显然深受CryptoPunks的启发：10,000张算法生成的图像，具有简单且可过滤的属性。出于实际目的，这两个收藏品是相似的。随后涌现的大多数PFP系列——Pudgy Penguins、Moonbirds、DeGods、Solana MonkeyBusiness（SMB）、Azukis等，都映射到相同的结构模型和历史范式。除了艺术表现形式本身，收藏品本身的设计并没有显著的差异（尽管显然他们有着非常不同的社区）。

显然，社区正在映射到市场的技术规范，但这种方式并不理想。市场应该映射到社区的独特特征。考虑到在DeFi、NFT、DAO、社交代币等方面利用和配置新元素的能力，在线社区的设计空间是巨大的。

人们普遍认为，NFT市场将进一步垂直化。这自然是有道理的，因为让一个域名交易所（如.eth 或.sol）与一个 PFP 交易所（如 BAYC 或 DeGods）具有相同的用户体验实际上是很奇怪的。市场应该围绕其托管的内容类型进行优化。

但这种思路并不应该是简单的一刀切。这种推理可以走得更远。NFT市场不仅要垂直化。至少，他们将：

  • 归他们所服务的社区所有

  • 集成更多非商业功能（例如直播、聊天、论坛等）。

社区主导的NFT交易平台

Magic Eden和OpenSea都非常有钱，这些钱是从哪里来的？

实际上就是NFT社区本身，因为平台会对每笔交易收取2-2.5%的交易费。

有人可能说交易费用终将归零，他们不会的。相反，我会争辩说，交易费用不应该转到与NFT社区分开的第三方交易所，而是应该回到社区本身。这种

转变是社区成熟的标志。当社区及其品牌足够强大可以独立运营时，它将抓住机会，通过为社区定制的功能和参与度来进一步稳固其财政并提高品牌忠诚度。

BAYC已经在这样做了：每笔交易的2.5%都归Yuga Labs所有（除了OpenSea收取的2.5%的费用外）。没有理由相信BAYC持有人特别忠于OpenSea。相反，如果Yuga Labs启动一个BAYC交易所，100%的费用会流回ApeCoin控制的社区财库，BAYC和MAYC的大部分交易量将很快转移到那里，只是为了提高品牌忠诚度和扩大社区财务的愿望。

问题在于，第三方NFT交易所寄生于NFT社区。它们并没有真正为社区提供价值（当然围绕 Launchpad 进行价格发现是有价值的，但这只在一个时间点上提供价值，而不具备持续性），并且它们会明目张胆地提取价值。存活下来的市场必须提供真正的价值——无论是让玩家挖掘洞察力、分析和数据，为潜在玩家提供发现新游戏的渠道，还是创建一个志同道合的玩家可以加入的社区。NFT市场必须提供的不仅仅是一个金融中介。

如果我的理论是正确的，为什么还没有发生这种情况？我认为这是因为托管NFT交易所有大量有意义的工作，而这项工作尚未完全商品化。

Metaplex和Grape Network以及Reservoir等正在构建所有必要的功能，以完全商品化通用交易所（例如索引、查询、搜索、策展等）。

例如，Grape Network 刚刚推出了第一个 DAO 拥有的 NFT 交易所。虽然目前还处在非常早期的阶段，但它正在快速成长。

一旦这种基础设施普遍可用，社区可以轻松托管自己的交易所，可能会发生一些有趣的事情：

  • 虚假和欺诈性的PFP数量将接近于零。这个很容易理解，如果有了 BAYC 官方的交易平台，那么这个平台显然只会呈现真实的 BAYC NFT 作品。这种情况下，用户只需要确定自己访问的域名没问题，那么他就几乎不可能购买到假冒的 NFT。

  • 社区将会在其聚集的任何应用中嵌入交易功能。目前，这方面主要指社交类应用：Discord、Twitter、Reddit 等。Web2 社交应用理论上可以更好地集成 NFT 并为 NFT 社区提供支持，但它们并没能做到这一点。我认为，这会为这类应用的加密原生版的蓬勃发展创造机会，比如 Satellite（加密原生版 Discord）和 Farcaster  和 Orbis（加密原生版 Twitter）。用户可在这些社交应用内出价/还价。

NFT交易所需要成为API优先的产品。将有一个单一的共享流动性池，可以在许多前端访问。它们也将是社区优先的产品。

建立基于交易的社区活动空间

人们喜欢在各种各样的社区线上空间中闲逛。如：

  1. 堡垒之夜的大厅

  2. Discord 的服务器

  3. Reddit 论坛的子版块（Subreddit）

  4. Twitter

  5. Telegram

不过这些体验是不连贯的。人们在一个频道上交流，往往只是为了在另一个频道上重新组织起来。这些社区实例之间几乎没有共享状态。

有两种方式可以把这些空间有效地串联在一起：游戏和 NFT 市场。人们会在游戏之外谈论游戏，然后进入游戏玩。同样，他们会在社交渠道上了解 NFT 社区正在做什么，然后去 NFT 市场完成交易。

社区不太可能聚集在一个地方。至少，总会有不针对社区的Twitter式的的全球信息流。因此，除了类似Reddit和Discord的产品外，社区还希望将其市场嵌入Twitter类产品中。因此，至关重要的是，为这些社区提供支持的NFT交易平台是优先开放 API 的。这类交易平台应该是一个旨在运行在其他用户界面中，从聊天到直播再到专用的二维HTML。

鉴于社区自然会通过各种社交渠道聚集，这对主要网站（例如bayc.com）意味着什么？有几件事：bayc.com成为社区活动空间的聚合器，以及该社区市场的主用户界面。

可以想象，当用户访问bayc.com时，应该会看到：

  • BAYC品牌新电影的预告片

  • BAYC元宇宙中著名社区成员的直播（包括社区本币的流媒体小费）

  • 近期社区活动的亮点，现实世界BAYC涂鸦等

在用户浏览时，商业元素应该被自然地嵌入整个网站。内容本身可能会根据用户的链上凭据和当前资产而动态调整（对新用户在首屏展示电影预告片，对 BAYC 老用户展示其最喜欢的社区成员在 BAYC 元宇宙中的实时直播）。

在当今存在的所有主要消费类Web2产品中，与我所描述最接近的产品可能是Twitch。然而，Twitch1）不够以社区为中心，2）显然没有被设计为支持上述多样性和细微差别。

社区发展得越多，产品运营就变得越复杂。这带来了自然的挑战和机遇，当在无数社区中成倍增加时，它变得更加复杂。这就是为什么NFT社区将远离通用的NFT交易市场的原因。

NFT和加密世界中的其他一切事物一样，从根本上讲是关于用户主权的。他们的交易平台也不例外。
