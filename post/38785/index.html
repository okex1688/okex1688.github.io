6 月 3 日结束的 ETH Shanghai 黑客松活动中涌现出了不少优质的 Web3.0 项目，在上百个参赛团队中，有四个脱颖而出，分别摘得金银铜奖。其中，Web3.0 邮箱应用 MetaMail 获得金奖，Arweave 生态 SQL 数据库解决方案 Kwil 获得银奖，DAO 成员信用评级工具 DAOU 和虚拟形象生成协议 VOKA AI 则获得铜奖。BlockBeats 在本文中对这四个项目的功能及应用场景进行了进一步的探索与分析。

金奖：MetaMail

电子邮件作为互联网大规模普及的关键性基础设施，直至今日依然在我们的生活中扮演着重要的角色。其最大的贡献之一，就是为用户省去了繁琐的私钥管理流程，现在，无论是交换信息还是注册账户，电子邮件都是用户步入互联网的第一扇大门。

而当前随着钱包以及 DID 应用的发展，用户进入 Web3.0 世界的门槛也正在不断降低。就像传统互联网一样，进入 Web3.0 的用户也需要一个高效的通讯应用，为 Web3.0 世界的信息流通提供「原生级」的服务。

显然，电子邮件作为底层逻辑极为相似、受众覆盖最为广泛的应用，是桥接 Web2.0 和 Web3.0 世界、服务 Web3.0 原生用户的最佳选择。而基于 Web3.0 打造的 MetaMail，兼容传统邮件协议、支持 Web3.0 用户使用域名服务创建自己的邮箱地址，并通过加密技术实现端对端内容加密，无疑是 Web3.0 通讯的一个最佳选择。

值得注意的是，在这次黑客松活动上 MetaMail 还基于 Web3.0 关系算法协议 KNN3 打造了用户推荐等功能，进一步凸显出其与传统电子邮件服务的区别与优势。

BlockBeats 在此次黑客松前对 MetaMail 和 KNN3 均有专访，详见《对话 MetaMail：Web3.0 为 Email 提供的完美方案》、《对话 KNN3：Web3 也需要可靠的算法》。

银奖：Kwil

对很多人来说，Arweave 就是一个单纯的数据存储平台，实际上 Arweave 同样能够支持智能合约，且其生态内应用场景在过去一段时间里不断蓬勃发展（BlockBeats 注，在《现在的 Arweave 生态应用，已经远不止存储那么简单了》中对 Arweave 生态有详细介绍）。在这一背景下，利用好 Arweave 自身的数据优势，为生态内的应用构建提供优质服务已经成为了一个非常不错的发展机会。

Kwil 就是一个用于构建去中心化协议和应用的 SQL 数据库解决方案，旨在构建出一个高度易用且可扩展的架构，以提升链上分散数据的可组合性。通过为 Web2.0 开发者提供熟悉且高度可扩展的去中心化 SQL（Structured Query Language）数据库，Kwil 能帮助开发者在更短的时间内完成 dApp 的开发。

在 Web3.0 的去中心化架构下，开发者需要处理大量分散、复杂的链上数据，这在很大程度上延长了应用的开发周期，以及应用本身的体验和可扩展性。而 KwilDB（Kwil 数据库）通过提供高度可扩展的去中心化数据存储和查询服务，极大简化了应用和协议的开发流程，使 dApps 能够无缝地插入其他应用程序中的数据。

通过将链上的去中心化数据纳入 SQL 数据库，Kwil 能够帮助 Web2.0 开发者更好地适应 Web3.0 开发环境，开发者不需要对当前的数据进行任何修改，就可以将应用程序从 Web2.0 移植到 Web 3.0 世界。此外，基于 KwilDB 构建的应用可以集成和利用来自其他 KwilDB 用户的数据集，开发者还可以利用其他应用的数据搭建自己的应用或协议，实现高度可组合性和可扩展性。

工作原理

我们知道，存储于 Arweave 上的数据是非结构化的，只有交易 ID。为了按时间顺序找到正确的数据，Kwil 节点通过区块索引器在 Arweave 上查找相关的数据，然后节点再利用该数据与在相同数据集上运行的数据库进行同步，Kwil 数据库中的数据找出正确的数据结构。

有意思的是，Kwil 支持应用或协议的开发者选择数据存储费用的支付方，开发者既可以承担所有相关成本，也可以以 Gas 的形式将成本转移至社区。而 Kwil 采用 Proof of Secret 机制，社区成员通过支付数据存储的费用，追溯验证数据的来源。

生态内应用

Koii

Koii Network 是一个去中心化注意力跟踪协议，基于内容流量为 Web2.0 和 Web3.0 的数据内容创作者提供激励。Koii 为开发者准备了一套功能齐全的开发工具，以构建可扩展的 dApps，KwilDB 则作为社区关系数据库囊括其中。

Space And Time

Space And Time 是一个企业级数据仓库解决方案，旨在帮助 dApps 使用可扩展的 OLAP 查询来集成跨链生态。KwilDB 是其集成的二级数据存储解决方案，任何无法纳入 Space And Time 主集群的数据都会被发送并存储在分片的 KwilDB 数据集上。

Kyve

Kyve 是针对旧区块状态的存储解决方案，旨在减轻 L1 公链中节点的存储需求。Kyve 的上传者和验证者通过协同工作，确保区块状态数据经过验证，以及以正确的方式在 Arweave 上存储和检索。KwilDB 是带激励机制的 Kyve 本地查询层，它使应用程序能够无权限地插入和检索已归档的区块链数据，并计算剩余支付以奖励社区。

Ecclesia

Ecclesia 是基于 KwilDB 构建的免费去中心化通信平台，它结合了 Slack 以及 Discord 的内容组织能力和 Twitter 的内容消费能力。

DeChat

DeChat 是基于 KwilDB 构建的去中心化抗审查社交网络，使用 Web2.0 用户熟悉的社交媒体 UI，旨在鼓励言论自由，并借助 Web3.0 的技术改进在现有的社交媒体模型。

铜奖：DAOU、VOKA AI

DAOU

在未来的 Web3.0 世界，DAO 无疑是人们工作和社交的主要场所，而当前从 Seed Club 到 Dework，针对 DAO 组织的一系列基础设施和应用工具也正不断完善，为 DAO 组织的筹建、任务分配、薪酬分发等工作提供服务。

DAOU 是一个为 DAO 成员提供信用评级的工具型应用。通过使用灵魂绑定 Token 和社交预言机（Social Oracle）两个机制，将用户在 Web3.0 社区中的活动数据转化为可验证的链上数据，包括在 Discord 参与活动、在 Github 贡献代码、在 DAO 中的角色等。

通过社交预言机机制，DAOU 激励同一 DAO 组织中的成员为其他成员生产的数据进行签名和验证，并根据这些数据给每个成员一个可靠的打分，这个分数就是 DAO 成员的信用评级。社交预言机验证的数据将通过 DAO 以灵魂绑定 Token 的形式发布，并可以对外使用，例如链上信用贷款、空投目标筛选等。（BlockBeats 注，在《V 神：「灵魂绑定」币将成为你们的区块链护照》中有关于灵魂绑定 Token 的详细介绍）

交互界面

首先进入 DAOU 网站，链接钱包并点击「Go To Dashboard」即可进入交互页面。左侧是用户所在的 DAO 组织，右侧则是相应 DAO 组织的成员，点击推特按钮可以查看这些成员的推特动态，中部则是 DAO 成员每日生产的数据。

点击进入任意一条数据集后则可以为 DAO 成员的数据进行验证。DAOU 将 DAO 成员的数据分为三类：Activities，参与社区活动或发言的次数、Connections，社区内的朋友数量、Contributions，Github 代码贡献或 Dework 完成任务数量。在这个页面我们还能看到已为该数据提供验证的 DAO 成员，以及各成员的数据评分。

VOKA AI

「虚拟形象」一词是当前国内元宇宙的产业中心，并通过广告合作、线上演出等场景为企业的虚拟人寻找商业价值。而在未来的 Web3.0 世界，强调所有权的虚拟形象也是用户体验游戏和社交时必不可少的基本应用。

VOKA Al 即是一个支持智能合约的虚拟形象生成协议，用户可以根据自己的需求，使用 PFP NFT 资产或 AI 算法来生成适合特定应用场景的虚拟形象。这些虚拟形象的应用场景广泛，包括视频会议、网络直播、虚拟社交，以及 Web3.0 游戏等等。此外，VOKA AI 正与优质设计师和服装品牌合作，打造各种虚拟服装。

交互界面

首先进入 VOKA AI 网站，链接钱包并创建帐户，即可开始生成虚拟形象。VOKA 将扫描钱包中可用的 NFT，用户在选择性别以及虚拟形象模版后便可基于自己的 NFT 生成虚拟形象。如果该钱包没有可用的 NFT，用户还可以使用「Anime PFP Generation」功能来创建自己的动漫 PFP。

生成虚拟形象后，用户可以对其衣服、发型、眼球等属性进行修改。生成的虚拟形象会存储在 VOKA 的服务器上，用户也可以选择将其铸造为 NFT。一旦用户将头像铸造为 NFT 后，协议将自动从系统中删除先前的动漫 PFP。

使用 VOKA 生成的 PFP 虚拟形象格式适用于大多数 3D 引擎以及元宇宙应用。未来团队还将将提供 SDK 来导入元宇宙创作者及游戏开发者的头像文件。此外，VOKA 还提供 PC 端表情捕捉软件，用于在线直播和社交。
