许多项目正在或打算在 Polkadot 和 Kusama 上构建。然而，有些人只是说说，并不会 真的这样做，这可能是因为他没有足够的资源来实现，或者他们只是想通过滥用 Polkadot 和 Kusama 的品牌来欺骗 社区。

分辨真假「波卡生态项目」并不总是一件容易的事。本指南旨在帮助你，在遇到一个看起来还不错的项目时怎么去研究它。请注意，这不是在告诉你某个项目是否「合法项目」，或者为你做出决定。

此外，一个合法的项目并不一定意味着它也将是一个成功的项目，本指南不应被视为财务或投资建议。

以下是该指南的上篇，主要列出一些识别真假项目的「硬指标」，下篇也会在近期发布。

注意「由 Polkadot 提供支持」或「Polka」前缀的情况

许多项目网站上的「由 Polkadot 提供支持」的声明通常会引起混淆。这通常意味着项目正在或打算使用 Substrate 在 Polkadot 生态系统上构建。但是任何项目都可以这样声称，因此项目网站上存在此声明并不意味 着该项目的合法性，当然这也不是 Web3 基金会的「批准印章」 。

对于名称中带有「Polka」前缀的项目也是如此。许多项目使用它来将自己与波卡生态联系起来，有些是合法的，有些只是为了蹭 Polkadot 的品牌效应。

必须满足的一些硬指标

1. 与你信任的实体有关联或者合作

新项目通常试图通过将自己与知名的实体联系起来来提高其可信度。想法很简单：「这些信誉良好的实体信任我们，所以如果你信任他们，你也应该信任我们」。事实上，与受信任实体的关联可以成为项目合法性的有力指标。

例如，如果一个项目获得了 Web3 基金会 Grant，这表明该项目确实建立在 Polkadot 生态系统上，如果他们已经交付 了所有里程碑，那么他们的代码很可能具有合理的质量。

Web3 基金会并不是生态系统中唯一提供资助的实体。生态系统中还有其他信誉良好的团队已经开发了平台或平行链，并为其生态项目提供资助以建立或扩展他们的生态。这些也是一个合法项目致力于在更广泛的 Polkadot 生态系统上构建的参考指标。

或者从你信任的风险投资公司那里获得资金，并且已知参与了其他知名的 Polkadot 项目，这也是一个很好的指标。或参与 Substrate Builders Program 的项目。

但是，声称拥有此类合作和拥有它们并不总是一回事。你始终需要验证 项目提出的任何声明，这可能是本指南中最重要 的内容。

例如，如果一个项目在其网站上有 Web3 基金会 Grant 的徽章或声称已收到 Grant，请检查他们是否收到过 grant 并且是否已经终止。可以在此处（https://github.com/w3f/General-Grants-Program/blob/master/grants/accepted_grant_applications.md）找到已成功申请 grant 的完整项目列表，你还可以在这里查看每个项目已交付的内容，以及他们的资助是否已终止。

同样，这也适用于风险投资资金或其他 grant。检查相应的网站以确保此类声明有效。

此外，你还需要了解合作的范围 。回到 Web3 grant 示例，它是有精确范围的。被资助的项目需要有可交付的结果，审查团队只检查代码并评估这些项目的特定可交付成果。因此，获得 Web3 资助，不意味着为有关团队的一般实践、项目寿命或团队正在构建的其他项目背书 。这就是为什么我们一再明确指出 Web3 grant 的徽章不应该显示在团队的登陆页面上。

同样，如果一个项目声称与信誉良好的实体合作，请通过他们的网站搜索与他们合作的项目、新闻稿或直接与他们联系来验证其合作范围以及是否确实是合作伙伴关系。如果你在哪个项目网站看到有 Web3 Foundation 的此类声明，可以确定这是错误的，因为 Web3 Foundation 不与生态系统项目合作或者对某个项目进行「认可」。

2. 开源

开源项目可以提高透明度，建立信任，并确保项目团队不会在背后做可疑的事情。此外，它还可以非常轻松地跟踪项目进度并查看团队在开发过程中的活跃程度。

但是，这并不意味着任何闭源项目都是不合法的，或者它背后的团队有什么要隐瞒的。许多团队选择将他们的代码保密以保护他们的知识产权。一些这样做的团队也获得了我们的 General Grant，grant 审查小组的成员会审查他们的私人代码。

开源项目允许你查看的另一件事是他们是否从其他开源项目复制代码。这不一定是坏事，因为这可能避免了重复造轮子的事情，但复制的代码应该归因于源代码。如果没有归因源代码，这应该会引发一些不好的事情，因为这个项目团队可能是抄袭别人的代码来伪装自己在技术上的能力。

分叉的 repo 很容易被发现，因为它指向原始 repo，但部分复制的代码可能不那么容易找到。这个工具（https://duckduckgo.com/?q=search+for+copied+source+code+plagiarism&t=ffab&ia=web）可以帮助查找抄袭的方法。

因此，封闭源代码的项目不一定是危险信号，它只是限制了验证项目这方面的能力。然而，一个开源项目无疑是其合法性的一个很好的、强有力的指标 ，因为阴暗和糟糕的想法很难在开源代码中长期隐藏。

3. 代码活跃度

如果一个项目团队不断更新他们的产品，这是一个很好的迹象，表明他们对开发是认真和热情的。定期发布新功能和升级、修复 bug、更新他们的网站并将这些更改通知社区，这是合法项目的良好标志。

此外，对于闭源项目来说，活跃的开发通常也意味着有良好的开发能力。当然，开源项目提供了直接通过其 GitHub 存储库监控开发活动的能力。

4. 扎实的文档

对于任何想要认真对待的项目，都应将建立一个扎实的文档视为强制性的 。几年前，这意味着一份白皮书，但现在，我们看到了它在向其他形式的文档的转变，例如描述各个方面的 wiki 页面。

无论文档的形式如何，它都需要存在，并且越详细越好。这是详细解释项目的地方，潜在的投资者和用户可以阅读它的工作原理、目标以及团队如何实现这些目标。

该文档还将让你了解团队的技术专长。如果他们有分析技术方面的文档，则表明他们知道他们在做什么。另一方面，如果他们只专注于代币经济学或仅从广义、模糊的角度分析他们的项目，这可能表明他们没有明确的实现目标的路径。

如果你想参考优秀文档的示例，请查看我们自己的 wiki https://wiki.polkadot.network/。当然，你不应该期望在新启动的项目中找到如此广泛的文档。毕竟，我们的 wiki 涵盖了整个生态系统，并且多年来一直在维护和补充，而且它仍然一直在更新中。但这是一个很好的例子，说明一个好的项目应该强调建立文档的作用和方式。

5. 信誉良好的团队

一些团队在他们的网站上非常显眼展示他们的团队成员，以及他们的社交媒体资料（通常是 LinkedIn）和 GitHub 帐户，供开发团队成员使用。这使潜在用户和投资者能够验证团队的凭据、跟踪记录和专业知识。

但是这里的关键词是验证！不要以表面价值来判断你在项目团队中看到的情况。查找他们并验证他们的记录。对提到的团队成员进行 Google 搜索。如果它是空的，或者唯一的结果是关于项目的，这表明他们的团队成员是假的。他们网站上的照片（如果有的话）也可能是假的。这些通常很容易识别，但如果你想要彻底查找，这里有一个关于如何进行反向图像搜索的指南。https://helpdeskgeek.com/how-to/free-reverse-image-search-tools/

在其他一些情况下，一些开发人员更喜欢保持匿名、使用假名，或者根本不提及团队成员。这不一定是坏事。也许团队是隐私的坚定支持者，或者他们希望他们的工作代表他们的言论和态度。尽管如此，你还是应该尝试找出该项目的幕后推手和他们在做什么。对于开发人员来说，他们的 Github 活动量可能更重要。其他团队成员可能会大量参与他们的社区，提供指导和答案，这也总是一个好的信号。

但是，如果团队不出现在任何地方并且仅通过代理与社区互动，那这可能是一个危险信号，你应该保持怀疑。

除了他们的社区之外，认真建立在 Polkadot 上的项目通常会与更广泛的 Polkadot 社区互动。他们活跃于各种 Polkadot 和 Kusama 频道，其中一些人是 Polkadot 大使，或者是生态系统中的重要成员。

6. 与 Polkadot 有明确的集成关系

在 Polkadot 和 Kusama 上构建项目的方式有很多。也许最直接的目标是成为平行链。一些最知名的 Polkadot 生态项目已经是 Kusama 上的平行链或准备成为一个平行链，并且当它们上线时，它们中的大多数也会成为 Polkadot 的平行链。

当然，不是所有的生态项目都能保证在 Kusama 和 Polkadot 中的任何一个网络上获得平行链插槽，所有项目都需要通过 auction 来赢得平行链插槽。

通过访问 polkadot.js.org/apps 上的平行链页面，可以快速验证  Kusama 上当前哪些项目是平行链。在平行线程页面中，你可以看到哪些项目正准备申请平行链插槽，拍卖页面显示哪些项目正在竞标下一个插槽，以及 crowdloan 页面显示哪些项目正在从其社区收集资金以参与 Auction。

但并非 所有 使用 Substrate 构建链的项目都旨在成为平行链。一些团队使用它只是因为它的基础设施可以来构建他们的定制链，没有任何连接到中继链的计划。还有一些项目可能旨在成为 Kusama 或 Polkadot 上的平行链。

然而，构建一个潜在的平行链并不是在 Polkadot 上构建并扩展其生态系统的唯一方法。 一个项目的目标可能是在平行链上构建一个 DeFi 平台，在 Statemint 上发布 stablecoin 或其他代币，构建一个去中心化的交易所或任何其他人们可能想到的 dApp，而无需触及中继链。这就是 Polkadot 的美妙之处！

但在所有这些情况下，他们在 Polkadot 上构建的计划，无论他们是什么，都应该在他们的网站和文档中明确说明。不过，最重要的是，你应该寻找有关他们计划如何实现这种集成的解释。 如果没有明确说明实现目标的步骤，只是将集成置于未来某个时间点的路线图，这意味着这几乎为零。 这些应该与你对团队的技术能力的研究一起进行评估。

对于已经在另一个网络上，例如在以太坊或币安智能链上运行，并在那里发行了 token 的项目尤其如此。许多项目这样做要么是为了筹集资金并测试其基础设施，要么是因为它们旨在构建「多链」解决方案，或两者兼而有之。但是因为这些项目目前不是建立在 Substrate 上的，所以在你的研究的过程中，它是否有明确的与 Polkadot 集成的计划是必不可少的，要确保它们确实会建立在 Polkadot 上。
