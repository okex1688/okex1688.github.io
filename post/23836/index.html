信标链

信标链（Beacon Chain）已运行超过6个月了，除了Prysm客户端停止生产区块几个小时并造成一些轻微晃动之外，很难想象它的情况还能变得更好。总的来说，遭遇罚没的情况要比我想象的更多一些，但其中的多数是由单一供应商引起的，据我们所知，所有的罚没都是因为节点运营商（无论是个人还是质押服务）的错误配置造成的，而不是涉及客户端或协议漏洞。请记住，验证者只会因为违反链规则而被罚没，而不是因为停机，如果你遵循良好实践和建议，那你就不会遭到罚没。

在过去的6个月中，验证者队列很少是空的，最开始时有21,063 个验证器，现在我们已经有了157,479 个验证器（加上140个因罚没而退出的验证器）。因此，平均而言，我们在整个时间段内每天大约新增735个验证器，这与每天900个的最大增长数相比相差不远。目前，以太坊2.0存款合约内的以太币数量达到了5,324,034 ETH（价值140亿美元），这是社区对以太坊的技术、团队、实现以及未来的信任投票。作为它的一部分，我感到十分谦卑。

Staking（质押）

想要在Windows系统上进行质押吗？好吧，Lighthouse也加入了派对，其他客户端也已经支持了它。如果你真的走Windows路线，这里有一些提示，它可以让你的安装保持良好的状态（无论你选择哪个客户端）。

工具

除了其他标准 API 端点（endpoint）之外，还建议客户端支持标准验证器 API 端点。这将有助于在客户端之间提供通用用户界面，目标是更容易切换客户端并提高网络上客户端的多样性。

合并（The Merge）

合并是以太坊从工作量证明（PoW）过渡到基于权益证明（PoS）共识机制信标链的点。

虽然Rayonism项目证明了合并的整体可行性，但它没有探索从PoW到PoS本身的实际转换。Mikhail最近发布了一份过渡过程的技术设计‌，有很多事需要谨慎对待，在合并之前，现有矿工的行为极有可能无法预测，因此在规划中需要考虑各种情况。

研究

  1. 矿工/验证者可提取价值正成为当前以太坊以及合并后以太坊的一大主题。这里有一篇文章提出了有关MEV对信标链去中心化影响的担忧‌。简而言之，MEV的经济性将使得独行质押者处于不利地位。我决不认为这是必然的，有几个原因，首先，这是研究者目前正在深入研究的领域（因此将其归入本节），像Flashbots这样的计划旨在使MEV民主化，而且很可能会取得成功。事实上，Vitalik就在一天前发表了一篇关于这一主题的帖子：提出者/区块构建者分离友好费用市场设计‌。另一个原因是，在未来几年，基础链很可能只用于L2解决方案的数据可用性，MEV将向上移动到rollup层，而不再是验证器要关注的问题。
  2. Georgios Konstantopoulos担心验证器由于丢失了取款密钥而无法收回其质押资产，他还提出了一种简单取款凭证轮换机制‌。这绝对是一个问题：我们最初使用的BLS提款密钥机制是一个陌生的的工作流程，几乎没有标准工具，我完全可以相信，质押者可能会无意中失去对其提款密钥的访问。现在我们有了0x01凭证，其使用了标准的以太坊帐户，这对未来的质押者而言将是一个好消息。尽管如此，这项提议不太可能得到实施：“非托管”的质押服务有太多机会窃取质押者的资金，但我相信这方面的研究工作将继续下去。
  3. Protolambda已经发布了他对合并后的分片路线的看法，并总结了未来有趣的挑战。很多工作都在幕后进行当中。

定期电话会议

实施者电话会议

第65期电话会议发生在6月3日。

  1. 议程‌
  2. 视频‌
  3. 我的笔记‌

这次电话会议要比以往更长，互动性也更强。有三个话题引起了讨论（在我的笔记中没有很详细的记录——如果你感兴趣的话，我建议你查看一下录音）。

首先，是否改变Altair规范中同步委员会的奖励结构，当前的方法可能会导致行为相同的验证者获得的奖励之间存在显著差异。改变这一点基本上是确定的，主要的讨论围绕着选择哪一个提案：一个是来自Potuz，另一个来自Vitalik，两个版本都很强大，但Vitalik的版本最终赢得了更多支持。

第二，关于是否包括Jacek提出的在状态中放入历史区块根累加器的提议，以及这最好是在单个协议升级中完成，还是跨2次协议升级完成。在这个阶段，有些人不愿意为Altair添加更多的元素，但大家都认为基本的想法是正确的。最终，Altair升级的最终规范（Alpha 7）发布被推迟到下周，以便对此进行更多的讨论。

第三，Rocket Pool 的 Joe Clapis 提出 0x02 提款凭证，以帮助强制执行合并后质押池中交易奖励的公平分配。对此作出决定并不紧迫，因此它更像是对这个问题的介绍。
