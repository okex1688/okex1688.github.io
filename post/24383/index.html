金色财经 区块链6月16日讯   Layer 1区块链通常可以从代币销售中获得巨额资金，而且也能获得大量支持来与以太坊竞争；不过，市场上现有的工具很难赶上迅速变化的用户需求，目前核心基础设施依然是区块浏览器、钱包、开发者工具、以及（经过测试的）智能合约代码。 Layer 1区块链能够通过代币激励来吸引用户，但随着 Rollups 提高以太坊的可扩展性，Layer 2和Layer 1之间的差异越来越小。

Layer 1区块链生态系统资源对比

如果我们想要比较 Web3 应用程序的开发和构建工具，可能需要关注的第一个问题就是：许多Layer 1区块链生态系统拥有非常庞大的可用资金，比如：

1、Solana最近获得了a16z和Polychain联合领投的3.14亿美元投资；

2、之后Dfinity又宣布推出一项高达2.23亿美元的开发者生态系统计划。

此外，通过初始代币销售获得的大量“财富”也是这些Layer 1项目的主要资金来源。比如：

Web 3基金会拥有波卡（Polkadot）DOT初始代币分配的30%——按照当前价格计算价值75亿美元。

随着时间的推移，大部分代币分配模式已经标准化，基本上是：大约 1/3 的代币分配给项目核心开发团队，10-20% 在公开发售前分配给投资者，其余的留给公开发售和生态系统奖励。

在区块链发展初期，比特币分配算是比较“公平”的，矿工可以按照共识机制获得激励（尽管早期阶段除了中本聪之外并没有太多比特币矿工），但现在的代币分配模式似乎发生了变化，有些类似于初创公司分配股权，早期生态系统参与者（比如开发者和投资者）往往能够拿到更高比例的激励。相比之下，以太坊、EOS和Cardano在项目初期都释放了大量代币（超过80%以上）。

1、NEAR 协议的代币分配模式是：11.7% 的代币分配给了早期生态系统、17.2% 的代币用于社区赠款。 

2、Polygon协议与以太坊社区更紧密地结合在一起，他们保留了 35% 的代币，用于生态系统和质押（Staking）奖励。

Layer 1区块链开发者对比

在引导区块链生态系统方面，资本并不是稀缺资源。相反，项目通过激励措施和对给定平台的流行程度的独特愿景来争夺开发人员。技术细节往往被高估，业务发展方面反而更为重要。

区块链通常有两个技术论点可以让开发人员在以太坊之外构建：

1、提高可扩展性（交易吞吐量）

2、不同的治理和安全模型

在可扩展性方面，扩展区块链的最简单方法是增加验证器要求（从而减少验证器数量）以获得额外的交易吞吐量。这有减少权力下放的缺点。

另一种方法是通过水平扩展，通常称为“分片”——即将网络分成多个部分，通过一些过程来确定如何在不同环境之间选择/重新分配验证器、每个分片的共识机制以及全局状态。在分片系统的一端，有以太坊 2.0 或 NEAR 协议方法，具有在分片之间混洗的全局验证器集。 

1、Avalanche 和 Polkadot 通过自定义验证器集，在每个子网的设计方式方面提供了更多的可定制性。 

2、Cosmos 最大限度地实现了区块链自治，每个连接的链负责管理自己的安全——该协议没有对安全的全局观点。 

3、Solana 旨在通过软件和硬件架构优化将垂直的、单一的可组合区块链扩展方法发挥到极致。

对于开发者来说，分片的方式意味着 去中心化应用程序在同一个分片链中是同步可组合的，而跨分片通信是异步的 (较慢)。这对于搭建像 1Inch 这样的 DEX 聚合器来比较不同交易场所的价格并以最佳价格进行原子兑换构成了一定的限制。

对分片方式的一个批评就是，当通过分片方式对一个去中心化的数据层进行扩展时，其带来的可扩展性好处是有限的。而使用 Rollups 的二层扩展可以在区块链之上实现更高的交易吞吐量，但可组合性会再次被限制在单个 Rollup 链内，而且跨 Rollup 通信是异步的。

可能很难理解为什么任何应用程序都应该离开（或部署在）以太坊之外。 毕竟，以太坊依然是目前最安全、运行时间最长的智能合约平台，所有其他协议/应用程序都在以太坊上，而区块链可以说是最去中心化的。在以太坊之外的另一条链上构建的常见原因是交易成本高，但 Compound 的创始人 Robert Leshner 曾表示“[Gateway] 不是扩展工具。”

支持像 Compound 这样的特定于应用程序的区块链“网关”，可能更多地看中了其主权性。 此外，它可能是Layer 1区块链和建立在其上的协议之间权力平衡转变的先兆。
