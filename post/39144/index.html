摘要

  • Web3 社交需要更多 Crypto-native 的创新玩法，而非 Web2 Copycat。社交范式的变革将是一场以「平台为中心」到「用户为中心」的社交文艺复兴，新叙事包括五个关键主题：社交图谱、去中心化身份、隐私保护、可组合性、代币化，我们仍处于社交范式变革的早期。

  • 社交赛道从下至上可划分成四个 Layers：数据层，协议层，隐私层和应用层。其中协议层最为核心。协议层长期看好能持续吸引用户产生新的关系、身份、荣誉的社交协议和基础数据网络。应用层看好能形成可持续 tokenomics 的社交应用。此外，灵魂绑定代币（SBT）形成的社交关系或许可以兼顾可组合性与弱金融属性，将链上社交代币的核心价值回归到关系证明本身，成为真正有效的社交凭证

Web3 去中心化的社交网络应当是什么样的？

如今很多 Web3 社交产品仍旧沿用 Web2 商业模式，但这样的单打独斗很难。我们需要的不是诸如 Web 3 Twitter，Web 3 Facebook 之类的 copycat，而是更多 native 的创新玩法。

漫谈 Web3 社交范式变革

Web2 每个人都生活在社交巨头们铸造的「围墙花园」中，用户待在「围墙花园」里的时间越长，「围墙」缔造者就对他们越了解，但一旦去到另一座花园我们必须舍弃所有的社交关系和数据，净身出户。在 Web3 塑造一个更加公正、自由、平等的互联网环境将成为可能。

正如欧洲文艺复兴带来了从「神」到「人」的人文主义思潮变革，Web2 社交到 Web3 社交的演化，也将是一场轰轰烈烈的从以「平台为中心」到以「用户为中心」的社交文艺复兴。图片

笔者认为，社交范式变革的新叙事将包括五个关键主题：社交图谱、去中心化身份、隐私保护、可组合性、代币化。

关系变革：社交图谱物归原主

在 Web3 的叙事中，用户社交图谱将不再被某个公司所垄断而是完整地保存在链上。

Web3 社交也许就像是元宇宙中的社交：你可以魔兽世界里看到你微信上的好友，像电影《头号玩家》中男主角与他们的小伙伴们在不同的场景中保持好友关系，甚至可以通过算法找到精神伴侣。

实际上社交图谱数据并不容易获得。

形成完整的社交图谱首先需要强大的多源数据清理、关联、归集算法能力。而且用户的关系不能纯粹从链上行为来分析，很多社交信息仍存储在链下服务器。最后，社交图谱往往不只是现有数据的管理，还需要多走一步通过开发产品功能来引导用户形成自己的关系网络数据。

身份变革：可追溯，无需证明的去中心化身份

究竟什么是 Web3 社交？下图简单明了地回答了这个问题。

在 Web1、Web2 时代，我们需要进行繁琐流程，证明自己是账号持有人。Web3 仅需使用钱包私钥对一个字符串进行签名就能证明地址归属权。

去中心化身份（DID）或许将成为 Web3 用户最重要的资产 。链上数据的透明和不可篡改确立了身份信任的基石，Dapp 数据跨项目读取调用则确保了身份的通用性与可组合性。无论是个人信誉、NFT 持有、金融借贷、票据、身份证书、执照护照等，都可以转化为用户拥有完全控制权且独有的链上信息，通过一个身份协议来访问所有应用。

  • 对于项目方而言，可以在冷启动阶段轻易找到空投的核心人群。

  • 对于社交 Dapp 而言，去中心化身份能原生地把社交关系和群组关系建立起来。特定的入群门槛可以原生地通过读取地址链上的资产信息和历史证明进行自动判断，无需证明。

隐私变革：多账号，信息授权，隐私计算，隐私交易

链上数据属于每个人自己，同时也对每个人可读，也许你会觉得这样的公开没有隐私可言？图片

数据隐私问题可以拆分成三个层面：社交应用层，交互层与数据存储层。

  • 在社交应用层，问题解决方式是：多账号与信息授权。多账号，即不同的钱包可以有不同的圈子和身份。但如果一个 DID 可以对应多个钱包地址，那更稳妥的解决方案是让用户在社交图谱、DID 协议中选择哪部分数据信息可以被公开读取。

  • 在交互层，引入隐私计算协议，在保障数据隐私安全的基础上，让数据以「可用不可见」的方式进行安全流通。

  • 在数据存储层，区块链数据不可变与公开的本质与隐私相矛盾，隐私问题变得更加复杂。由于数据一经上链就无法更改，而将账户和交易全部加密又非常昂贵和耗时。Mid-way solutio 可能是事前让用户选择哪部分数据需要隐私保护，不想公开数据直接进行链下存放。

可组合性：可组合性之于软件就像复利之于金融

可组合性将使加密货币处于金融和其他行业革命的前沿，这将像互联网一样从根本上改变行业，并将开辟一个无限可能性的新世界。

我们认为「可组合性」包含几方面要素：

  • 软件可组合性：即「直接获取其他协议生成的资产或数据，然后与另一个协议组合使用」的能力。这样的不断组合和被组合，让每一个创新的成果都成为下一轮创新的起点，在软件组合之中诞生更强大的模式。如，借贷协议与社交协议的组合可以形成「社交信贷」。

  • 数据可组合性：新产品不再需要从头开始累积用户数据，去中心化数据库把用户的身份信息、社交图谱、社交历史等数据存储在同一套数据模型中（如 Ceramic），供 Dapps 进行统一访问和原生互操作。

  • 身份可组合性：Dapp 数据可以跨项目读取调用，不像 Web2 那样被隔离存储在中心化的服务器上，确保了用户身份在链上世界的通用性与可组合性。

  • 可拆卸性：用户应当有权力选择仅向特定和应用协议授权部分个人信息，并可以随时取消第三方信息 access。

代币化：为收益而来，为网络而留

如果 Web2 是「为了工具而来，为了网络而留」，那么 Web3 将是「为了收益而来，为了网络而留」。

Web3 社交应用中出现了有很多有意思的共享收益玩法，值得长期关注：

  • 粉丝经济：Rally 帮助创作者发行粉丝代币，去掉中间商抽成。随着创作者人气上升，代币价值也会上升，创作者和持币粉丝共享这一收益。

  • 创作内容 NFT 化：Mirror 帮助作者将一篇文章铸造为 NFT 并发起 DAO 众筹。可以把每篇文章 NFT 理解为基金，每个人都可以持有一定的份额，而持有人的主要收益就是 NFT 转手费用的分红以及份额的升值空间。

  • 社会关系 NFT 化：Lens Protocol 将用户社会关系铸造为 NFT，这些 NFT 将随着内容 NFT 的增值而增值，在创作者和用户，内容之间形成了一种微妙的共生关系；这为社区提供了良好土壤，Atem Network 就通过 NFT 将社区连接起来，用户可以在 NFT 社区中讨论、交易并且共同治理。

Mirror 文章众筹

然而，我们能观察到几个共性问题：

  • 绝大多数的创作者由于粉丝不够多，得不到足够的激励。最多的打赏或 NFT 众筹都流向了头部项目，越不缺钱的人，越能拿到钱。

  • NFT 铸造 Gas 费太高，中小创作者和用户铸造 NFT 的收益可能还赶不上 Gas 费。未来 NFT-driven 应用的解决方案很有可能是上 L2 或者在其他公链做 Appchain，才能把费用降下来。

Web3 社交全景图

笔者认为，社交从下至上可以分成四个 Layer：数据层，协议层，隐私层和应用层。

数据层

数据层包括去中心化数据网络和数据库，如 Arweave, IPFS，Ceramic，Kwil 等，用于存储用户链上身份数据，社交关系和通信内容。

由于链上数据不全、存取效率不高等原因，目前部分数据仍在 AWS 等中心化的服务器中。随着更多用户数据上链，去中心化数据库将迎来广阔的发展空间。

Ceramic

Ceramic 是一个基于 IPFS 构建的去中心化的、可跨链的、能管理动态内容数据的数据库服务。

  • Ceramic 通过存储数据流（Stream）解决了动态存储问题，还提供了非常强大的内嵌身份验证机制 3ID DID 和 IDX 跨链身份协议。目前已有多个社交项目在 Ceramic 上开发，比如社交图谱 CyberConnect，Web3 推特 Orbis，即时通讯平台 The Convo Space 等等。

  • 去年 12 月份 CyberConnect 用户大量涌入曾导致 Ceramic 宕机，其高并发解决方案有待时间检验。

协议层

协议层主要包括去中心化身份和社交图谱协议，笔者认为这将是社交赛道的核心层。主要原因如下：

  • 普适性极强，不论用户使用哪款社交 Dapp，用户都需要用社交图谱和 DID。

  • 协议层有较强的规模效应，未来很可能会跑出来一家或者几家全网通用协议。

  • 社交图谱和 DID 赛道都处于发展早期，一方面是行业内部尚未形成统一的标准，另一方面用户的认知较浅，行业竞争格局尚不成熟，存在较高的投资价值。

从更宏大的视角看，为了构建一个真正自由言论的互联网环境我们需要的是协议，而不是平台。正如《Protocol, Not Platforms》一文所论述的，做协议至少给了我们自由的可能性，做平台又陷入到了围墙花园式的旧话语体系中。

推荐阅读《Protocols, Not Platforms: A Technological Approach to Free Speech》—— MIKE MASNICK

社交图谱

社交图谱赛道中的热门项目包括 CyberConnect, Lens Protocol,5 Degree 基本总结如下：

核三大产品都支持 Profile (个人主页) 和链上社交关系，Lens Protocol 和 CyberConnect 则有内容和通讯方面的扩展，我们重点比较一下两者：

CyberConnect

CyberConnect 旨在聚合链上链下数据，提供一个标准的数据层。开发者可以通过 API 实现调用。所有的 DID 的关系数据和 Dapp 的使用数据将会通过 Ceramic Network 存储在 IPFS 上。

这种方案的优点是轻量、可插拔，CyberConnect 注册用户达 140 万 +，70 + 项目已经实现 API 接入。

缺点是目前社交图谱能提供的功能有限。同时，无成本的社交图谱读写在接入中容易产生冗余数据。此前 Facebook 的社交图谱就曾深受 Zynga 困扰，在处理和接入数据上需要更多平衡。

Lens Protocol

Lens 旨在成为开源的社交基础设施，它将生态内所有的社会关系 NFT 化以构成社交图谱；Lens 也自建了协议标准，吸引开发者在此基础上开发功能模块和应用，更像是做公链生态的思路。

Lens 的优势：

  • 它是 Polygon 上的智能合约，模块间的可组合性非常强，生态内的应用可以共享社交内容、关系和模块组件；CyberConnect 是基于 IPFS 的协议，可组合性要差一些。

  • Lens 将个人主页、内容、粉丝关系铸造成 NFT，具备较强的货币化空间和想象力，e.g: Lens 个人主页 NFT 一个月内地板价 10X，未来还会有更多的 NFT 发行活动。

  • 相较于 Deso 社交网络公链，Lens 的生态与经济体系更加开放，这对于社交网络而言非常重要。Deso 生态中的社交 App 均需要使用 $DESO 代币，而出现明显流动性不足，用户量较少。

Lens 的局限：

  • 从落地情况上看，Lens 目前只是建立了社交联系，和图谱还有一定的距离，需要开发者自己去抓链上数据（目前大部分开发者使用 The Graph）。

  • 当然 Lens 还面临着另一个问题，它每一次交互都需要调出小狐狸钱包支付 Gas，用户体验不够流畅，CyberConnect 则是把内容存入到 IPFS，用户体验更好一些。

Lens 生态建设模式较重，对项目方的市场拓展能力要求高。Lens 背靠 Aave 团队，目前还在内测试阶段但整体建设进度较好，头部项目包括 Phaver (Share-to-earn 社交应用)，Lenster (去中心化论坛) 等，Dapps 可以依靠 Lens 的精准流量进行快速冷启动。Lens 未来与 Aave 业务的协同潜力也值得长期关注。

去中心化身份 DID

Amber Group 于 2021 年 11 月勾勒了 DID 生态系统。DID 位于 Layer3，处于整个生态的枢纽位置。

DID 项目可以大致划分为四个赛道：链下身份认证类、链上身份聚合类、链上信用评分类、链上行为认证类。值得强调的是，DID 成熟形态尚不清晰并没有严格的定义，有些项目是身份协议，有些是凭证网络或者身份应用，且专注于不同的应用场景。未来 DID 很可能是一个整体其中有各种组成部分。

a.链下身份认证

假设一个人毕业于斯坦福大学，曾在谷歌当工程师，在硅谷有两套房，如果上述信息能够得到认证而与这个人持有的一个地址绑定，那么这将为他在链上的社交、借贷等各种活动提供加持。

BrightID

BrightID 是真人身份验证的代表性项目，它构建了一种社交身份网络，目前其已有超过 5.7 万名用户。用户需要预约 zoom 视频会议，通过人脸识别和验证官复核保证身份唯一性。

Verite

Verite 由 UDSC 的发行方 Circle 公司于 5 月 10 日推出，其中 KYC 是其身份验证的关键之一，具体细节有待项目方进一步披露。

b.链上身份认证

这类的 DID 项目并不关注线下的情况，而是通过「数字身份」这一概念实现链上身份信息的聚合管理

Unipass

Unipass 是这类项目的代表之一，它做的事情正是链上身份的聚合管理：通过一个 Unipass ID，用户可以绑定邮箱、多个 ETH 地址，并连接社交图谱（CyberConnect）、信息聚合平台（RSS3）等协议，变成用户在 Web3 的「One Pass」。

Spruce

Spruce 是一个数字身份认证系统，其产品套件为用户的身份验证、凭证和存储提供支持，提供数据去中心化访问控制以及 Web2 API 与 Web3 之间的互操作。与 Web2 现有平台账号的绑定，对于一些已经在 Web2 中有一定影响力的人而言，是一个很不错身份增强方案。

c.链上信用评分

这类项目的初心非常直接 —— 拓展 Defi 借贷的场景，把现实世界中金融行业的信用机制在 Web3 中找到某种实现。

ARCx

ACRx 通过发行「DeFi Passport」，根据每个 DeFi Passport 信用分来量化其信誉度。相比于前两类 DID 项目，链上信用评分类项目更聚焦于用户「数字金融身份」，使用的也多是链上交易数据而非其社交数据。

d. 链上行为认证

这类项目的核心逻辑是，引导用户参与某些合作方要求的活动、行为，并对其颁发链上认证。

Project Galaxy

Project Glaxy 严格意义上来说不是身份协议，而是一种凭证网络。用户连接钱包后即可生成一张「银河身份证」，根据地址的历史行为，自动贴上比如「Uniswap 交易员」、「OpenSea 交易员」等等认证。项目方与 Galaxy 交互会被收取费用，同时 Galaxy 也会给予对应的 curator（数据提供方）代币奖励。

隐私层

隐私计算协议旨在让数据以「可用不可见」的方式进行安全流通。主要包括零知识证明、安全多方计算、基于现代密码学的联邦学习、可信执行环节（TEE）等技术。目前比较知名的区块链隐私计算网络有 Oasis Network、PlatON、Phala Network、ARPA、Aleph Zero、Findora 和 Deeper Network 等。

  • 隐私计算网络目前的实际应用场景亦比较有限，因为很少被产业型项目所采用，部分技术也尚不成熟。

  • 未来社交需求爆发或将为隐私计算赢得更多用户，但是隐私层的投资优先级或许并不会像其他层这么高。因为一个用户需要隐私处理的数据可能只有一小撮，且还需要满足 web3 社交需求普及 + 用户隐私保护意识觉醒 + 愿意为其付费这三个前提。

应用层

Web3 社交应用图谱 - 按社交形式分类

低门槛的社交应用也许能成为普通人进入 Web3 汪洋大海的一次尝试。

社交赛道中仍没有跑出现象级的产品，主要的问题是：很多应用只有 token 获取场景，缺乏消耗场景，用户习惯性「挖提卖」，无法形成网络效应；token 无限制地流向二级市场也导致价格持续走低。

笔者认为可能的社交产品形态有：

  • 基于 NFT 的社交应用。NFT holder 可以加入 holder 群，信用评分高的用户可以直接找到 NFT 买卖家进行 P2P 交易、Swap、借贷。其中典型如 Atem Network，希望通过 NFT 形成社区，用户可以在自己持有 NFT 的社区中讨论、交易并且共同治理社区。

  • 跨平台跨链的 IM 应用。用户在一个 Dapp 中管理 Gamefi、Dao 社区、NFT 社群内的好友关系，也能通过状态更新了解朋友们正在玩哪款游戏，又入手了哪些 NFT。

  • Gamefi 打金的玩法。一个典型的例子是 Phaver, 是 Lens 生态中领先的 Share-to-earn 应用。用户可以通过改善他人体验获得代币奖励（发布优质内容，发掘优质内容，审核信息），消耗代币以提升自己的体验（内容质押、购买流量、订阅付费内容、铸造 NFT 等）。

  • 基于 Defi 经历 or 个人投资成就的大 V 社区，类似雪球。用户可以将个人挖矿经历、投资成就展示在 profile 页，大 V 们输出优质内容获得代币奖励，也可以发行粉丝代币。

  • 基于 Dao 的内容社交应用。依靠 DAO 实现社区自治和内容审核，应用产生的所有收益也将根据治理提案分配给 DAO 成员。

投资方向

Web3 社交赛道处于早期阶段，竞争格局尚不清晰；社会关系 NFT 化、粉丝经济等新兴商业模式值得长期关注。

应用层机会：看好能形成良好经济循环、利益共享的社交应用，如 mirror。核心是判断产品是否有足够的造富效应和可持续的 tokenomics，以留住核心用户 + 优质创作者并形成内生经济循环。

协议层机会：长期看好能吸引用户产生新的关系、身份、荣誉徽章的协议和数据网络，这类产品一方面是社交数据的基础设施，另一方面也是 C 端流量入口。

SoulBound Tokens 的社交网络探索：今天社交协议主要方案还是围绕 NFT 或者链下数据库，前者金融属性过强、后者由于不上链而导致可组合性较弱；基于 SoulBound Tokens 形成的社交关系可能可以兼顾可组合性与弱金融属性，将链上社交 token 的核心价值回归关系证明本身，成为真正有效的社交凭证。
