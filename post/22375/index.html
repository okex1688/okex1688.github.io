Polkadot 的阶段上线即将进入另一个篇章。在我们即将推出平行链功能以及其背后的拍卖和众贷时，让我们来看看接下来会发生什么。

Polkadot 大约从 10 个月前开始上线，目前包括了前四个阶段，最后一次是在去年 7 月底，Polkadot 网络实现了完全的去中心化和无需许可。现在，第五阶段，Polkadot 上线的最后阶段开始了 。

Polkadot 是一个异构的可扩展的多链。在其短暂的历史中，Polkadot 已经展示了一个高度去中心化和包容性的 Staking 系统（任何人只要拥有 1 个 DOT 就可以参与 Polkadot 的 Staking 系统并获得奖励)、无缝升级其核心逻辑和开放的、功能性的治理。然而，到目前为止，它只是单独存在的一条链。随着平行链功能的推出，这种情况即将发生改变，异构分片系统的最终愿景即将成为现实 。

上周，Parity Technologies（开发 Polkadot 最初实现的核心团队）发布了 Polkadot 0.9 版，并向前迈进了一大步。这个版本，现在由 Kusama 网络的绝大多数人运行，包括在网络上部署平行链的必要逻辑。它还包括运行平行链插槽拍卖的最终逻辑和众贷的逻辑，可用于众筹拍卖竞标。Kusama 已经升级到包含在这个版本（v9010）中的逻辑，所以现在终于准备好了 平行链！

时间表

Polkadot 的平行链计划预计在两件 事情发生后启动：

  • 首先，应该对所有新逻辑进行全面的外部审计 。
  • 其次，Kusama 金丝雀网络执行至少一次成功的包含众贷的拍卖 和实现至少一个功能性的平行链 来证明新逻辑是可行的。

有关的审计工作已经开始，我们期望在不久的将来完成。Kusama 的启动之路已经开始了，预计在未来几天 分三步完成。

首先，将在 Kusama 上部署 Shell 平行链， 今天该议案刚刚通过 治理的批准。Shell 平行链是一个 “空” 的平行链，它产生区块，但是除了可升级之外没有其他功能。它没有用户帐户，Staking，治理或余额的概念。它甚至没有时间戳模块来理解“时间”。取而代之的是，它所做的只是侦听中继链治理授权它的升级的消息。收到这样的消息后，Shell 平行链将允许其自身升级到中继链授权的任何新逻辑。这是一个暂时的环境，作为第一个公共利益平行链，它的身份 id 为 1,000（第一个系统平行链的身份 id 为 0）。在 Kusama 上部署 Shell 平行链需要通过治理 motion 和公投，不过这在今天已经全部通过 了！

（PolkaWorld 在第一时间参与了投票）

第二步是进行一次升级 。在这种情况下，治理将投票决定将 Shell 平行链升级 为 Statemine 平行链。Statemine 是 Polkadot 的 Statemint 平行链在 Kusama 网络的一个实例，它是一种公共利益平行链，可以发行可替代和不可替代的任意资产。它还是一条用于保持和转移中继链本地代币（KSM / DOT）的低成本链。

如果 Shell 平行链在运行 24 小时 后而没有任何问题，并且在 Westend 测试网上成功进行了相同的升级，我们将提出升级议案。与最一开始的部署一样，升级需要完整的治理 motion 和公投。

第三步是开始拍卖 。一旦具有交易和共识的平行链在 Kusama 网络的 900 个验证人下运行的很好，就会开始进行拍卖！我们已经在 Rococo 测试网上进行了测试！看到 Statemine 正常运行之后，我们将确信所有第三方团队的链也将正常运行。

目前，大多数 KSM 都锁定在 Kusama 的 Staking 系统中，并且无法转移如果没有提前 7 天通知的话。因此，Kusama 网站将提前一个星期宣布 第一次拍卖的开始日期，以及预计的进一步拍卖时间表。目前，Kusama 网络预计将进行 5 次拍卖 ，每次间隔 7 天 。如果没有发现重大问题，则可以在其后进行进一步拍卖。在 Kusama 的第一次拍卖成功完成之后，我们希望 Polkadot 的拍卖会在不久之后进行。在确定拍卖的最终参数后，将与拍卖时间表一起发布。

未来

在接下来的两个月中，将致力于可靠性，重构和性能 。目前，有意将交易量限制在一个保守的上限，以最大程度地减少由于当前未优化的实施而引起的任何系统性问题的机会。网络只允许 500 毫秒，而不是在每个块中执行大约 2,000 毫秒的工作，从而将有效交易量减少了 75％ 以上。随着代码的快速更新和更加健壮，这些人为的限制将被解除，交易量将朝着理论的限制增加。

在这段开发期之后，除了最初设想的功能之外，还有针对 Polkadot 的三个新功能 ：Parathreads，Off-chain XCMP 和 Spree。虽然所有这三个方面的工作将同时进行，但由于功能复杂，Parathreads 可能会出来第一个版本，然后 Spree，由于其复杂性，会在最后出来。

关于 Parathreads 的功能已经写了很多很棒的文章，例如这篇《波卡平行线程：“随用随付” 的平行链》。简而言之，Parathread 是随用随付的平行链，它们不需要平行链团队赢得平行链插槽拍卖（或被接受为公共利益平行链），而是平行链代码可以直接为希望编写的每个区块直接付款。如果一个平行链很少需要编写区块（例如一个 oracle 链）或仅在特定时期内编写区块，那么不需要通过赢得一个 6 个月插槽来编写他们的区块是非常具有吸引力 的。

Off-chain XCMP 是对 0.9 系列发布的基本 XCMP 的升级。基本的 XCMP 允许消息通过中继链路由消息数据在平行链之间传递消息，而 off-chain XCMP 直接在两个平行链的收集人之间路由数据时实现了相同的目的。这意味着中继链不再是消息传递的瓶颈，并使其具有高度可伸缩性，但仍保持相同级别的安全性和去中心化性。

最后，Spree （也称为 Secure Protected Runtime Execution Enclaves 安全保护的运行时执行区域）是一项功能，允许在所有平行链上安全且均匀地分片相同的逻辑。这使得彼此不信任彼此的业务逻辑或治理机制的平行链可以进行交互。使用此功能，可以在陌生链之间移动代币，当 NFT 在链之间迁移时保持它们的属性，甚至共享智能合约，都可以使用该功能实现。
