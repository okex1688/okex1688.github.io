「互联网档案馆」是一个非盈利性的、提供互联网资料档案阅览服务的数字图书馆，是布鲁斯特·卡利于 1996 年创办的，该“档案馆”提供的数字资料有网站、网页、图形材料音乐、视频、音频、软件、动图和数百万个书籍副本的永久性储存。 除此之外，该档案馆也是网络开放与自由化的倡议者之一。

该档案馆中最重要的服务之一是 Wayback Machine ，译名“网站时光机”，早在 1996 年就拥有超过 3000 亿个网页，是为研究人员、历史学家提供一个可以保存数字文物的地方；也可以用来娱乐，比如可以看到谷歌早在 2001 年时的样子； 也可以访问一些被关闭而不再存在的网站。

「互联网档案馆」的创始人布鲁斯特和 Filecoin 基金会主席玛尔塔共同做客了播客节目，深入探讨了他们如何利用去中心化网络为未来的用户保留互联网，以及对于 Web 3.0 的思考。

互联网档案馆背后的愿景

去中心化的网络真的可以帮助这些互联网上的信息吗？

「互联网档案馆」布鲁斯特

我们最初是尝试对互联网进行存档，但实际上目标是成为互联网上的档案馆，并朝着存储“世间万物”的方向发展。我们如何去建立数字图书馆？我们能否做到让所有人都能获得所有知识？这就是我几十年前的互联网愿景。凭借我们现在拥有的技术，我们实际上可以实现这一目标。

我记得我年轻的时候走进图书馆，它似乎是“无限的”，有好几层，每一层的书架都在“无限伸展”。但它实际上不是无限的，它甚至只是人类书籍中的一小部分，更不用说期刊、电视节目以及你可以访问的其他东西。所以我的想法是，互联网是让任何人都可以成为出版商，每个人的声音都可以在互联网上发表，并且会有图书馆可以去记录它们，组织它们，使它们具有持久的价值。

所以网络是朝着这个方向迈出的很好的一步，但它实际上有很多缺点。例如一部网络电影，实际上只能从它们来自的 Web 服务器获取，并且你只能得到一个 URL 地址（统一资源定位符）。即使该电影可以从许多不同的地方获得，但它也只有一个 URL，所以当源头上的 URL 被停止访问，或者被删除，那么它将会在网络上消失。这不是一种正确处理文化的方式。

所以这个时候就体现出了网络内容副本化的重要性，就像你在多个资料库中拥有多个副本一样。

我们一直试图通过“网站时光机”来填补丢失的网站，我们每天收集近 10 亿个的网页。事实上这更像是一个 GitHub，你可以在所有不同的网站上回滚时间，查看它们在所有时间的网页副本。虽然通过使用这样的方式收集网页很简单，但是为什么我们不尝试做一些实际上更好的事情呢？现在我们有了 JavaScript，它允许用户在我们的 Web 程序中发布代码；我们有了有效的哈希代码，这些与 BitTorrent、Fliecoin 等去中心化存储网络有关。那么，我们想建立的更好的东西是什么？我们想建立一个私密、健全的网络，你可以通过在网络上发内容来真正赚钱，而不仅仅是通过第三方。因此，我们在 2016 年呼吁建立一个去中心化的网络。当时我们的开发团队聚在一起讨论，认为这些去中心化技术是我们前进的方向。现在我们已经完成了其中的一些工作，因为我们现在有了智能合约。这很酷。但是我们可以得到存储吗？我们能否获得某种类似 BitTorrent 的去中心化存储服务，在那里我们可以用哈希代码处理事务，而不仅仅是一个地址？这样的技术正在开发，我们希望看到有好的、有益的方法来引导这些技术。

对于 Web 3.0 的思考

「互联网档案馆」布鲁斯特

从 1980 年开始，我们就知道新一代的出版业即将到来，它将是数字化的。我们有一个愿景，那就是将有一个新的网络系统。它必须是开放的协议，并且必须是一个任何人都可以加入的系统。我在这方面的贡献是尝试构建 WAIS，这是一个基于开放协议的系统，具有分布式服务器和分布式客户端。我们在 1989 年开始创建，并于 1992 年在互联网上公开，这远远早于现在的互联网。它是一个系统，人们可以去获取他们所需的专业知识，从某种意义上说，它更像现在苹果公司的人工智能 Siri，在 WAIS 中你可以向存储了不同类别知识的服务器提出问题，然后它们会给出答案。它有一个简单的 URL 系统，也有一个内置的支付系统。但是最后我们失败了。当互联网出现时，人们意识到它是如此的易于使用，所以人们更喜欢网络，以至于 WAIS 的搜索系统被整合了，所以我们现在最出名的可能是作为互联网上的第一个搜索引擎，但它只是一个基础设施。

对于 Web 3.0 ，我的想法是用户可以与许多“大玩家”进行比赛，我们需要做到让所有人都可以参与其中，你不必活在别人的平台上，你也不必按照特定平台的规则去参与。

我认为目前的互联网走错了路。由于各种错误信息和虚假信息的出现，于是互联网上出现了一些拥有更好的算法的“大玩家”，他们有权力控制想要显示的信息。这是非常危险的。

在 Web 3.0 中，我们确实应该拥有过滤器机制，这样你就不会一直收到垃圾邮件。但是这不应该像现在被集中式的控制。那么，我们如何去构建一个拥有大量参与者、大量赢家和大量机会的系统，以在此基础上构建下一代网络？也许在 Web 3.0 中不会实现，那么我们也将会有 4.0、5.0 和 6.0。

我发现回顾历史是预测未来的好方法。如果你在 1400 年代和 1500 年代采用非常早期的印刷品，那么直到 1600 年代初期才真正获得基于版税的系统，这样你就可以得到报酬去写书，并进行非常简单的交易：你从某人那里购买一本书，大部分钱留在书商那里，然后一部分返还给印刷商，一部分返还给作者。早期的人类花了很长时间来发展这套体系。

所以现在，我们处于互联网的早期阶段，我们被“广告网络”所拥有。我们没有很好的”基于版税“的系统。那么我们该如何去做呢？

我们可以将其构建为 Web 3.0，以便有没有集中形式的垄断 。如果成功不了，那么让我们确保我们将达到 4.0、5.0 和 6.0。Web 3.0 听起来不错，这将有助于使一切民主化。

Filecoin 基金会主席玛尔塔

对我来说，去中心化网络最酷的地方在于能够保存人类最重要的信息。「互联网档案馆」在保护互联网方面一直在做着如此惊人的工作，不仅是今天你正在保护互联网的工作，而是你的工作可以在未来保存人类最重要的信息。对于我和 Filecoin 而言，Web 3.0 意味着创建一个去中心化的存储网络，你不必依赖一两个大玩家来有效地存储所有网络。取而代之的是创建一个系统的能力，在该系统中，即使一些节点会发生故障，但是整个系统仍然可以抵御危险，并且用户可以在其中获取信息，而不仅仅是被少数公司所拥有。 

对正在构建 Web 3.0 的开发人员的建议

要注意不同机制和方式的垄断。这些垄断基本上是个人、组织、协议，似乎在控制着一切。那么我们如何去构建一个足够灵活的系统来绕过这些类型的垄断呢？我想知道我们建立的 Web 3.0 网络协议是否真的能很好地为我们服务。

在我们构建这些系统时，我们还应该让新玩家有空间加入，而无须获得老玩家的许可。 

从更积极的方面来说，Web 3.0 这个集“全球智慧、全球大脑、全球智能”的技术可以让网络成为一个更聪明、更高效、更有趣、更有趣、更自由的世界。但这将需要每个人共同参与，打击错误信息并建立一个“全球大脑”。
