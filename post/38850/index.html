6 月 11 日，Twitter 创始人 Jack Dorsey 推出的加密项目 TBD 宣布将在比特币网络上推出一个名为“Web5”的去中心化网络标准平台，旨在“绕过”现有的 Web3 概念。

如今大多数加密项目都选择使用所谓的 Web3 概念并提供了一种基于区块链构建应用程序的方法。但在大多数情况下，Web3 需要依赖于以太坊或其他支持智能合约的区块链，以更加便捷地满足可编程性要求和 Token 支付。

但是，Jack Dorsey 似乎并不认可这种想法，他试图在比特币网络上构建 Web 组件，这种策略通常不是区块链应用程序的理想选择。

从技术上讲，Web5 将专注于身份而非 Token 化支付，简单点说就是——TBD 计划构建一个去中心化 Web 平台（DWP），让开发人员使用去中心化标识符（DID）和去中心化 Web 节点（DWN）来创建去中心化 Web 应用程序（DWA）。

据悉，Web5 平台将使用 ION，这是一个由微软和去中心化身份基金会开发的比特币身份层。（星球君 o-daily 注：TBD 是 Jack Dorsey 旗下支付公司 Block, Inc. 最近创建的一个项目，旨在专注于开发比特币基础设施。）

从实际意义上将，Web5 希望解决现实个人数据保护问题，确保不会将这些数据出售给第三方，该项目旨在将去中心化身份和数据存储带入 Web 应用程序，同时将数据所有权归还给用户。

接下来，星球君（ID：o-daily）就和大家一起来看看 Jack Dorsey 发布的 16 页 PPT 里究竟为我们描述了一个什么样的“Web5 未来”吧。

在如今的 Web 模式里，人即是用户，但人们并不拥有自己的数据或身份，企业为人提供账户，各种数据也被 APP 存储下来。而 Web5 的个人数据由个人控制，可用于个人和社交通讯、金融交互、以及其他所有 APP，为了创建一个以个体为中心的全新“DAPP 和协议”类别，我们必须要让人们自己拥有身份并掌握对自己数据的控制权。

去中心化 Web 平台（DWP）能让开发人员使用去中心化标识符（DID）和去中心化 Web 节点（DWN）来创建去中心化 Web 应用程序（DWA），这样就能将身份和数据控制权和所有权归还给个人，这就是所谓的“Web5”。

Web5 有三个核心：

1、去中心化身份：自我控制的身份能够支持去中心化身份授权和路由；

2、可验证的证书：为密码表示和验证声明提供标准数据形式和模式；

3、去中心化 Web 节点：数据存储和消息中继节点将作为去中心化 APP 和协议的基础。

去中心化标识符（DID）：

1、标识符将自主生成、自主控制；

2、没有中心化提供商和授信机构；

3、没有特别实用型Token或主观共识；

4、抵抗各种形式阻隔；

5、ID可见。

上图为可验证证书在银行金融机构的一个应用示例。

去中心化Web节点是一个新兴的数据存储和中继标准，支持任何类型的实体，比如个人、组织等，可以发送和存储加密或公共消息和数据，可为基于节点构建的各类去中心化APP和协议提供支持，主要特点包括：普遍可寻址、语义发现、可复制、异步消息线程、安全、支持任何身份类型。

Web5 将会推出一个身份钱包，具体功能包括：

1、数据管理：提供UI和功能管理去中心化Web节点中存储的APP和凭证数据；

2、DID 功能：支持创建、更新和恢复 DID；

3、证书功能：为验证方提供签署、验证、发行和展示证书；

4、DID 授权；

5、语境管理。

Web5 网络拓扑

去中心化标识符和去中心化 Web 节点结合在一起之后，将会构建出一个支持 DID 安全即时通讯、数据分享和证书交易的网络，可以用所有类型语义数据的通用标准取代一次性协议交换（比如加密消息传递协议、照片共享协议等）。

可视化 Web5 和去中心化 Web 平台堆栈

1、去中心化 Web 应用程序（DWA）包括：tbDEX 和其他 APP；

2、去中心化 Web 平台包括：去中心化 Web 应用模型、去中心化 Web 节点、去中心化标识符。

什么是 PWA？

PWA（progressive web app）代表渐进式网络应用程序，是当今所有主流浏览器都实现的可安装网络应用程序标准。

从渐进式网络应用程序到去中心化 Web 应用程序。

当前世界上的许多活动都需要在参与者之间建立信任，去中心化标识符 + 去中心化 Web 节点允许个人、组织和公司发布任何人都可以发现和独立验证的证书。

如果站在一个较高角度来看，tbDEX 是一个基于去中心化 Web 节点的线程消息传递和数据交换协议层，在去中心化标识符和去中心化 Web 节点基层之上运行。

Web5 音乐用例：在不同音乐 APP 里一遍又一遍复制自己的播放列表，这个操作可能会让许多人感到不悦，Web5 可以解决。

Web5 酒店预订用例：在当前 Web2 环境下，您的个人偏好、机票、预订和其他旅行数据散布在各种不同的酒店、航空公司和旅行应用程序中，混乱不堪，去中心化标识符 + 去中心化 Web 节点可以帮助统一这些流程和体验。

总结

Jack Dorsey 并不喜欢 Web3 这个概念，长期以来对其一直持批评态度，甚至还用“RIP Web3 VCs”这样的华语抨击 Web3 投资者。2021 年，Jack Dorsey 宣布辞去 Twitter 首席执行官一职，现在已全身心投入到加密支付公司 Block 上，而且还计划推出比特币硬件钱包和 DEX。

现在，Jack Dorsey 认为 Web5 的重要性可能超过了一切，他甚至在一条推文中提到“Web5 可能是[我的团队]对互联网最重要的贡献”。

不过，Web5 真的就是“Web2 + Web3”这么简单吗？也许只有时间能告诉我们答案。
