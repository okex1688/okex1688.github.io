ZORA是一种去中心化协议，其实现了任何人都可以在其中无许可地买卖 NFT。ZORA创建了一系列工具，使开发者可以轻松开始构建。ZORA所有的存储库都是开源的，具有MIT 许可证或GPL-3.0 许可证。

主要表现为ZORA 堆栈，包含智能合约、ZDK、API、NFT Hooks、NFT渲染。

智能合

ZORA V3 由几个核心合约和一组不断扩展的模块组成。模块是向协议添加特定功能的不可升级合约。

模块：具有可添加到 ZORA V3 的特定功能的合同

模块管理器：管理用户已批准使用代币的模块以及可以将哪些新模块添加到协议中

转移助手：处理模块的 ERC-20 和 ERC-721 代币转移

批准代币到 V3

代币必须首先获得其各自的 Transfer Helper 合约批准，然后才能被 ZORA 模块使用。例如，所有 ERC-721 代币的批准都必须通过 ERC-721 Transfer Helper。完成后，用户需要通过向模块管理器提交交易来批准哪些模块可以访问其批准的令牌。这意味着用户只需批准 ZORA V3 的代币合约一次，然后批准哪些 ZORA 模块可以访问他们的代币。

批准模块

用户可以批准单个 ZORA 模块或选择性地批量批准某些模块。每次将新模块添加到协议中时，用户都必须选择加入并批准新添加的模块才能访问他们的令牌。批准一个模块意味着所有已批准给 Transfer Helpers 的代币都将能够被该模块访问。

API

ZORA API作用使检索任何 ERC-721 上的 NFT 元数据和市场数据，用来监控链上事件、分析销售活动等等。

ZDK

ZDK是与 ZORA 的 API 交互的开发工具包，其API具体的类别如下：

聚合属性

获取有关集合的所有属性的统计信息

收藏

根据地址获取特定 NFT 集合的数据。

集合

获取一组 NFT 集合的数据。

集合统计聚合

获取特定集合的统计信息，例如总供应量、所有者数量和销量。

事件

获取与集合、令牌或所有者地址关联的所有事件，例如 Transfers、Mints、Sales、Approvals。

地板价格

获取所有 Zora 市场中最便宜的可用 NFT 数据。

市场

获取在 Zora 市场上活跃的 NFT，例如立即购买、优惠、拍卖。

铸造数据

获取任何 NFT 或任何 NFT 组的历史铸币数据。

nftCount

获取集合中 NFT 总供应量的数据。

所有者计数

从集合中获取拥有 NFT 的唯一地址的数量。

所有者计数

获取某些所有者地址持有的 NFT 数量，例如集合的顶级持有者。

销售额

获取跨多个市场的任何 NFT 的销售数据，例如 Zora、OpenSea、LooksRare、Foundation 等。

销量

获取集合在所有商城中的总销量。

搜索

根据字符串输入搜索 NFT 或集合。

代币

在给定合约地址和 tokenId 的情况下获取单个 NFT 的数据，根据查询参数获取一组令牌的数据。

NFT Hooks

NFT Hooks是与ZORA的API交互的最简单方法，由一个数据提取类和关联的挂钩组成，用于加载 NFT 数据。API 批处理和缓存请求，这意味着您可以在整个页面中使用挂钩，而无需担心显着的性能损失。NFT Hooks 默认为以太坊主网，如果要将配置设置为特定网络，需要与NFTFetchConfiguration组件一起使用挂钩。

NFT 渲染：

ZORA 提供了多种工具来轻松显示 NFT。

通过NFT iframe可将zora.co上显示的任何 NFT 嵌入到您自己的网站中。所有 iframe 都是移动响应的，并且与文本、图像、视频和音频兼容。可以通过输入要在 embed.zora.co 显示的 NFT 的zora.co URL来生成 iframe。

还可以使用渲染 NFT 内容的 React 组件，该库由React可用于渲染 ERC-721 NFT 的组件组成。库中包含默认样式和获取 URL，以及缓存机制以及轻松自定义字符串、呈现组件和添加不同样式的能力。

可以直接链接到 ZORA.co 接口，也可以通过使用 ZDK 构建功能来覆盖。
