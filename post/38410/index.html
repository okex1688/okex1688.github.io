作为 NFT 项目，该如何看待费用？

我们对许多还未上线的 NFT 项目和游戏说过，费用往往是关键的优先事项。通常，他们对高昂的以太坊费用持谨慎态度，并希望为他们的项目提供成本较低的选项——要么是因为铸币资产的运营成本很高，要么是希望他们的用户能够以较低费用进行交易。这实际上对我们也是有好处的，作为我们营销的一个关键部分，Layer 2是访问更具扩展性、更便宜的以太坊的一种方式。

然而，我们对这些项目的建议始终是，它们的主要目标不应该只是降低费用，而应该是最大限度地提高创造大型可持续经济的机会。如果一个平台或工具收取 50% 的费用，但可能会为你的社区创造 10 倍大的蛋糕，和一个次佳选择相比，选择前一个对您的项目来说将是个不错的决定。这不仅适用于区块链层，也适用于发射台等上层工具。

以太坊项目支付的费用最多，交易量最大，而 Tezos 项目几乎没有支出，也没有交易量。如果你建立某个含有1000 个 NFT 的集合正在大肆宣传，那么支付以太坊费用显然是值得的。选择平台的关键是为项目的长期未来识别功能、成本和流动性之间的正确权衡。低费用很好，我们经常夸赞在 Immutable上低费用——但那只是宣传自己是“最便宜”的区块链，没有任何关于流动性或其生态系统的信息，它们可能不是构建项目的正确地方。

我们的第二个原则是激励一致性对于 web3 的成功至关重要。理想情况下，用户、开发人员和市场都受到激励，创造一个价格强劲、成交量大和流动性强的健康经济。那些使您的成功与平台无关的费用结构，或者当您做得不好时让平台从您的项目中获取利益的费用结构，都不太可能创造长期成功。

好的，这就是我们的框架——Immutable 如何实现这个目标？在我们深入研究 Immutable 的费用结构之前，我们需要回到区块链费用开始的地方。

gas  费

以太坊（最流行的智能合约区块链）的gas费一直是启动许多 NFT 项目的噩梦，仅在去年，以太坊用户就在这些费用上花费了数十亿美元。

gas费至关重要，因为区块链是一种资源严重受限的环境。每笔交易都必须由每个节点执行（在以太坊的上，5000+），因此交易需要比简单地在链下执行该程序多几个数量级的计算能力。gas费确保交易时用户支付的费用与其消费成一定的比例，并防御拒绝服务攻击，比如去年 9 月导致 Solana 中断 17 小时的攻击。

gas 机制的工作原理是为每项操作（例如加、减、存储数据）指定成本，然后将一笔交易中执行的每项操作相加，达到总的gas消耗：

每个区块都有一个gas“限制”——该区块中所有交易总的gas 消耗。由于区块之间的时间大致固定，这意味着协议的总容量是有上限的。我们通过一个简单的机制来决定谁可以使用这个固定容量：谁愿意为他们消耗的每单位 gas 支付最多的费用？虽然也有一些区块链采用了稍微复杂的计算（最有名的是以太坊的 EIP-1559），但许多区块链只是简单地将每笔交易使用的gas x “gas价格”的值来确定费用，。

gas费使用区块链的“原生代币”支付——以太坊的 ETH、Polygon 的 MATIC、Solana 的 SOL。 随着区块链使用量的增加，对该原生代币的需求也在增加。随着原生代币作为货币逐渐被接受，这些费用成为区块链的主要货币化机制，尽管目前只有以太坊的收入与其市值相匹配。

未来，低费用的区块链将被迫应对这样一个事实，即可能需要大幅改变其原生代币的价值增值机制以证明代币的市值是合理的——这可能会导致费用增加，或产生其他提取租金类型的代币经济学（tokenomics)。

一个简单的原因：虽然rollup使状态和计算更便宜，但更便宜并不意味着无限便宜！对于像 Polygon 这样的高吞吐量侧链的情况：玩赚钱游戏 Sunflower 的玩家意识到网络的低成本使得发送数十万笔交易变得可行，这实际上导致了网络对其他人不产生价值。对此，Polygon 将其最低 gas 费用提高了 30 倍——这是处理这种吞吐量增加的必要让步。即使每笔交易的增量成本随着时间的推移而降低，即使rollup仍然需要为证明和数据可用性支付成本，区块链和rollup都需要一个反垃圾邮件机制。

如果 gas 成本对于防止网络垃圾邮件至关重要，那么 Immutable 如何避免收取 gas 费呢？难道我们会让自己和自己的游戏容易受到垃圾邮件攻击吗？要回答这个问题，让我们回溯到 Immutable X 的起源。

区块链项目通常从一项技术开始，然后通过逆向思维定义解决方案可以提供的最佳体验。在设计 Immutable X 时，我们询问“高质量的游戏项目需要什么才能成功”，然后从第一原则中着手实现这一目标。我们的核心信念是，原生代币的基本交易成本增加了用户和开发人员不必要的摩擦。如果您正在开发一款需要大众接受的游戏，那么这种摩擦可能是无法克服的。

要使用 Flow，你需要 FLOW。要使用Polygon，你需要 MATIC。每次铸币或交易资产时，都需要以网络中的基础货币支付费用。当然，这笔费用可能很小——但最大的阻力是让用户和开发人员在你链上的原生货币从 0 美元→> 0 美元。有一些变通方法（如元交易（metatransaction）），但它们变得更复杂了，并且仍然需要有人付费！此外，像 Solana 这样的区块链甚至要求你不断地以原生代币支付“状态租金”，防止你的资产随着时间的推移从链中被删除。

在 Immutable X 上，我们希望完全消除用户或开发人员拥有任何原生代币的要求。为了达到这一点，我们需要一个策略：

1. 对价值流动的交易（例如一级和二级销售）收取费用，并从价值本身而不是额外的原生代币中收取费用

2. 不要对没有价值流动的交易（例如铸币、转账）收取费用

3. 使用反垃圾邮件技术防止滥用这些免费交易

4. 用产生价值的交易费用补贴免费交易

前两点与我们的激励一致原则直接相关——如果你还没没有成功的销售记录，或者没有创造繁荣的用户对用户的经济，那么 Immutable 就不会赚钱。但是，我们是否还需要防御垃圾邮件——如何阻止有人铸币 10 亿资产并造成他人网络堵塞？这是我们的关键优势——因为我们从目标出发，能够选择最适合自己目标的技术。

Immutable 作为单个 zk-rollup “定序器”的角色为我们提供了许多 L1 区块链不可用的工具。因为所有交易都是由我们线性化的，所以我们可以实施所有标准化的web2 保护以防止垃圾交易——我们向合作伙伴发布 API 密钥、实施定制的速率限制、利用商业保护产品等，从而能够直接降低垃圾邮件的风险。

如果 Immutable 能够实施限制，这是否意味着网络是中心化的？我们要明白其中的重要区别，用户始终可以选择不信任，将其资产迁移到以太坊 L1，并在那里完成交易。相比之下，在最近 NFT 高交易量时期，一些 Solana 验证器向 Magic Eden 提议阻止交易，这将使用户无处可去（幸好验证器没有继续该提议）。对于游戏来说，在好的情况下接受单个定序器以获得最佳的用户体验，在最坏的情况下能够依赖以太坊共识，这提供了一个两全其美的办法。

好吧，这就是高级哲学——但你提及交易费用了吗？

用户对用户交易费用

我们认为，游戏 NFT 的未来包括所有价位的资产，类似于 CS:GO 和魔兽世界等现有游戏的资产经济。许多资产将以现有 NFT 交易中常见的奢侈品定价，但我们相信，从长远来看，绝大多数 NFT 将是中低价值的实用商品。我们是围绕支持这一部分的增长制定的定价策略。 Immutable 对所有二级交易收取 2% 的“协议费用”，每笔交易不收取固定费用。这使我们能够支持在固定成本gas费较高的平台上进行不合算的交易。

例如，许多 Gods Unchained 卡的价格约为 1.00 美元。而在以太坊上，OpenSea 上的标准 NFT 交易成本约为40 美元，gas 价格为 100 gwei，这使该资产的价值相形见绌。 在 Immutable X 上，交易该资产只会产生 0.02 美元的费用。 该协议可能不会从这些交易中最小的一笔获得巨额利润——但支持蓬勃发展的游戏需要一个存在各种价值资产的经济体。 Immutable 的目标是使整个经济尽可能具有流动性。 此外，提前知道确切的费用（尽可能不收取gas费）可以让你制定出更可靠的交易策略并更有效地做出一个市场。

根据交易数量获利的平台有一个目标，而且只有一个目标：增加交易数量。相比之下，Immutable 有明确的动机，以尽可能高的价格尽可能频繁地交易资产。如果 Immutable 想要赚更多的钱，我们应该开发能够创造高价值营业额的产品，比如新的交易类型，或者改善交易体验。这与我们的项目和建造者建立了极好的长期一致性。

没有上限的百分比会为高价值资产带来大笔费用吗？这当然是 NFT 生态系统和所有主要 NFT 市场的标准，例如OpenSea、LooksRare 和 Fractal 使用无上限的百分比费用——但我们希望 Immutable X 在费用方面明显优于其他平台，因此这是一个重要的考虑因素。同时，我们激励一致的核心原则也在这里发挥作用——我们希望受到激励，使这些资产更有价值，并让协议获得它在高端创造的部分价值！

我们正在积极探索这方面的几种选择，包括按比例收取费用、费用上限和不做任何改变，并将保持社区更新。我们还引入了最低交易金额——最初为 0.001 美元（或等值可替代的兑换货币）。我们目前还没有看到很多低于这一费用的 NFT 交易，打算在未来根据需求和我们的证明成本调整这个数字。

这种用户对用户交易的协议费用是我们本周启用的费用——但在 Immutable X 生态系统中还有其他类型的费用。

版税

Immutable X 允许铸币者为在协议范围内持续存在的资产设置版税。与许多平台不同，Immutable 支持在每笔资产的基础上自定义版税和收税人。这意味着同一集合中的不同资产可能有不同的费用——这些费用将在每个协议市场上得到尊重，并自动分配给最多 10 个收税人。与大多数 L1 生态系统相比，这是一次重大升级，即使在创建 EIP-2981 等标准之后，版税权通常在整个市场中都不受尊重。再者，我们的版税系统是基于我们从游戏开发者那里听到的反馈——为了支持蓬勃发展的 UGC 生态系统，或者为了满足复杂的许可证发行义务，游戏开发者需要这种灵活性来处理他们的资产。在 Immutable，我们喜欢版税，因为我们喜欢激励机制——我们希望游戏积极创造出鼓励资产周转的机制。

目前，我们仅支持在初始铸币后的较低水平的版税，以防止项目以后提高版税损害其用户。

市场费用

Immutable 收费策略的最后一部分是 Immutable 独有的。通过我们的协议订单簿（Protocol Orderbook），任何市场都可以为他们创建的任何订单附加费用。当这些订单在协议上的任何市场上完成时，原始市场仍可以收取费用。这种激励市场彼此共享流动性——你的订单更有可能在 100 多个其他市场上完成，而不是只能在你创建的市场上完成。这些费用没有限制——市场应该为他们的产品设定他们认为合理的费用！

Immutable 以协议的客户身份运行自己的市场，因此我们也是这个市场的参与者。我们将自己视为成功 IMX 市场的案例研究——不削减费用，只专注于为特定目标客户增加价值。全球订货记录确保市场可以提供更相关的用户体验，而不必担心流动性。

现在我们列出了许多不同的费用类型——所有这些费用不会导致客户价格上涨吗？是的，不一定。我们相信所有这些参与者只能收取他们可以证明的费用——如果他们增加了价值，用户就会付费。如果不是，用户将切换到其他市场或项目，直到这些费用降低。

初 级 销售

当然，许多 NFT 项目也从初级销售中获得价值（以及大部分早期收入）。我们认为将激励措施与该领域的游戏保持一致很重要，但我们再次希望解决游戏面临的具体挑战。游戏以各种复杂的方式铸造资产，包括通过土地销售、战利品式机制或直接销售。在其他区块链上，无论销售价格或资产数量如何，你都需要支付与这种销售机制的复杂性成比例的gas费。 但Immutable 的策略不同。

我们不像其他区块链那样按铸币费收取费用，而是从初级销售额中抽取较少的 2%，并允许无限制的免费铸币（使用前面讨论的垃圾邮件保护方式）。通过这种方式，在游戏里可以实现他们选择的任何游戏设计，并且（假设它们成功），我们都会赚钱。这通常会导致低价值、大批量铸币厂的成本更低，而高价值、小批量铸币厂的费用更高。 Immutable 还提供了许多工具来使初级销售更加成功，包括铸币 API、元数据存储和同步批量铸币。在别的区块链上，这是由其他工具（例如 NFT 端口）处理的，这些工具都要收取自己的费用。很难直接比较所有这些工具，而且它们解决的问题不同，价格也不同。这正是我们要提出的观点——你需要整体调用有助于项目成功的平台——如果 Immutable 比次佳平台增加了 2% 的销售额，那么使用 Immutable 并支付 2% 的费用将是一个可盈利的决定。

StarkNet

我们刚刚已经宣布，将在 StarkNet 上推出对 Immutable X 的支持，允许需要 EVM 兼容性/智能合约支持的项目加入我们的生态系统。在 StarkNet 上，情况会有所不同：每笔交易都需要支付 gas 费，因为 Immutable 将不再是系统中的单一定序器。当你使用我们的交易合约或平台（类似于 Solana 上的 Metaplex 或 L1 上的 0x）时，Immutable 的费用将被计入。

正如上周与社区分享的那样，我们还为 NFT 创建了版税标准。不幸的是，由于 StarkNet 支持自定义智能合约，高级用户可以避免这些费用。尽管如此，通过定义版税的早期标准并直接与市场合作，我们的目标是确保 StarkNet 生态系统从一开始就尊重创作者的版税。

我们有信心让所有最好的游戏和市场选择通过 Immutable X 进行交易。 Immutable 产品的深度垂直整合将确保通过我们的协议始终获得游戏和市场的最佳交易体验，即使需要我们帮助游戏在不同的rollup上发布。

总结

从第一天开始，我们对 Immutable X 的目标就一直如此：允许团队构建世界级的 web3 游戏，而不会因技术限制而使他们的愿景妥协。廉价 gas 绝对是一种改进——但完全消除 gas 是 Immutable X 特有的UX 和开发者体验的巨大优势。

Immutable 独特的定位和专注让我们能够更有效地解决像游戏这样高质量、大规模的 NFT 项目的核心问题。全面定价和收费的目标是使我们的激励措施与这些项目及其用户保持一致，然后利用收费获得的收入进行再投资，让它们取得更大的成功。

非常高兴能在本月推出协议费用和质押，未来我们将继续试验费用结构，以最大限度地提高协议的长期交易量和权益奖励池的规模。
